    工作流 26：从活动素材到有温度的宣传故事的 AI 多专家工作流¶

    适用岗位：工会干部、宣传员、企业文化岗、HR 品牌

    真实场景：一场职工运动会刚结束，或者一位劳模刚受到表彰，领导要求三天内产出新闻稿、短视频脚本和公众号推文。你手头只有一堆照片、一份活动方案和几条采访时记下的零散笔记。

    最终目标：产出一套有温度、有细节、有真实人物的宣传内容包，包括新闻稿、短视频脚本、发布标题和宣传总结。

    输入材料清单¶

    开始前请准备以下材料：

    • 活动基本信息（时间、地点、参与人数、活动名称）
    • 采访记录或现场笔记（哪怕只有几句话也算）
    • 参与人物信息（姓名、岗位、一句话背景）
    • 活动照片说明（描述几张有代表性照片里有什么）
    • 活动主题或口号（如有）

    工作流总览¶

    节点 1：活动素材整理专家
  ↓ 输出：结构化素材包（时间线 + 人物列表 + 关键事件）
节点 2：人物故事挖掘专家
  ↓ 输出：人物故事提案（含人物、背景、变化、细节）
节点 3：情绪主线设计专家
  ↓ 输出：情绪主线方案（主情绪 + 叙事角度 + 核心金句方向）
节点 4：新闻稿生成专家
  ↓ 输出：新闻稿（800 字左右）
节点 5：短视频脚本专家
  ↓ 输出：60 秒短视频脚本
节点 6：标题与金句专家
  ↓ 输出：标题组（5 条）+ 金句组（3 条）
节点 7：真实性审查专家（跨内容全局审查）
  ↓ 输出：真实性审查报告
节点 8：宣传复盘专家
  ↓ 输出：复盘报告 + 改进清单
最终交付：完整宣传内容包

    专家节点详解¶

    每个节点包含六部分：节点定位、输入与输出、使用顺序、提示词包（A 快速 / B 专家 / C 自查 / D 返修）、交付给下游节点、人工验收清单。

    关于节点 7（真实性审查专家）：这是一个跨内容的全局合规审查节点，负责对新闻稿和短视频脚本进行事实核查，与其他节点的"自查审稿版"性质不同——它是一个独立的审查节点，而不是某个节点的内部模式。

    其他节点的 C 自查审稿版和 D 返修优化版是当前节点内部的工作模式，不是新的专家角色。

    节点 1：活动素材整理专家¶

    1.1 节点定位¶

    把散乱的活动素材整理成结构化的素材包，建立时间线、人物档案和关键事件清单，为后续所有节点提供统一的素材底座。素材整理质量直接决定人物故事挖掘的深度——没有好的素材包，人物故事节点挖掘不出真实细节。

    1.2 输入与输出¶

    输入：活动基本信息 + 采访笔记 + 照片说明 + 人物信息

    输出：结构化素材包（时间线 + 人物档案 + 关键事件清单 + 可用细节清单）

    1.3 使用顺序¶

    1. 先用「快速生成版」得到结构化素材包初稿。
    2. 需要深度挖掘故事线索时，改用「专家增强版」。
    3. 用「自查审稿版」检查人物档案是否有具体言行、时间线是否完整。
    4. 有问题则用「返修优化版」补充。
    5. 对照 1.6 验收清单确认，通过后将素材包交给节点 2 和节点 4。

    1.4 提示词包¶

    A. 快速生成版¶
    请将以下活动素材整理成结构化的素材包。

活动素材：【粘贴所有原始素材，包括笔记、采访记录、照片描述】

输出：
1. 活动时间线（按时间顺序列出关键事件）
2. 人物档案（姓名、岗位、在活动中的角色、有记录的言行）
3. 关键事件清单（最有故事性的 3-5 个瞬间）
4. 可用细节清单（具体的数字、场景描述、人物表情或行为）
    B. 专家增强版¶
    你是一位纪实报道编辑，擅长从零散素材中提炼故事线索。

任务：对以下活动素材进行系统整理，重点挖掘可以讲故事的材料。

输入：
- 活动素材：【粘贴所有原始素材】

整理维度：
1. 时间线（活动前/活动中/活动后，各有哪些关键节点）
2. 人物档案（每个出现的人：姓名、岗位、背景、具体行为或发言）
3. 情绪变化点（哪个瞬间有情绪转折，例如：原本紧张的选手赢得了比赛）
4. 可视化细节（可以用画面呈现的具体场景）
5. 数据亮点（参与人数、获奖情况、持续年限等有说服力的数字）

输出：素材包（分类整理）+ 遗漏信息清单（需要补充采访的问题）
    C. 自查审稿版¶

    此为当前节点的自查模式，不是新的专家角色。

    请检查以下素材包是否存在以下问题：
1. 是否有人物只有名字和岗位，没有任何具体言行记录？
2. 时间线是否有明显的空白段（某个时段完全没有内容）？
3. 是否缺少具体数字或可量化的信息？
4. 关键事件是否有不够具体、无法还原场景的描述？

【粘贴素材包】

逐条说明问题，提示需要补充什么材料。
    D. 返修优化版¶
    根据自查意见，补充素材包中的缺失内容。对于无法补充的信息，标注"待补充"并说明如何获取。

原始素材包：【粘贴原始素材包】
自查意见：【粘贴自查结果】

    1.5 交付给下游节点¶

    将完整素材包（含时间线、人物档案、关键事件、可用细节）复制，同时交给：

    • 节点 2（人物故事挖掘专家）：用于挖掘人物故事。
    • 节点 4（新闻稿生成专家）：作为新闻稿的事实底座。

    1.6 人工验收清单¶

    • [ ] 是否至少有 2 个人物有具体的言行记录（不只是名字和岗位）？
    • [ ] 关键事件是否有具体的场景描述（能想象出画面）？
    • [ ] 是否有至少 1 个可用的具体数字？

    节点 2：人物故事挖掘专家¶

    2.1 节点定位¶

    从整理好的素材中找出最有故事感的人物线索，提炼人物的背景、变化和关键细节，为宣传内容提供真实的人物支撑。没有真实人物的工会宣传，读完留不下任何印象。

    2.2 输入与输出¶

    输入：节点 1 的结构化素材包

    输出：人物故事提案（1-2 个人物，含背景、参与动机、活动中的变化、关键细节、故事方向）

    2.3 使用顺序¶

    1. 先用「快速生成版」得到人物故事提案初稿。
    2. 需要深度挖掘人物故事弧线时，改用「专家增强版」。
    3. 用「自查审稿版」检查故事是否有真实变化、细节是否具体。
    4. 有问题则用「返修优化版」调整。
    5. 对照 2.6 验收清单确认，通过后将故事提案交给节点 3、节点 4、节点 5。

    2.4 提示词包¶

    A. 快速生成版¶
    请根据以下素材包，找出最有故事感的 1-2 个人物，并为每个人物提炼一个故事方向。

素材包：【粘贴节点 1 的素材包】

对每个人物输出：
- 人物背景（2 句话）
- 参与活动的动机或代表性（为什么他/她特别值得写）
- 活动中发生了什么变化（从哪里到哪里）
- 最打动人的一个细节（具体的言行或瞬间）
- 故事方向（一句话概括这个故事讲什么）
    B. 专家增强版¶
    你是一位人物特稿编辑，专注于发现普通人身上的故事价值。

任务：从以下素材包中识别最具故事潜力的人物，提炼完整的人物故事方案。

素材包：【粘贴节点 1 的素材包】

人物评估标准：
1. 有变化（参与活动前后有什么不同）
2. 有细节（有具体的可还原的场景）
3. 有代表性（这个人代表一类群体，读者容易产生共鸣）
4. 不只是获奖代表（最好是普通参与者，而不只是官方代言人）

对推荐人物输出：
- 人物小传（100 字以内，第三人称）
- 故事弧线（背景 → 触发点 → 活动中的关键时刻 → 变化或收获）
- 可用细节清单（可以直接写入文章或脚本的具体描述）
- 适合切入的叙事角度（例如：从他的手说起 / 从那句话说起）
    C. 自查审稿版¶

    此为当前节点的自查模式，不是新的专家角色。

    请检查以下人物故事提案是否存在以下问题：
1. 故事是否过于"高大全"，缺少普通人的真实感（只有荣誉没有挣扎）？
2. 细节是否足够具体（能还原画面，还是只是描述性词语）？
3. 故事弧线是否有明确的变化（从什么状态到什么状态）？

    节点 7：真实性审查专家¶

    7.1 节点定位¶

    这是一个跨内容的全局合规审查节点，负责对新闻稿和短视频脚本进行事实核查，确保所有内容有素材支撑、没有编造或夸大。与其他节点的"自查审稿版"不同，真实性审查针对的是跨节点的内容一致性和事实准确性，任何节点自查无法覆盖的"全局性失真问题"都在本节点处理。

    7.2 输入与输出¶

    输入：节点 4 新闻稿 + 节点 5 短视频脚本 + 节点 1 原始素材包（用于比对事实）

    输出：真实性审查报告（问题清单 + 严重程度分级 + 通过/需修改结论）

    7.3 使用顺序¶

    1. 先用「快速生成版」做基础事实核查。
    2. 需要深度审查情感夸大、代表性问题和遗漏信息时，改用「专家增强版」。
    3. 用「自查审稿版」复查审查报告本身是否遗漏问题（可选）。
    4. 审查结论为"通过"后，将内容包交给节点 6 发布；结论为"需修改后通过"时，必须返回节点 4 或节点 5 修改后重新审查，不允许直接发布。
    5. 对照 7.6 验收清单确认。

    7.4 提示词包¶

    A. 快速生成版¶
    请对以下宣传内容做真实性审查，对照原始素材检查是否有超出素材的编造或夸大。

原始素材包：【粘贴节点 1 的素材包】
新闻稿：【粘贴新闻稿全文】
短视频脚本：【粘贴脚本全文】

检查项：
1. 所有数字是否能在素材包中找到来源？
2. 人物描述是否超出了素材记录的范围？
3. 是否有"一定""必然""从不"等过于绝对的表达？
4. 是否有无法核实的情感描述？

输出：问题清单（标注具体位置）+ 通过/需修改结论
    B. 专家增强版¶
    你是一位新闻审核编辑，专门在内容发布前识别所有可能引发质疑的内容。

任务：对以下宣传内容做完整的真实性审查。

输入：
- 原始素材：【粘贴节点 1 的素材包】
- 新闻稿：【粘贴新闻稿】
- 短视频脚本：【粘贴脚本】

审查维度：
1. 事实准确性（数字、时间、地点、人名是否正确）
2. 合理推断边界（哪些内容是基于素材的合理推断，哪些是超出素材的创作）
3. 情感夸大检查（情感描述是否有素材支撑，还是 AI 自动渲染的情绪）
4. 代表性问题（这个人物是否能真实代表所描述的群体）
5. 遗漏问题（是否有重要的负面或复杂情况被刻意省略）

输出：

- 总体评估（通过 / 需修改后通过 / 建议重写）
- 修改优先级建议
    C. 自查审稿版（可选）¶

    此为当前节点的自查模式。真实性审查本身是跨节点的全局审查，此自查用于检查审查报告自身是否遗漏了重要问题。

    请检查以下真实性审查报告是否存在以下问题：
1. 审查是否只检查了明显的数字错误，遗漏了情感夸大问题？
2. 是否有已标注"建议优化"的问题实际上应该标为"需修改"？

【粘贴真实性审查报告】
    D. 返修优化版¶
    根据真实性审查报告中的问题，修改对应的宣传内容。删除无法核实的描述，用更保守的表达替换夸大表述。

原始内容：【粘贴需要修改的新闻稿段落或脚本段落】
审查报告：【粘贴审查结果】
可补充的真实细节（如有）：【填入补充材料】

    7.5 交付给下游节点¶

    审查结论为"通过"后：

    • 将审查结论通知节点 6（标题金句专家），确认内容可以使用该标题发布。
    • 通知节点 8（宣传复盘专家）内容已通过审查，可以发布并开始记录传播数据。

    审查结论为"需修改后通过"时：返回节点 4（新闻稿）或节点 5（脚本）修改，修改完成后必须重新送审，不允许直接发布。

    7.6 人工验收清单¶

    • [ ] 所有出现的数字是否都能在原始素材中找到来源？
    • [ ] 人物情感描述是否有素材支撑（或明确标注为合理推断）？
    • [ ] 真实性审查结论是否为"通过"（结论为"需修改后通过"时，必须返回修改后重新审查，不算通过）？

    节点 8：宣传复盘专家¶

    8.1 节点定位¶

    设计活动宣传的复盘框架，评估宣传效果，提炼可复用的经验和下次改进的具体建议。复盘的目的是提炼可以下次直接用的经验，而不是评价本次好不好。

    8.2 输入与输出¶

    输入：发布后的反馈数据或主观评估 + 本次宣传内容包

    输出：宣传复盘报告 + 可复用资产清单 + 下次改进清单（3-5 条具体操作建议）

    8.3 使用顺序¶

    1. 先用「快速生成版」得到复盘报告初稿。
    2. 需要系统性分析可复用资产时，改用「专家增强版」。
    3. 用「自查审稿版」检查分析是否有具体依据、改进建议是否可操作。
    4. 有问题则用「返修优化版」完善。
    5. 对照 8.6 验收清单确认，复盘完成后归档本次经验。

    8.4 提示词包¶

    A. 快速生成版¶

输出：
1. 目标完成情况
2. 做得好的地方（2-3 条，说明原因）
3. 需要改进的地方（2-3 条，说明原因）
4. 下次改进建议（3 条具体操作建议）
    B. 专家增强版¶
    你是一位企业内容运营复盘专家。

输出：复盘报告 + 可复用资产清单 + 改进行动清单
    C. 自查审稿版¶

    常见错误¶

    错误 1：跳过人物故事，直接写新闻稿

    表现：新闻稿全是活动概况，没有人物，读完不知道记住了谁，也感受不到温度。

    修复：必须先完成节点 2，确定人物故事和可用细节，再进入节点 4。没有人物的工会宣传稿是失败的。

    错误 2：真实性审查跳过或走形式

    表现：内容里出现了素材里没有的数字，或者人物说了素材里没有记录的话。

    修复：每次完成节点 4 和节点 5 后，必须执行节点 7 的真实性审查，逐条比对素材包。

    错误 3：金句变成口号

    表现："为职工服务，为企业发展贡献力量"这类金句无法打动任何人，只是完成任务。

    修复：金句必须来自真实人物的真实表达，或者真实场景的具体描述。如果想不到，说明节点 2 的细节挖掘还不够深。

    人工验收清单¶

    • [ ] 新闻稿里是否有至少一个真实人物，并有具体细节（而不只是职务和荣誉）？
    • [ ] 通读新闻稿，是否没有空洞套话（如有，必须返回修改）？
    • [ ] 短视频脚本是否完全没有口号式台词？
    • [ ] 所有数字是否都在素材包里有来源？
    • [ ] 真实性审查是否通过？

    延伸玩法¶

    • 变体 1：系列化：用节点 2 挖掘出的多个人物，为同一次活动做成系列报道，每期聚焦一个人。
    • 变体 2：年度总结版：将全年多次活动的节点 1 素材包汇总输入，用节点 3 设计年度情绪主线，生成年度宣传总结。
    • 进阶组合：与"工作流 01（短视频内容创作工作流）"结合，用本工作流产出的短视频脚本，进入工作流 01 的节点 5（分镜设计），完成从脚本到视频的全链路生产。
