    工作流 24：从票据到报销申请的 AI 多专家财务报销工作流¶

    适用岗位：行政人员、财务专员、各部门报销经办人、财务主管

    真实场景：月底报销季来了，员工手上一堆票据——有发票、有收据、有网购截图、有境外小票——需要在财务截止日前整理好、归类对、填写准，还要避免被财务退回反复折腾。

    最终目标：一套完整的报销交付物，包含：票据整理清单、费用归类表、异常票据识别报告、报销申请表草稿、附件整理说明、审核预检清单。

    ⚠️ 合规声明：本工作流严禁编造或推断任何票据信息。所有票据字段（金额、日期、商家名称、税号）必须来自票据原件。无法确认的信息一律标注「需人工核实」，不得自行填写。

    输入材料清单¶

    • 所有票据原件（发票、收据、网购截图、电子票据，扫描件或照片均可）
    • 报销单位的费用分类标准（如有，提供财务科目对照表）
    • 差旅/活动背景说明（出差目的、活动名称、参与人员）
    • 公司报销政策（标准餐补、住宿上限、差旅标准等）
    • 历史报销记录（如有，用于对照）

    工作流总览¶

    节点 1：票据信息提取专家
  ↓ 输出：票据信息清单（字段化）
节点 2：费用归类专家
  ↓ 输出：费用分类对照表
节点 3：异常票据识别专家（招牌节点）
  ↓ 输出：异常票据报告（分级标注）
节点 4：政策合规检查专家
  ↓ 输出：合规检查报告
节点 5：报销申请表生成专家
  ↓ 输出：报销申请表草稿
节点 6：附件整理说明专家
  ↓ 输出：附件清单与整理说明
节点 7：财务审核预检专家
  ↓ 输出：审核预检清单
节点 8：退回问题处理专家
  ↓ 输出：退回问题处理记录
节点 9：报销归档专家
  ↓ 输出：报销归档记录
最终交付：完整报销申请包

    专家节点详解¶

    每个节点包含六部分：节点定位、输入与输出、使用顺序、提示词包（A 快速 / B 专家 / C 自查 / D 返修）、交付给下游节点、人工验收清单。

    C 自查审稿版和 D 返修优化版是当前节点内部的工作模式，不是新的专家角色。

    核心原则：票据信息只来自票据原件。无法识别的字段标注「需人工核实」，不猜测，不填写默认值，不编造。

    节点 1：票据信息提取专家¶

    1.1 节点定位¶

    将每张票据的关键字段逐一提取为结构化信息，为后续归类和审核打基础。票据信息混乱或不完整是报销被退回的首要原因，先提取清楚才能准确判断。

    1.2 输入与输出¶

    输入：所有票据原件（扫描件/照片/电子票据）

    输出：票据信息清单（每张票据一行：编号、类型、日期、金额、商家、税号、票据状态）

    1.3 使用顺序¶

    1. 先用「快速生成版」批量提取票据字段。
    2. 票据清晰度较差或字段复杂时，用「专家增强版」。
    3. 用「自查审稿版」检查是否有推断填写的字段。
    4. 有问题则用「返修优化版」修正。
    5. 对照 1.6 验收清单确认，通过后交给节点 2 和节点 3。

    1.4 提示词包¶

    A. 快速生成版¶
    你是票据信息提取专家。根据以下票据描述，逐张提取关键字段，输出表格：票据编号 | 票据类型 | 开票日期 | 金额（含税）| 商家/供应商名称 | 税号（如有）| 票据状态（清晰/模糊/残缺）。无法从票据识别的字段填写「需人工核实」，不得推断或填写默认值。
票据信息：【粘贴每张票据的描述或 OCR 识别内容，逐张标编号】
    B. 专家增强版¶
    你是票据字段提取专家，专注于将各类票据转化为结构化可核查的信息记录。

输入：票据原件描述（OCR 识别结果或人工描述）

对每张票据提取以下字段：
1. 票据编号（自编，如 R001、R002）
2. 票据类型：增值税专用发票 / 增值税普通发票 / 收据 / 网购截图 / 电子发票 / 境外票据 / 其他
3. 开票日期（年/月/日，格式统一）
4. 金额（含税总金额，单位元，保留两位小数）
5. 商家/供应商名称（票面原文）
6. 税号（如票面有，原文抄录；无则填「无」）
7. 票据备注（票面注明的用途，如有）
8. 票据状态：清晰 / 模糊（需人工确认）/ 残缺（关键字段缺失）

约束：
- 所有字段必须来自票据原件，不得推断
- 无法识别的字段一律填「需人工核实」，不填猜测值
- 金额模糊时填「需人工核实」，不按常见金额推测

输出：结构化表格，每张票据一行
    C. 自查审稿版¶

    此为当前节点的自查模式，不是新的专家角色。

    请检查以下票据信息清单：
① 是否有任何字段是推断填写的（不是来自票据原件）？
② 模糊或无法识别的字段是否已标注「需人工核实」？
③ 金额是否保留两位小数？
④ 日期格式是否统一？
【粘贴清单内容】
逐条列出问题。
    D. 返修优化版¶
    根据自查意见修正票据清单。保留：来自票据原件的准确字段。修正：将推断值替换为「需人工核实」、统一日期格式、修正金额格式。
原始清单：【粘贴原文】
自查意见：【粘贴自查结果】

    1.5 交付给下游节点¶

    将票据信息清单交给节点 2（费用归类）和节点 3（异常识别）。标注所有「需人工核实」项，这些项在报销提交前必须由经办人手动确认。

    1.6 人工验收清单¶

    • [ ] 每张票据都有编号且不重复
    • [ ] 所有字段来自票据原件，无推断值
    • [ ] 无法识别的字段已标注「需人工核实」
    • [ ] 金额格式统一（两位小数，单位元）
    • [ ] 日期格式统一

    节点 2：费用归类专家¶

    2.1 节点定位¶

    将每张票据归入对应的费用科目。归类错误会导致会计科目不平、报销被退回、甚至影响税务处理。归类标准以公司财务科目对照表为准，无对照表时按常规科目处理并标注供财务确认。

    2.2 输入与输出¶

    输入：票据信息清单（节点 1）+ 公司费用分类标准（如有）

    输出：费用分类对照表（票据编号 → 建议科目 → 归类依据 → 需确认项）

    2.3 使用顺序¶

    1. 先用「快速生成版」生成初版分类。
    2. 科目较复杂或有混合消费票据时，用「专家增强版」。
    3. 用「自查审稿版」检查归类是否有依据。
    4. 有问题则用「返修优化版」修正。
    5. 对照 2.6 验收清单确认，财务人员复核后交给节点 4。

    2.4 提示词包¶

    A. 快速生成版¶
    你是费用归类专家。根据以下票据清单，将每张票据归入对应费用科目（差旅费/餐费/办公用品/交通费/住宿费/会议费/培训费/其他），给出归类依据（票据类型+商家名称）。无法确定科目的标注「需财务确认」，不强行归类。
票据清单：【粘贴节点 1 清单】
费用科目标准：【粘贴公司标准，或填「使用常规科目」】
    B. 专家增强版¶
    你是财务费用归类专家，专注于将票据准确归入对应会计科目。

输入：票据信息清单 + 公司费用分类标准

执行步骤：
1. 逐张归类：基于票据类型、商家名称、票面用途，归入最匹配的费用科目
2. 归类依据：每条说明"根据XX（票面信息）归入XX科目"
3. 混合消费处理：一张发票含多类费用时，标注「混合消费，需拆分或人工确认」

    节点 3：异常票据识别专家（招牌节点）¶

    3.1 节点定位¶

    在提交报销前，主动识别可能导致退回或合规风险的异常票据。异常识别不是为了"抓人"，而是让经办人在提交前就知道哪些票据需要补充说明、哪些需要替换、哪些可能无法报销——避免提交后被财务退回反复折腾。

    识别边界：异常识别基于票据本身的可观察特征（字段缺失、格式问题、逻辑矛盾）和公司报销政策。不对票据真实性做最终判断，存疑票据一律标注「需人工核实」，由人工决定是否接受。严禁基于推断编造票据问题，也严禁对存疑票据擅自标注为「伪造」。

    3.2 输入与输出¶

    输入：票据信息清单（节点 1）+ 公司报销政策

    输出：异常票据报告（分三级：关注 / 需补充说明 / 建议退回，每项附具体异常描述）

    3.3 使用顺序¶

    1. 先用「快速生成版」快速扫描明显异常。
    2. 用「专家增强版」系统检查全部异常类型。
    3. 用「自查审稿版」确认异常标注有依据且分级准确。
    4. 有问题则用「返修优化版」修正。
    5. 对照 3.6 验收清单确认，将报告交给经办人处理后再提交报销。

逐一检查以下异常类型：

**字段类异常：**
1. 日期异常：票据日期晚于报销日期、票据日期与出差/活动时间不符
2. 金额异常：金额字段为「需人工核实」（无法读取）、金额明显异常（如住宿单日超过政策上限）
3. 税号缺失：增值税发票缺少税号（影响税务抵扣）

**标注规则：**
- 关注：问题轻微，附说明后通常可报销

    7.1 节点定位¶

    在正式提交财务审核前，模拟财务审核视角做最后检查。财务退回的常见原因都是可提前规避的——本节点是最后一道自检关卡。

    7.2 输入与输出¶

    输入：报销申请表（节点 5）+ 附件清单（节点 6）+ 异常票据报告（节点 3）+ 合规报告（节点 4）

    输出：审核预检清单（通过项 / 待确认项 / 建议修正项）

    7.3 使用顺序¶

    1. 用「快速生成版」做最终扫描。
    2. 用「专家增强版」系统检查所有常见退回原因。
    3. 用「自查审稿版」确认无遗漏。
    4. 有问题则用「返修优化版」补全。
    5. 对照 7.6 验收清单确认，全部通过后提交。

    7.4 提示词包¶

    A. 快速生成版¶
    你是财务审核预检专家。根据以下报销材料，检查常见退回原因：①票据完整性（是否有缺失）②金额一致性（申请表与票据金额是否一致）③政策符合性（是否有超标未处理的项）④附件齐全性（是否有申请表提到的票据未附上）。每项给出通过/待确认/建议修正。
报销材料摘要：【粘贴申请表和清单关键信息】
    B. 专家增强版¶
    你是财务报销审核预检专家，从财务审核视角检查常见退回原因。

检查清单（逐一判断）：
1. [ ] 申请表信息完整（申请人/部门/日期/事由/签字位置已预留）
2. [ ] 费用明细与附件票据一一对应（无多报无漏附）
3. [ ] 合计金额与明细加总一致
4. [ ] 无超标未处理项（超标已获特批或已移除）

6. [ ] 境外票据已附翻译或说明

    此为当前节点的自查模式，不是新的专家角色。

【粘贴清单内容】

    D. 返修优化版¶
    根据自查意见补全预检清单。保留：已通过的检查项。修正：补充遗漏检查项、为待确认项添加具体说明。
原始清单：【粘贴原文】
自查意见：【粘贴自查结果】

    7.5 交付给下游节点¶

    全部通过后提交财务审核。如有退回，进入节点 8 处理。

    • [ ] 10 项检查全部完成
    • [ ] 无「建议修正」项遗留
    • [ ] 所有「需人工核实」已处理
    • [ ] 经办人已签字确认

    节点 8：退回问题处理专家¶

    8.1 节点定位¶

    财务退回后，快速定位退回原因、生成处理方案，避免反复退回。退回处理越快，对经办人和财务双方效率越高。

    8.2 输入与输出¶

    输入：财务退回说明 + 原报销申请材料

    输出：退回问题处理记录（问题分析 + 处理方案 + 修改后的材料清单）

    8.3 使用顺序¶

    1. 收到退回说明后，用「快速生成版」分析原因。
    2. 退回原因较复杂或涉及票据替换时，用「专家增强版」。
    3. 用「自查审稿版」确认处理方案完整。
    4. 处理完毕后用「返修优化版」更新材料。
    5. 对照 8.6 验收清单，重新过一遍节点 7 后提交。

    8.4 提示词包¶

    A. 快速生成版¶
    你是退回问题处理专家。根据以下财务退回说明，分析退回原因、给出处理方案：①退回的具体问题（引用退回说明原文）②需要修改的材料（说明修改内容）③需要替换或补充的票据 ④重新提交前需完成的步骤。不得在处理中编造新的票据信息。
退回说明：【粘贴财务退回说明】
    B. 专家增强版¶
    你是财务退回处理专家，专注于快速定位和解决报销退回问题。

输入：退回说明 + 原申请材料

执行步骤：
1. 退回原因分类：字段错误 / 票据问题 / 政策超标 / 附件缺失 / 流程问题
2. 逐一处理方案：每个问题的具体处理步骤（修改申请表 / 补充附件 / 替换票据 / 获取特批）
3. 无法处理的问题：部分退回原因无法在经办人层面解决（如票据已遗失），标注「需上级决定」
4. 重新提交清单：列出修改后需重新确认的所有材料

约束：处理方案不得包含编造票据信息的步骤；票据遗失时说明「该笔费用可能无法报销」而非「补开发票」。

输出：处理记录 + 修改后材料清单

自查意见：【粘贴自查结果】

    8.5 交付给下游节点¶

    处理完成后，将修改后的材料重新过一遍节点 7（预检），通过后再次提交。最终通过后进入节点 9 归档。

    • [ ] 修改后材料已重新过节点 7 预检

    节点 9：报销归档专家¶

    9.1 节点定位¶

    报销审核通过后，整理归档记录，形成可追溯的报销档案。良好的归档习惯在税务检查或内部审计时能快速提供依据。

    9.2 输入与输出¶

    输入：已审核通过的报销申请全套材料

    输出：报销归档记录（归档编号 + 归档内容清单 + 存档位置）

    9.3 使用顺序¶

    1. 审核通过后用「快速生成版」生成归档记录。
    2. 涉及大额或多批次报销时，用「专家增强版」。
    3. 对照 9.6 验收清单确认，存入档案系统。

    常见错误¶

    错误 1：模糊字段自行填写

    表现：票据金额模糊，经办人凭印象填了一个"差不多"的金额。

    修复：节点 1 所有无法确认的字段一律标注「需人工核实」，金额不确认不提交。

    错误 2：票据归类错误导致会计科目不平

    表现：出租车费被归入差旅费（合理），但高铁票也被归入交通费（应是差旅费），导致科目拆分不一致。

    修复：节点 2 的归类标准以公司财务科目对照表为准；无对照表时，向财务确认后再填写。

    错误 3：异常票据未处理就提交

    表现：节点 3 识别出有一张日期是三个月前的发票（超出报销有效期），但经办人没有处理就一并提交，导致整份报销被退回。

    修复：节点 3 的异常报告中「建议退回」等级的票据，必须在提交前由经办人决定处理方式，不得跳过。

    错误 4：合计金额与票据不一致

    表现：申请表合计 3,240 元，但票据加总实际是 3,420 元，导致被财务发现退回。

    修复：节点 5 的自查审稿版强制做加总验算，任何差异都必须修正后才能提交。

    延伸玩法¶

    • 差旅报销批量处理：将节点 1-3 应用于整个团队的差旅票据，统一提取后再分人归类，提高团队报销效率。
    • 报销政策快速查询：将公司报销政策整理成节点 4 的政策对照表，在节点 4 的 B 专家增强版中常备，每次报销直接调用。
    • 与工作流 16 组合：将会议纪要（工作流 16）中记录的会议费用和参与人员信息，作为节点 5 报销申请表的事由说明依据，一致性更高，审批通过率更高。
