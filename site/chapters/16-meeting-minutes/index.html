    工作流 16：从会议录音或文字到可追踪纪要的 AI 多专家工作流¶

    适用岗位：行政人员、项目专员、部门助理、会议组织者、团队负责人

    真实场景：一场一小时的会议刚结束，你手上是一份录音转文字稿（或自己记的零散笔记），需要在当天整理出一份能发给所有参会人、能被追踪执行的会议纪要——而不是把发言流水账复制一遍。

    最终目标：一份可追踪的会议纪要，包含：会议基本信息、议题清单、关键观点、明确区分的决策事项、可执行的待办任务（含责任人和时间）、风险与争议点，以及会后追踪清单。

    输入材料清单¶

    • 会议记录（录音转文字稿 / 速记 / 个人笔记，任一形式）
    • 会议基本信息（会议主题、时间、参会人名单、主持人）
    • 会议目的（如已知：决策会 / 同步会 / 头脑风暴）
    • 已有的会议议程（如有）
    • 需要重点关注的事项（如有特定关切）

    工作流总览¶

    节点 1：会议材料整理专家
  ↓ 输出：清洗后的结构化会议记录
节点 2：议题识别专家
  ↓ 输出：议题清单（按讨论顺序）
节点 3：发言观点提取专家
  ↓ 输出：各议题下的关键观点（标注发言人）
节点 4：决策事项提取专家（招牌节点）
  ↓ 输出：分级的决策事项清单
节点 5：待办任务拆解专家
  ↓ 输出：可执行待办清单
节点 6：责任人与时间节点校准专家
  ↓ 输出：补全责任人和时间的待办清单
节点 7：风险与争议点识别专家
  ↓ 输出：风险与争议清单
节点 8：纪要格式生成专家
  ↓ 输出：完整会议纪要
节点 9：会后追踪清单专家
  ↓ 输出：会后追踪清单
最终交付：可追踪会议纪要

    专家节点详解¶

    每个节点包含六部分：节点定位、输入与输出、使用顺序、提示词包（A 快速 / B 专家 / C 自查 / D 返修）、交付给下游节点、人工验收清单。

    C 自查审稿版和 D 返修优化版是当前节点内部的工作模式，不是新的专家角色。

    核心原则：纪要必须区分事实、观点、决策、待办、风险。不能把所有发言都写成"会议决定"，也不能编造会议没有产生的结论。

    节点 1：会议材料整理专家¶

    1.1 节点定位¶

    将杂乱的录音转文字稿或零散笔记整理为结构清晰、可逐段处理的会议记录。原始记录常有口语重复、跑题、口误，先清洗才能准确提取信息。

    1.2 输入与输出¶

    输入：原始会议记录 + 会议基本信息

    输出：清洗后的结构化会议记录（去除口语冗余，保留完整信息，标注发言人）

    1.3 使用顺序¶

    1. 先用「快速生成版」清洗记录。
    2. 记录混乱或多人交叉发言时，改用「专家增强版」。
    3. 用「自查审稿版」检查是否丢失了关键信息、是否擅自归纳。
    4. 有问题则用「返修优化版」修正。
    5. 对照 1.6 验收清单确认，通过后交给节点 2 和节点 3。

    1.4 提示词包¶

    A. 快速生成版¶
    你是一位会议记录整理员。请整理以下会议记录。

原始记录：【粘贴录音转文字或笔记】
参会人：【填入名单】

要求：
1. 去除口语重复、语气词、明显口误
2. 保留所有实质信息，不归纳不删减观点
3. 标注每段发言的发言人（能识别时）
4. 标记听不清或不确定的部分
    B. 专家增强版¶
    你是一位专业会议记录整理员。

任务：将原始会议记录清洗为结构清晰的可处理记录。

输入：
- 原始记录：【粘贴录音转文字或笔记】
- 参会人名单：【填入】
- 会议主题：【填入】

处理步骤：
1. 去除口语冗余（重复、语气词、跑题闲聊），但保留全部实质内容
2. 按发言人归集发言（标注谁说了什么）
3. 对识别不清的发言人标注"发言人不明"，不臆测
4. 标记不确定/听不清的部分（用【存疑】标注），不脑补
5. 保持时间顺序，便于后续追溯

输出格式：
- 结构化记录（按时间顺序，每段标注发言人）
- 存疑清单（听不清/发言人不明的部分）

约束（关键）：只做清洗和归集，不做归纳、不下结论、不补充原文没有的内容；不确定的一律标注，绝不脑补。

质量标准：清洗后的记录信息无损失，后续节点可据此准确提取议题、观点、决策。

常见失败情况：清洗时擅自归纳导致观点失真；脑补听不清的内容；把闲聊误删但其实含有效信息。
    C. 自查审稿版¶

    此为当前节点的自查模式，不是新的专家角色。

    请检查以下清洗后的记录是否存在以下问题：
1. 是否在清洗时擅自归纳或下了原文没有的结论？
2. 是否脑补了听不清的内容（应标注【存疑】而非编造）？
3. 是否丢失了某些看似闲聊但含有效信息的内容？
4. 发言人标注是否准确（不确定的是否标"发言人不明"）？

【粘贴清洗后的记录】

逐条说明问题。
    D. 返修优化版¶
    根据自查意见修正清洗记录。还原被擅自归纳的观点，将脑补内容改为【存疑】标注，补回误删的有效信息，修正发言人标注。

原始清洗记录：【粘贴原始记录】
自查意见：【粘贴自查结果】

    1.5 交付给下游节点¶

    将清洗后的结构化记录复制，交给节点 2（议题识别专家）和节点 3（发言观点提取专家）。

    1.6 人工验收清单¶

    • [ ] 是否保留了全部实质信息，没有擅自归纳？
    • [ ] 听不清的部分是否标注【存疑】而非脑补？
    • [ ] 发言人标注是否准确（不确定的标"发言人不明"）？

    节点 2：议题识别专家¶

    2.1 节点定位¶

    从会议记录中识别出讨论了哪些议题，为后续按议题组织纪要打下框架。会议常在多个议题间跳跃，先把议题理清，纪要才有结构。

    2.2 输入与输出¶

    输入：节点 1 清洗后的记录 + 会议议程（如有）

    输出：议题清单（议题名称 + 讨论范围 + 是否有结论）

    2.3 使用顺序¶

    1. 先用「快速生成版」识别议题。
    2. 议题交叉或会议无明确议程时，改用「专家增强版」。
    3. 用「自查审稿版」检查议题是否完整、是否有遗漏或合并不当。
    4. 有问题则用「返修优化版」修正。
    5. 对照 2.6 验收清单确认，通过后交给节点 3。

    2.4 提示词包¶

    A. 快速生成版¶
    你是一位会议议题梳理员。请从以下记录识别讨论的议题。

会议记录：【粘贴节点1的记录】
会议议程（如有）：【填入或留空】

输出：
1. 议题清单（按讨论顺序，每个议题一句话概括）
2. 每个议题是否产生了结论（是/否/部分）
    B. 专家增强版¶
    你是一位会议议题分析专家。

任务：从会议记录中准确识别所有讨论议题。

输入：
- 会议记录：【粘贴节点1的记录】
- 会议议程：【填入或留空】

处理步骤：
1. 识别记录中讨论的每个独立议题

4. 如有议程，对比实际讨论与议程的差异（议程外新增的、议程内未讨论的）

输出格式：

约束：议题划分基于实际讨论内容，不强行套议程；一个话题被反复讨论应合并为一个议题；不把闲聊当议题。

质量标准：议题清单完整覆盖会议实际讨论，颗粒度适中（不过碎也不过粗）。

常见失败情况：把同一议题的多次讨论拆成多个；把闲聊或寒暄当议题；遗漏了简短但重要的议题。

    节点 4：决策事项提取专家（招牌节点）¶

    4.1 节点定位¶

    这是本工作流的招牌节点。 准确区分会议中"真正决定了的事"和"只是讨论过的事"。会议纪要最常见、也最危险的错误，就是把讨论当成决定。本节点必须把事项分入以下类别：

    类别 判定标准
    已明确决定 有明确结论且无人反对（或决策人拍板）
    讨论未决 讨论过但没有形成结论
    需进一步确认 倾向某方案但还需确认才能定
    有责任人无截止时间 定了谁做，但没定什么时候
    有截止时间无明确责任人 定了时间，但没明确谁负责

    把"讨论过"误写成"已决定"，会导致没人真正执行却以为定了；反之会导致已定的事被当成还在讨论。

    4.2 输入与输出¶

    输入：节点 3 各议题观点 + 节点 1 清洗记录

    输出：分级的决策事项清单（事项 + 类别 + 依据 + 缺失信息）

    4.3 使用顺序¶

    1. 先用「快速生成版」初步分类决策事项。
    2. 会议决策模糊或表态含混时，必须使用「专家增强版」。
    3. 用「自查审稿版」检查是否把讨论误判为决定。
    4. 有问题则用「返修优化版」修正。
    5. 对照 4.6 验收清单确认，通过后交给节点 5。

    4.4 提示词包¶

    A. 快速生成版¶

1. 已明确决定（有明确结论且无人反对）
2. 讨论未决（讨论过但无结论）
3. 需进一步确认（倾向某方案但需确认）
4. 有责任人无截止时间
5. 有截止时间无明确责任人

    你是一位资深会议决策分析专家，深知"讨论"和"决定"的区别。

输入：

分类标准（逐项判定）：
1. 已明确决定：有明确结论，决策人拍板或全员无异议
2. 讨论未决：充分讨论但未形成结论
3. 需进一步确认：有倾向方案但需进一步确认/上级批准才能定
4. 有责任人无截止时间：明确了谁做，但未定时间
5. 有截止时间无明确责任人：定了时间节点，但未明确负责人

约束（关键）：模糊表态不得归为"已决定"；每个判定必须有记录中的原话支撑；信息不全必须标注，不补全。

质量标准：决策分类准确，纪要读者能清楚知道哪些真定了、哪些还没定、哪些缺信息。

常见失败情况：把"我觉得可以"这类倾向性发言当成决定；把热烈讨论当成已达成一致；漏标缺失的责任人或时间。

    7.1 节点定位¶

    识别会议中暴露的风险、未解决的争议和潜在隐患，单独列出以便跟进。这些往往是会议中一带而过、但事后最容易出问题的部分。

    7.2 输入与输出¶

    7.3 使用顺序¶

    4. 有问题则用「返修优化版」修正。
    5. 对照 7.6 验收清单确认，通过后交给节点 8。

1. 识别未解决的争议（各方意见不一致且未达成共识的）
2. 识别会议中明确提出的风险（有人指出的）

    节点 8：纪要格式生成专家¶

    8.1 节点定位¶

    将前序所有节点的产出整合为一份格式规范、结构清晰的会议纪要。纪要要让没参会的人也能看懂会议结果，让参会的人能据此追踪执行。

    8.2 输入与输出¶

    输入：节点 2 议题 + 节点 3 观点 + 节点 4 决策 + 节点 6 待办 + 节点 7 风险

    输出：完整会议纪要

    8.3 使用顺序¶

    1. 先用「快速生成版」生成纪要。
    2. 正式纪要或对外发送时，改用「专家增强版」。
    3. 用「自查审稿版」检查纪要是否区分了决策/讨论/待办、是否齐全。
    4. 有问题则用「返修优化版」修正。
    5. 对照 8.6 验收清单确认，通过后交给节点 9。

    常见错误¶

    错误 1：把讨论当成决定

    表现：会议只是讨论了某方案的可能性，纪要里却写成"会议决定采用 X 方案"，导致没人真正去推进却以为已经定了。

    修复：执行节点 4（招牌节点）的严格分类，"已明确决定"必须有原话支撑，模糊表态归入"讨论未决"或"需确认"。

    错误 2：待办没有责任人和时间却不标注

    表现：纪要列了一堆待办，但很多没写谁负责、什么时候完成，发出去后无人认领。

    修复：执行节点 6，缺失的责任人和时间如实标注"待确认"，并在"需会后确认事项"中汇总，而非留空或自行指派。

    错误 3：纪要夹带会议没说的内容

    表现：AI 整理时"脑补"了一些看似合理但会议没讨论的结论或建议。

    修复：每个节点都坚持"只整理实际发生的内容，不新增"，节点 8 自查时核对纪要内容是否都能在记录中找到来源。

    人工验收清单¶

    • [ ] 纪要是否清楚区分了"已决定"和"讨论未决"？
    • [ ] 每个"已决定"事项是否真的在会议中拍板了？
    • [ ] 待办的缺失责任人/时间是否标注"待确认"？
    • [ ] 纪要是否没有夹带会议未实际产生的内容？
    • [ ] 会后追踪清单是否覆盖所有需跟进事项？

    延伸玩法¶

    • 变体 1：快速纪要版：对短会或同步会，只走节点 1（整理）→ 节点 4（决策提取）→ 节点 6（待办校准）→ 节点 8（生成），省略观点和风险节点。
    • 变体 2：决策追溯版：对重大决策会，强化节点 4 和节点 3，详细记录每个决策的讨论过程和各方观点，便于日后追溯决策依据。
    • 进阶组合：与"工作流 17（领导汇报工作流）"结合，将会议纪要的决策和成果作为汇报材料的输入；多次会议的追踪清单可汇总成项目进展看板。
