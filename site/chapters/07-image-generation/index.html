    工作流 07：从创意需求到稳定图像提示词的 AI 多专家工作流¶

    适用岗位：设计师、自媒体创作者、品牌运营、短视频创作者、活动宣传人员

    真实场景：你需要为品牌活动生成一批视觉一致的宣传图，或者为短视频制作系列封面图，但每次生成的图像风格不统一、人物换脸、背景乱飘——你需要一套可以稳定复用的图像提示词体系。

    最终目标：一套可稳定复用的图像提示词体系，包含：视觉风格定义、主体描述词库、一致性控制词组、负面提示词、出图审查标准，以及适用于不同场景的提示词变体。

    输入材料清单¶

    • 创意需求描述（一句话或一段话，说明要生成什么）
    • 使用场景（宣传海报 / 系列封面 / 故事插图 / 产品展示）
    • 目标工具（Midjourney v6 / Stable Diffusion / 即梦 / DALL-E）
    • 参考风格（可选，1-3 张参考图的文字描述，或风格关键词）
    • 画幅要求（16:9 / 9:16 / 1:1 / 4:3）

    工作流总览¶

    节点 1：需求解析专家
  ↓ 输出：创意拆解清单（主体 + 场景 + 情绪 + 用途）
节点 2：视觉风格定位专家
  ↓ 输出：风格关键词组 + 参考画风说明
节点 3：主体元素设计专家
  ↓ 输出：主体描述词组（人物 / 物体 / 场景的具体描述）
节点 4：构图与镜头语言专家
  ↓ 输出：构图方案 + 景别 + 视角 + 焦距描述
节点 5：光影色彩专家
  ↓ 输出：光线方案 + 配色策略 + 情绪色调词组
节点 6：图像提示词生成专家
  ↓ 输出：完整可用英文提示词（结构化）
节点 7：一致性控制专家（招牌节点）
  ↓ 输出：一致性词库 + 系列图复用方案
节点 8：负面提示词专家
  ↓ 输出：针对性负面提示词组
节点 9：出图审查与返修专家
  ↓ 输出：审查报告 + 修正后提示词
最终交付：可稳定复用的图像提示词体系

    专家节点详解¶

    每个节点包含六部分：节点定位、输入与输出、使用顺序、提示词包（A 快速 / B 专家 / C 自查 / D 返修）、交付给下游节点、人工验收清单。

    C 自查审稿版和 D 返修优化版是当前节点内部的工作模式，不是新的专家角色。

    节点 1：需求解析专家¶

    1.1 节点定位¶

    将模糊的创意需求拆解为图像生成所需的结构化要素，防止在生成时因描述笼统导致 AI 理解偏差。"一张好看的海报"和"深色背景、单人、科技感、竖版海报"之间的差距，就是这个节点解决的问题。

    1.2 输入与输出¶

    输入：原始创意需求描述 + 使用场景说明

    输出：创意拆解清单（主体对象 + 场景环境 + 情绪基调 + 用途约束 + 禁止元素）

    1.3 使用顺序¶

    1. 先用「快速生成版」得到拆解清单初稿。
    2. 需求较复杂或多人物场景时，改用「专家增强版」。
    3. 用「自查审稿版」检查拆解是否完整、是否有遗漏约束。
    4. 有问题则用「返修优化版」补充。
    5. 对照 1.6 验收清单确认，通过后交给节点 2。

    1.4 提示词包¶

    A. 快速生成版¶
    你是一位图像需求分析师。请将以下创意需求拆解为图像生成所需的结构化要素。

需求描述：【粘贴创意需求】
使用场景：【填入：宣传海报 / 系列封面 / 插图 / 产品展示】

输出：
1. 主体对象（画面中心是什么）
2. 场景环境（背景、地点、时间）
3. 情绪基调（暖/冷/中性，情绪关键词）
4. 用途约束（竖版/横版，有无文字区域，有无品牌色要求）
5. 禁止元素（哪些不能出现，例如：不能有文字水印、不能有多余人物）
    B. 专家增强版¶
    你是一位资深视觉创意总监，专注于将模糊创意需求转化为精准的图像生成指令。

任务：对以下创意需求进行系统性拆解，输出可直接指导后续节点的结构化需求清单。

输入：
- 创意需求：【粘贴需求描述】
- 使用场景：【填入场景】
- 目标工具：【填入 AI 绘图工具名称】

拆解维度：
1. 主体层（Who/What）：画面中心对象的身份、动作、数量
2. 场景层（Where/When）：地点类型、时间段、天气/氛围
3. 情绪层（How it feels）：情绪关键词 + 视觉情绪对应（暖色/冷色/高对比/低饱和）
4. 功能层（Usage constraints）：版式要求、留白需求、品牌色限制、禁止内容
5. 差异化层（What makes it unique）：这张图与同类图的区别是什么

约束：禁止输出无法视觉化的需求（例如"有内涵"不算需求）；所有元素必须是可以在画面中呈现的视觉内容。

输出：5个维度的结构化需求清单 + 优先级标注（核心要素 vs 加分要素）
    C. 自查审稿版¶

    此为当前节点的自查模式，不是新的专家角色。

    请检查以下需求拆解清单是否存在以下问题：
1. 是否有无法视觉化的描述（例如"有品味""高端感"，未说明具体视觉特征）？
2. 主体对象描述是否精确到数量和状态（"人"不够，需要"一位站立的青年女性"）？
3. 用途约束是否完整（版式比例、留白位置是否说明）？
4. 是否遗漏了禁止元素（例如：需要配文字但忘了说明留白区域位置）？

【粘贴需求拆解清单】

逐条说明问题。
    D. 返修优化版¶
    根据自查意见修改需求拆解清单。将抽象描述替换为可视化元素，补充缺失的数量/状态信息，完善用途约束和禁止元素。

原始清单：【粘贴原始清单】
自查意见：【粘贴自查结果】

    1.5 交付给下游节点¶

    将结构化需求清单完整复制，交给节点 2（视觉风格定位专家）。节点 2 将基于情绪基调和功能约束，确定图像的视觉风格方向。

    1.6 人工验收清单¶

    • [ ] 主体对象描述是否精确到数量、状态和关键特征？
    • [ ] 是否有明确的版式要求（横版/竖版/比例）？
    • [ ] 是否列出了至少 1 条禁止元素？

    节点 2：视觉风格定位专家¶

    2.1 节点定位¶

    基于需求拆解结果，确定图像的视觉风格方向，输出可直接用于提示词的风格关键词组。同样是"科技感"，赛博朋克风和极简科技白风的提示词完全不同——这个节点负责明确化。

    2.2 输入与输出¶

    输入：节点 1 的结构化需求清单（情绪层 + 功能层）

    输出：风格类型标签（2-3 个）+ 主风格关键词组（可直接用于提示词）+ 禁用风格说明

    2.3 使用顺序¶

    1. 先用「快速生成版」得到风格关键词初稿。
    2. 需要在多个风格方向中选择时，改用「专家增强版」做对比分析。
    3. 用「自查审稿版」检查风格词是否与需求匹配、是否是目标工具认识的词汇。
    4. 有问题则用「返修优化版」修正。
    5. 对照 2.6 验收清单确认，通过后交给节点 3。

    2.4 提示词包¶

    A. 快速生成版¶
    你是一位视觉风格顾问。请根据以下需求清单，确定图像的视觉风格方向。

需求清单：【粘贴节点1的输出】
目标工具：【填入工具名称】

输出：
1. 风格类型标签（2-3个，例如：电影级写实 / 日系扁平插画 / 深色科幻概念图）
2. 主风格英文关键词组（可直接加入提示词，10-15个词）
3. 禁用风格说明（哪些风格与需求不符，需要排除）
    B. 专家增强版¶
    你是一位资深视觉风格总监，熟悉 Midjourney/SD 的风格关键词效果。

任务：基于以下需求清单，提供 3 个可选风格方向，并推荐最适合当前用途的方向。

输入：
- 需求清单：【粘贴节点1的完整输出】
- 目标工具：【填入工具名称】

每个风格方向输出：
- 风格名称（具体到亚类型，例如：Wes Anderson 电影色调，而不只是"电影风格"）
- 英文关键词组（10-15个，可直接用于提示词）
- 适用场景说明（为什么这个风格适合当前需求）
- 潜在问题（使用这个风格时需要注意什么）

推荐理由：最终推荐哪个方向，说明与需求的匹配度。

约束：关键词必须是目标工具实际支持的词汇，不能自造词；中文需求必须翻译为英文关键词。

    这是本工作流的招牌节点。 解决 AI 图像生成中最常见的"系列图不一致"问题：同一角色换脸、同一场景背景变化、同一服装细节改变、不同图片风格漂移。一致性控制不是在每张图上加同样的风格词那么简单，而是要建立一套完整的固定词体系，涵盖人物一致性、服装一致性、场景一致性、画风一致性和系列图统一感五个维度。

    7.2 输入与输出¶

    输入：节点 6 的完整提示词 + 节点 3 的主体描述（人物/物体关键特征）

    输出：五维度一致性词库 + 系列图复用提示词模板 + 一致性使用说明

    7.3 使用顺序¶

    1. 先用「快速生成版」得到基础一致性词库。
    2. 有人物或多场景需要精确一致时，必须使用「专家增强版」。
    3. 用「自查审稿版」检查各维度词库是否完整、是否存在自相矛盾。
    4. 有问题则用「返修优化版」完善。
    5. 对照 7.6 验收清单确认，通过后同时交给节点 8 和节点 9。

    常见错误¶

    错误 1：主体描述放在风格词之后

    表现：提示词以 cinematic style, 8k resolution 开头，主体描述放在第 5 个词之后，导致 AI 优先处理风格而对主体理解偏差。

    修复：主体核心词（身份 + 关键外貌）必须是提示词最前面的 3 个词，风格词放在中段。

    错误 2：一致性词库建立后不完整复制

    表现：系列图中某几张忘记粘贴快速调用前缀，或只粘贴了一部分，导致人物外貌在那几张图中发生变化。

    修复：建立快速调用前缀后，每次生成新图时必须完整复制前缀并粘贴到提示词最前面，一个词都不能省略。

    错误 3：负面提示词与正面提示词冲突

    表现：想生成长发角色，但负面词里出现了 long hair（可能是复制粘贴时的失误），导致 AI 生成混乱的发型结果。

    修复：执行节点 8 的自查审稿版，逐字比对正面词和负面词，确认没有相同词汇同时出现在两边。
