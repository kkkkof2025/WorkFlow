    工作流 05：从故事文本到可生成角色设定图的 AI 多专家工作流¶

    适用岗位：插画师、漫画创作者、短剧美术、游戏设计师、AI 绘图创作者

    真实场景：你已经写好了一段故事文本或短剧剧本，需要为主要角色生成可以直接用于 Midjourney/Stable Diffusion 的角色设定图，要求正面/侧面/背面三视图风格统一、不同角色之间有明显视觉区分度。

    最终目标：每个主角的完整角色设定文档，包含：角色内核分析、外貌特征描述、服装道具设计、三视图图像提示词（英文）、风格一致性描述词库、角色差异化方案。

    输入材料清单¶

    开始前请准备以下材料：

    • 故事文本或剧本片段（含角色出场的描述段落，500 字以上）
    • 需要设计的角色列表（姓名 + 在故事中的基本身份）
    • 目标视觉风格（写实 / 动漫 / 古风 / 赛博朋克 / 其他）
    • AI 绘图工具（Midjourney v6 / Stable Diffusion / 即梦等）
    • 参考风格图（可选，1-3 张风格参考图的文字描述）

    工作流总览¶

    节点 1：角色内核提取专家
  ↓ 输出：每个角色的内核关键词（性格+动机+身份定位）
节点 2：年龄身份校准专家
  ↓ 输出：年龄/时代/身份对外貌的影响清单
节点 3：外貌特征设计专家
  ↓ 输出：详细外貌描述（面部+体型+肤色+发型）
节点 4：服装道具设计专家
  ↓ 输出：服装设计描述 + 标志性道具
节点 5：三视图提示词专家
  ↓ 输出：正面/侧面/背面三视图英文提示词
节点 6：风格一致性专家
  ↓ 输出：角色一致性描述词库 + 跨场景复用方案
节点 7：角色差异化审查专家（跨角色全局审查）
  ↓ 输出：差异化审查报告 + 修改建议
节点 8：图像生成提示词修正专家
  ↓ 输出：修正后的完整提示词组
最终交付：每个角色的完整设定文档包

    专家节点详解¶

    每个节点包含六部分：节点定位、输入与输出、使用顺序、提示词包（A 快速 / B 专家 / C 自查 / D 返修）、交付给下游节点、人工验收清单。

    关于节点 7（角色差异化审查专家）：这是一个跨角色的全局审查节点，负责检查多个角色之间是否存在外貌雷同问题，性质与其他节点的自查审稿版不同——它是比较多个角色的独立审查节点。

    其他节点的 C 自查审稿版和 D 返修优化版是当前节点内部的工作模式，不是新的专家角色。

    节点 1：角色内核提取专家¶

    1.1 节点定位¶

    从故事文本中提取每个角色的性格特征、行为动机和身份定位，为外貌设计提供内在依据。外貌设计必须从角色内核出发——一个冷酷的反派和一个温柔的女主，外貌气质应当有本质差异，而不是"换个发色"。

    1.2 输入与输出¶

    输入：故事文本或剧本片段 + 角色列表

    输出：每个角色的内核关键词组（3-5个性格关键词 + 核心动机 + 身份标签 + 气质方向）

    1.3 使用顺序¶

    1. 先用「快速生成版」得到每个角色的内核关键词。
    2. 故事文本较长或角色关系复杂时，改用「专家增强版」。
    3. 用「自查审稿版」检查关键词是否精确反映故事中的角色特质。
    4. 有问题则用「返修优化版」修正。
    5. 对照 1.6 验收清单确认，通过后交给节点 2。

    1.4 提示词包¶

    A. 快速生成版¶
    你是一位角色设计师。请从以下故事文本中提取每个角色的内核关键词。

故事文本：【粘贴故事文本】
角色列表：【粘贴角色姓名和基本身份】

对每个角色输出：
- 性格关键词（3-5个，例如：冷静、腹黑、外冷内热）
- 核心动机（他/她最驱动自己行动的欲望或恐惧）
- 气质方向（例如：权贵压迫感 / 邻家少女清新感 / 反社会神秘感）
    B. 专家增强版¶
    你是一位资深角色设计总监，擅长将文字角色转化为视觉语言。

任务：从以下故事文本中精准提取每个角色的内核特质，为后续外貌设计提供可视化方向。

输入：
- 故事文本：【粘贴故事文本】
- 角色列表：【粘贴角色姓名和基本身份】

提取维度：
1. 性格关键词（3-5个，直接影响面部表情习惯和体态）
2. 核心动机（驱动行为的根本欲望，影响角色常态表情）
3. 身份标签（社会阶层/职业/时代背景，影响服装气质）
4. 气质方向（用一个视觉比喻描述，例如"寒冬腊月的第一场雪"）
5. 视觉禁忌（该角色绝对不适合的外貌元素，例如：不能有可爱系卷发）

约束：关键词必须能直接转化为视觉描述，不能使用"有内涵""很特别"等无法可视化的词汇。

输出：每个角色的内核提取表格（含5个维度）
    C. 自查审稿版¶

    此为当前节点的自查模式，不是新的专家角色。

    请检查以下角色内核提取是否存在以下问题：
1. 性格关键词是否能转化为视觉元素（不能包含无法画出来的词汇）？
2. 气质方向描述是否足够具体，能给设计师明确的视觉方向？
3. 不同角色的内核关键词是否有明显区分，而非都是"温柔善良"之类的通用词？
4. 视觉禁忌是否有效排除了与角色气质不符的外貌方向？

【粘贴角色内核提取内容】

逐条说明问题。
    D. 返修优化版¶
    根据自查意见修改角色内核提取。将无法可视化的词汇替换为具体可描绘的特质，区分不同角色的关键词，补充视觉禁忌。

原始提取：【粘贴原始内核提取】
自查意见：【粘贴自查结果】

    1.5 交付给下游节点¶

    将每个角色的内核关键词组完整复制，交给节点 2（年龄身份校准专家）。节点 2 将基于内核关键词和身份背景，判断年龄和时代对外貌的具体影响。

    1.6 人工验收清单¶

    • [ ] 每个角色的性格关键词是否都能转化为视觉元素？
    • [ ] 不同角色的气质方向是否有明显区分？
    • [ ] 是否为每个角色标注了视觉禁忌（不适合的外貌元素）？

    节点 2：年龄身份校准专家¶

    2.1 节点定位¶

    确认角色的年龄、时代背景和社会身份对外貌设计的具体影响，为外貌特征设计设置精确的约束条件。同样是"20岁女性"，古代贵族和现代程序员的外貌气质完全不同——这个节点负责将这些差异显式化。

    2.2 输入与输出¶

    输入：节点 1 的角色内核关键词（含身份标签）

    输出：每个角色的校准参数（年龄范围 + 时代背景 + 身份对外貌的5项影响说明）

    2.3 使用顺序¶

    1. 先用「快速生成版」得到校准参数初稿。
    2. 历史背景复杂或需要精确服饰考据时，改用「专家增强版」。
    3. 用「自查审稿版」检查校准参数是否与故事设定一致。
    4. 有问题则用「返修优化版」修正。
    5. 对照 2.6 验收清单确认，通过后交给节点 3。

    2.4 提示词包¶

    A. 快速生成版¶
    你是一位角色设定师。请根据以下角色内核，判断年龄、时代背景对外貌设计的影响。

角色内核：【粘贴节点1的内核关键词】

对每个角色输出：
- 推荐年龄范围（精确到5年区间，例如：22-27岁）
- 时代背景说明（例如：现代都市 / 民国上海 / 架空古代）
- 身份对外貌的3项影响（例如：贵族身份→皮肤白皙、手部保养好、无劳动痕迹）
    B. 专家增强版¶
    你是一位具有服装史和造型设计背景的角色设定专家。

任务：基于角色内核，为每个角色提供精确的年龄与身份校准参数，确保外貌设计符合时代和身份逻辑。

输入：
- 角色内核关键词：【粘贴节点1的完整输出】

校准维度：
1. 精确年龄（精确到年龄区间，并说明判断依据）
2. 时代背景（说明对外貌的具体影响，不只是标注"古代"）
3. 社会阶层对外貌的影响（肤色、手部、体态、面部状态）
4. 职业对身体特征的影响（武将有肌肉、文人有书卷气等）

    节点 7：角色差异化审查专家¶

    7.1 节点定位¶

    这是一个跨角色的全局审查节点，通过比较所有角色的一致性词库，识别外貌高度雷同的角色对，并提出差异化修改方案。如果两个主角的固定词几乎相同，AI 生成时会不断产生"两个人长得一模一样"的问题，必须在提示词层面强制拉开差距。

    7.2 输入与输出¶

    输入：所有角色的一致性词库（来自节点 6）

    输出：差异化审查报告（雷同问题清单 + 每对雷同角色的修改建议）

    7.3 使用顺序¶

    1. 先用「快速生成版」快速扫描外貌雷同问题。
    2. 多角色且故事设定上本来就外貌相似时，改用「专家增强版」找到区分点。
    3. 用「自查审稿版」检查审查报告是否遗漏了潜在雷同。
    4. 根据审查结论返回节点 3-6 修改对应角色的外貌设计。
    5. 对照 7.6 验收清单确认，通过后交给节点 8。

    常见错误¶

    错误 1：跳过内核提取直接设计外貌

    表现：生成的角色外貌看起来好看，但与故事中的人物气质完全不符，导致读者/观众认为"这不是我想象中的那个角色"。

    修复：必须先完成节点 1，从故事文本中提取角色内核，再以内核为约束设计外貌。

    错误 2：多角色外貌设计雷同

    表现：主角和女配角都是"黑色长发、白皙皮肤、纤细身材"，AI 生成时不断把两个人画成同一个人。

    修复：必须执行节点 7 的差异化审查，为每对相似角色在至少 2 个维度上强制拉开差距。

    错误 3：三视图提示词不一致

    表现：正面图生成出来是蓝色眼睛，侧面图变成了棕色眼睛，背面图又改了发型。

    修复：执行节点 5 的自查审稿版，确认三张图的固定外貌词完全相同，然后执行节点 6 建立词库确保后续生成的一致性。

    人工验收清单¶

    • [ ] 每个角色的外貌设定是否都来自故事文本的角色内核，而非随意设计？
    • [ ] 三视图提示词中，三张图的固定外貌词是否完全相同？
    • [ ] 不同角色之间是否有明显的视觉区分度（不能让人混淆）？
    • [ ] 服装设计是否符合角色的时代背景，没有时代错误？
    • [ ] 提示词是否可以直接复制粘贴到目标 AI 工具中生成图像？

    延伸玩法¶

    • 变体 1：多服装版：用节点 4 的换装建议，为同一角色生成不同场景服装版本的提示词（战斗服/正装/便服）。
    • 变体 2：表情包版：在三视图基础上，用节点 8 的场景模板生成同一角色的多种表情变体（生气/开心/悲伤/惊讶）。
    • 进阶组合：与"工作流 04（短剧策划工作流）"结合，用本工作流的角色档案直接对接短剧人物关系节点的输出，形成"策划→角色设定→开始绘制"的完整链路；与"工作流 06（视频分镜工作流）"结合，将节点 6 的一致性词库直接用于分镜中的角色连续性控制。
