    工作流 04：从关键词到完整短剧提案的 AI 多专家工作流¶

    适用岗位：短剧编剧、内容策划、IP 开发者、自媒体团队

    真实场景：你拿到一个关键词或热门话题（例如"霸总逆袭""女性复仇""古代穿越"），需要在三天内产出一份完整的短剧提案，包含类型定位、故事梗概、主角设定、分集结构和平台推荐策略，可以直接投给平台或制作团队。

    最终目标：一份完整的短剧提案书，包含：题材定位报告、爆点要素清单、故事梗概（含主线支线）、人物关系图文版、分集大纲（15-20集）、每集情绪钩子设计、平台适配策略。

    输入材料清单¶

    开始前请准备以下材料：

    • 核心关键词或话题方向（例如："女主穿越成炮灰""男主失忆后爱上仇人"）
    • 目标平台（抖音短剧 / 快手小剧场 / 腾讯微短剧 / 其他）
    • 集数目标（15集 / 20集 / 30集）
    • 受众偏好（女频 / 男频 / 通吃，年龄层偏好）
    • 参考竞品（可选，1-2部同类型短剧名称）

    工作流总览¶

    节点 1：题材定位专家
  ↓ 输出：题材类型 + 受众画像 + 市场空间评估
节点 2：爆点判断专家
  ↓ 输出：爆点要素清单 + 情节雷区清单
节点 3：故事梗概专家
  ↓ 输出：故事梗概（主线 + 支线，600-1000字）
节点 4：人物关系专家
  ↓ 输出：主要角色档案 + 人物关系网络
节点 5：分集结构专家
  ↓ 输出：分集大纲（每集一句话 + 结尾悬念）
节点 6：情绪钩子专家
  ↓ 输出：每集开头钩子 + 结尾悬念设计
节点 7：平台适配专家
  ↓ 输出：平台适配建议 + 发布策略
节点 8：提案审稿专家
  ↓ 输出：提案审查报告 + 修改建议
最终交付：完整短剧提案书

    专家节点详解¶

    每个节点包含六部分：节点定位、输入与输出、使用顺序、提示词包（A 快速 / B 专家 / C 自查 / D 返修）、交付给下游节点、人工验收清单。

    C 自查审稿版和 D 返修优化版是当前节点内部的工作模式，不是新的专家角色。

    节点 1：题材定位专家¶

    1.1 节点定位¶

    从关键词出发，判断短剧的类型属性、目标受众和市场竞争空间。题材定位是整个提案的地基——定位不清，后续梗概和分集结构都会出现方向偏差。

    1.2 输入与输出¶

    输入：核心关键词 + 目标平台 + 受众偏好

    输出：题材类型标签（2-3个）+ 核心受众画像 + 市场空间评估（竞争度、机会点）+ 内容方向建议

    1.3 使用顺序¶

    1. 先用「快速生成版」得到题材定位初稿。
    2. 需要更深入的市场分析时，改用「专家增强版」。
    3. 用「自查审稿版」检查定位是否清晰、受众是否具体。
    4. 有问题则用「返修优化版」修正。
    5. 对照 1.6 验收清单确认，通过后交给节点 2。

    1.4 提示词包¶

    A. 快速生成版¶
    你是一位短剧内容策划师。请根据以下关键词，判断短剧题材定位和目标受众。

关键词：【填入关键词】
目标平台：【填入平台】

输出：
1. 题材类型标签（2-3个，例如：霸总爱情 / 女频逆袭 / 古装玄幻）
2. 核心受众画像（年龄、性别、核心情绪诉求）
3. 市场竞争评估（高/中/低竞争，说明理由）
4. 内容方向建议（如何做出差异化，一句话）
    B. 专家增强版¶
    你是一位资深短剧项目开发总监，熟悉各平台算法偏好和用户消费心理。

任务：对以下关键词进行系统性题材定位分析，输出可执行的内容方向。

输入：
- 关键词：【填入关键词】
- 目标平台：【填入平台】
- 受众偏好：【填入女频/男频/通吃及年龄层】
- 参考竞品（可选）：【填入同类短剧名称】

分析步骤：
1. 判断该关键词对应的主流叙事类型（权谋 / 甜宠 / 逆袭 / 悬疑等）
2. 识别目标受众的核心情绪诉求（爽感 / 共鸣 / 刺激 / 代入感）
3. 评估市场饱和度（同类内容数量、近期是否有爆款）
4. 找出差异化切入点（哪个角度竞争最少、潜力最大）

约束：不写"潜力巨大""前景广阔"等空洞表达；差异化建议必须具体到叙事策略层面。

输出格式：题材定位报告（含类型标签、受众画像、市场评估、差异化方向，共4个模块）
    C. 自查审稿版¶

    此为当前节点的自查模式，不是新的专家角色。

    请检查以下题材定位是否存在以下问题：
1. 题材类型标签是否过于宽泛，无法指导后续创作（例如只写"爱情类"）？
2. 受众画像是否具体到情绪诉求层面（不能只写"18-35岁女性"）？
3. 差异化建议是否是具体的叙事策略，而非"要有创意"这类废话？
4. 市场评估是否说明了理由，而不只是高/中/低的结论？

【粘贴题材定位输出内容】

逐条说明问题。
    D. 返修优化版¶
    根据自查意见修改题材定位报告。保留已确定的类型方向，针对每条问题做具体调整。

原始输出：【粘贴原始题材定位】
自查意见：【粘贴自查结果】

重点修正：类型标签精确化、受众画像具体化、差异化建议可操作化。

    1.5 交付给下游节点¶

    将题材定位报告完整复制，交给节点 2（爆点判断专家）。节点 2 将基于定位方向判断哪些情节要素能引发传播。

    1.6 人工验收清单¶

    • [ ] 题材类型标签是否精确到亚类型（不是宽泛的"爱情"，而是"霸总甜宠"或"虐恋反转"）？
    • [ ] 受众画像是否包含了核心情绪诉求（不只是年龄性别）？
    • [ ] 差异化方向是否是具体可执行的叙事策略？

    节点 2：爆点判断专家¶

    2.1 节点定位¶

    基于题材定位，分析哪些情节要素、人设冲突和叙事结构能引发用户传播和留存。爆点判断决定了故事梗概的骨架——哪些情节必须有，哪些是雷区。

    2.2 输入与输出¶

    输入：节点 1 的题材定位报告

    输出：爆点要素清单（5-8条必须包含的情节要素）+ 情节雷区清单（3-5条要避免的常见失误）

    2.3 使用顺序¶

    1. 先用「快速生成版」得到爆点要素初稿。
    2. 需要更深入分析传播机制时，改用「专家增强版」。
    3. 用「自查审稿版」检查爆点是否与受众情绪对应。
    4. 有问题则用「返修优化版」修正。
    5. 对照 2.6 验收清单确认，通过后交给节点 3。

    2.4 提示词包¶

    A. 快速生成版¶
    你是一位短剧内容分析师。请根据以下题材定位，列出这类短剧的核心爆点要素和情节雷区。

题材定位：【粘贴节点1的题材定位报告】

输出：
1. 爆点要素清单（5-8条，说明每条为何能引发传播或留存）
2. 情节雷区清单（3-5条常见失误，说明为什么观众会流失）
    B. 专家增强版¶
    你是一位短剧爆款内容研究员，专注研究短视频平台的内容传播规律。

任务：基于以下题材定位，系统分析该类型短剧的爆点结构，输出可直接用于指导剧本创作的要素清单。

输入：
- 题材定位：【粘贴节点1的完整报告】

分析维度：
1. 人设爆点（什么样的人设组合最容易产生冲突和情绪共鸣）
2. 情节爆点（哪类情节节点最容易引发弹幕、转发、追更）
3. 节奏爆点（信息密度、反转频率、情绪高峰的时机设计）
4. 情节雷区（该类型短剧最常见的导致弃剧的剧情问题）

约束：每条爆点要素必须说明"为什么有效"（指向受众的哪种情绪）；不能只列通用爆点，必须针对该题材类型。

输出：爆点要素清单（带情绪机制说明）+ 情节雷区清单（带弃剧原因说明）
    C. 自查审稿版¶

    此为当前节点的自查模式，不是新的专家角色。

    请检查以下爆点要素清单是否存在以下问题：

    • [ ] 第 1 集的开头钩子是否能在 5 秒内制造冲突或悬念？

    节点 7：平台适配专家¶

    7.1 节点定位¶

    针对目标平台（抖音/快手/微短剧平台）的算法规则、用户行为和投放逻辑，对提案进行平台专项调整，输出发布策略建议。不同平台对集长、标题、封面、更新节奏要求不同，不做适配直接投稿容易错过流量窗口。

    7.2 输入与输出¶

    输入：节点 1 题材定位（含目标平台）+ 节点 5 分集大纲 + 节点 6 钩子方案

    输出：平台适配建议报告（含集长建议、标题策略、封面建议、更新节奏、首集设计要点）

    7.3 使用顺序¶

    1. 先用「快速生成版」得到平台建议初稿。
    2. 需要针对多平台做差异化策略时，改用「专家增强版」。
    3. 用「自查审稿版」检查建议是否针对具体平台特性。
    4. 有问题则用「返修优化版」修正。
    5. 对照 7.6 验收清单确认，通过后交给节点 8。

    节点 8：提案审稿专家¶

    8.1 节点定位¶

    这是一个跨内容的全局审查节点，负责对完整提案进行系统性审查，检查各模块之间的一致性、爆点覆盖完整性和市场竞争力。提案送出前的最后一道质量关卡。

    8.2 输入与输出¶

    输入：所有前序节点的完整输出（题材定位 + 梗概 + 角色 + 分集 + 钩子 + 平台策略）

    输出：提案审查报告（问题清单 + 修改建议 + 通过/需修改结论）

    8.3 使用顺序¶

    1. 先用「快速生成版」做快速整体审查。
    2. 需要竞争力评估时，改用「专家增强版」。
    3. 用「自查审稿版」检查审查报告本身的完整性（可选）。
    4. 结论为"通过"后，整合为完整提案书；结论为"需修改"时，返回对应节点修改后重新审查。
    5. 对照 8.6 验收清单确认。

    8.4 提示词包¶

    A. 快速生成版¶

2. 主要角色的动机是否能解释分集大纲中的核心情节推进？
3. 是否所有爆点要素都在分集大纲中有具体呈现？
4. 分集节奏是否合理（无拖沓段落）？

    B. 专家增强版¶
    你是一位短剧项目评审专家，同时具备编剧视角和平台采购视角。

任务：对以下完整提案进行系统审查，从内容质量和市场竞争力两个维度评分，输出可执行的修改建议。

输入：

1. 内容完整性（所有必要模块是否齐全，无缺失）

    常见错误¶

    错误 1：题材定位没完成就写梗概

    表现：梗概写完后发现故事定位不清晰，受众在哪里、爽点是什么都不明确，导致分集时不知道该强化哪些情节。

    修复：必须先完成节点 1 和节点 2，在有清晰题材定位和爆点要素的情况下再开始写梗概。

    错误 2：分集大纲连续注水

    表现：中间有 4-6 集只有对话和感情推进，没有情节进展，导致观众流失。

    修复：执行节点 5 的自查审稿版，检查是否有连续 3 集无明显情节进展，有则立即压缩或合并。

    错误 3：提案模块各自为战

    表现：题材定位说是"女性复仇爽剧"，故事梗概却写成了"虐恋疗愈"，角色动机又与两者都不匹配。

    修复：执行节点 8 的内部一致性审查，确保所有模块围绕同一个题材核心展开。

    • [ ] 题材定位、梗概、角色、分集四个模块是否内部一致（不互相矛盾）？
    • [ ] 分集大纲中是否覆盖了节点 2 输出的所有爆点要素？
    • [ ] 是否没有连续 3 集以上缺乏情节进展的拖沓段落？
    • [ ] 每集的结尾悬念是否有变化，而非重复同一模式？
    • [ ] 提案审稿专家的审查结论是否为"通过"？

    延伸玩法¶

    • 变体 1：IP 改编版：将节点 3 的输入替换为已有小说的章节摘要，快速生成短剧改编提案。
    • 变体 2：多平台版本：完成一份核心提案后，用节点 7 分别生成抖音版、快手版和腾讯版三份不同适配策略。
    • 进阶组合：与"工作流 05（角色设定工作流）"结合，用本工作流的角色档案直接驱动角色设定图的生成；与"工作流 01（短视频创作工作流）"结合，将分集大纲中的高潮集快速剪辑为宣传短视频。
