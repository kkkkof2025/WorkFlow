# 工作流 19：从群聊记录到可复用知识库的 AI 多专家工作流

**适用岗位**：社群运营、培训组织者、项目协作人员、知识管理人员、团队负责人

**真实场景**：你管理的工作群或学习群积累了大量聊天记录，里面藏着有价值的问答、共识和资料，但翻起来费劲、新人重复问同样的问题。你需要把群聊沉淀成一个可复用的知识库——而不是把聊天记录原样存档。

**最终目标**：一个可复用的知识库，包含：清洗后的有效内容、主题分类、高频问题、有效回答、明确区分的共识结论、待办事项、资料链接清单、FAQ，以及知识库更新方案。**不把聊天里的所有观点都当成结论。**

---

## 输入材料清单

- 群聊记录（导出的聊天记录，含时间和发言人）
- 群的性质（工作群 / 学习群 / 项目群 / 社群）
- 沉淀目的（建 FAQ / 整理资料 / 提取待办 / 全面沉淀）
- 已有知识库（如有，用于增量更新）
- 关键人物（群里的权威/负责人，其表态权重不同）

---

## 工作流总览

```
节点 1：群聊清洗专家
  ↓ 输出：清洗后的有效记录
节点 2：主题聚类专家
  ↓ 输出：主题分类清单
节点 3：高频问题识别专家
  ↓ 输出：高频问题清单
节点 4：有效回答提取专家
  ↓ 输出：问题对应的有效回答
节点 5：共识结论整理专家（招牌节点）
  ↓ 输出：分级的内容（共识/观点/争议/待确认）
节点 6：待办事项提取专家
  ↓ 输出：待办清单
节点 7：资料链接整理专家
  ↓ 输出：资料链接清单
节点 8：FAQ 生成专家
  ↓ 输出：FAQ 文档
节点 9：知识库更新专家
  ↓ 输出：知识库更新方案
最终交付：可复用知识库
```

---

## 专家节点详解

每个节点包含六部分：节点定位、输入与输出、使用顺序、提示词包（A 快速 / B 专家 / C 自查 / D 返修）、交付给下游节点、人工验收清单。

**C 自查审稿版和 D 返修优化版是当前节点内部的工作模式，不是新的专家角色。**

> 核心原则：必须区分**共识、个人观点、争议、待确认**。群里有人说了不等于是结论，热闹讨论不等于达成一致。

### 节点 1：群聊清洗专家

#### 1.1 节点定位

将原始群聊记录清洗，去除寒暄、表情、无关闲聊，保留有信息价值的内容。群聊噪音多，先清洗才能高效提取知识。

#### 1.2 输入与输出

**输入**：原始群聊记录 + 群性质

**输出**：清洗后的有效记录（保留发言人和时间，去除噪音）

#### 1.3 使用顺序

1. 先用「快速生成版」清洗记录。
2. 记录量大或闲聊与有效信息交织时，改用「专家增强版」。
3. 用「自查审稿版」检查是否误删有效信息、是否改变原意。
4. 有问题则用「返修优化版」修正。
5. 对照 1.6 验收清单确认，通过后交给节点 2 和节点 3。

#### 1.4 提示词包

##### A. 快速生成版

```
你是一位群聊记录清洗员。请清洗以下群聊记录。

群聊记录：【粘贴记录】
群性质：【填入】

要求：
1. 去除寒暄、表情包、纯灌水
2. 保留所有有信息价值的发言（问答、观点、资料、决定）
3. 保留发言人和时间
4. 不改变发言原意，不归纳
```

##### B. 专家增强版

```
你是一位群聊内容清洗专家。

任务：清洗群聊记录，保留有价值内容，去除噪音。

输入：
- 群聊记录：【粘贴记录】
- 群性质：【填入】
- 沉淀目的：【填入】

处理步骤：
1. 去除噪音（寒暄、表情、纯附和"对""赞"、与主题无关的闲聊）
2. 保留有价值内容（问题、回答、观点、资料链接、决定、争论）
3. 保留发言人和时间戳（用于后续判断权威性和时序）
4. 对引用/回复关系做标注（谁回复谁，便于还原问答对）
5. 标记看不懂或缺乏上下文的发言（不脑补）

输出格式：
- 清洗后记录（保留发言人、时间、回复关系）
- 已去除内容类型说明（去掉了哪些类型的噪音）

约束（关键）：不改变发言原意；不归纳不下结论（后续节点处理）；不确定是否有价值的内容先保留并标注；不脑补缺失上下文。

质量标准：清洗后保留全部有价值信息，问答和回复关系清晰，后续可准确提取。

常见失败情况：把含有效信息的"闲聊"误删；改变发言原意；丢失回复关系导致问答对不上。
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下清洗记录是否存在以下问题：
1. 是否误删了含有效信息的发言（即使看起来像闲聊）？
2. 是否改变了发言的原意？
3. 问答的回复关系是否保留（谁回复谁）？
4. 是否脑补了缺失的上下文？

【粘贴清洗记录】

逐条说明问题。
```

##### D. 返修优化版

```
根据自查意见修正清洗记录。补回误删的有效信息，还原被改变的原意，补充回复关系标注，移除脑补内容。

原始清洗记录：【粘贴原始记录】
自查意见：【粘贴自查结果】
```

#### 1.5 交付给下游节点

将清洗后的记录复制，交给节点 2（主题聚类专家）和节点 3（高频问题识别专家）。

#### 1.6 人工验收清单

- [ ] 是否没有误删含有效信息的发言？
- [ ] 是否没有改变发言原意？
- [ ] 问答的回复关系是否保留？

### 节点 2：主题聚类专家

#### 2.1 节点定位

将清洗后的群聊内容按主题聚类，让分散在不同时间的相关讨论归集到一起。群聊话题跳跃，按主题归类才能形成知识结构。

#### 2.2 输入与输出

**输入**：节点 1 清洗记录

**输出**：主题分类清单（主题 + 包含的讨论 + 讨论热度）

#### 2.3 使用顺序

1. 先用「快速生成版」聚类主题。
2. 主题交叉或记录跨度长时，改用「专家增强版」。
3. 用「自查审稿版」检查聚类是否合理、是否有遗漏。
4. 有问题则用「返修优化版」修正。
5. 对照 2.6 验收清单确认，通过后交给节点 5。

#### 2.4 提示词包

##### A. 快速生成版

```
你是一位内容聚类员。请将群聊内容按主题聚类。

清洗记录：【粘贴节点1】

输出：
| 主题 | 涉及的讨论摘要 | 讨论热度（高/中/低）|
按内容实际涉及的主题聚类，不强行归类。一条发言可归入多个主题。
```

##### B. 专家增强版

```
你是一位内容主题聚类专家。

任务：将群聊内容按主题归集，形成知识结构。

输入：
- 清洗记录：【粘贴节点1】
- 群性质：【填入】

处理步骤：
1. 识别群聊涉及的所有主题
2. 将相关发言（即使分散在不同时间）归集到对应主题
3. 标注每个主题的讨论热度（参与人数、发言量）
4. 标注主题的状态（已充分讨论 / 讨论中 / 一带而过）
5. 识别跨主题的关联

输出格式：
| 主题 | 讨论摘要 | 参与发言数 | 热度 | 讨论状态 |

约束：聚类基于实际内容，不强行归类；颗粒度适中（不过碎不过粗）；一条发言可属于多个主题。

质量标准：主题分类清晰，相关讨论得到归集，热度和状态如实反映。

常见失败情况：主题划分过细或过粗；把不相关的讨论硬归一类；遗漏一带而过但重要的主题。
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下主题聚类是否存在以下问题：
1. 主题颗粒度是否合适（不过碎不过粗）？
2. 是否把不相关的讨论硬归为一类？
3. 是否遗漏了一带而过但有价值的主题？
4. 讨论热度和状态标注是否准确？

【粘贴主题聚类】

逐条说明问题。
```

##### D. 返修优化版

```
根据自查意见修正主题聚类。调整颗粒度，拆分硬凑的类别，补回遗漏主题，校正热度状态。

原始聚类：【粘贴原始聚类】
自查意见：【粘贴自查结果】
```

#### 2.5 交付给下游节点

将主题分类清单复制，交给节点 5（共识结论整理专家）。

#### 2.6 人工验收清单

- [ ] 主题颗粒度是否合适？
- [ ] 是否没有把不相关讨论硬归一类？
- [ ] 是否覆盖了重要主题？

### 节点 3：高频问题识别专家

#### 3.1 节点定位

识别群里被反复问到的高频问题，这些是 FAQ 的核心来源。高频问题往往是新人痛点、流程不清或文档缺失的信号。

#### 3.2 输入与输出

**输入**：节点 1 清洗记录

**输出**：高频问题清单（问题 + 出现次数 + 提问者）

#### 3.3 使用顺序

1. 先用「快速生成版」识别高频问题。
2. 问题表述多样、需归并同义问题时，改用「专家增强版」。
3. 用「自查审稿版」检查是否漏掉高频问题、是否误并不同问题。
4. 有问题则用「返修优化版」修正。
5. 对照 3.6 验收清单确认，通过后交给节点 4。

#### 3.4 提示词包

##### A. 快速生成版

```
你是一位高频问题识别员。请找出群里反复出现的问题。

清洗记录：【粘贴节点1】

输出：
| 问题 | 大致出现次数 | 是否有人解答 |
把表述不同但本质相同的问题归并。只统计实际出现的问题，不臆造。
```

##### B. 专家增强版

```
你是一位高频问题分析专家。

任务：识别群里被反复提及的高频问题。

输入：
- 清洗记录：【粘贴节点1】

处理步骤：
1. 识别所有提问（显式问题和隐式求助）
2. 将表述不同但本质相同的问题归并（如"怎么报销"和"报销流程是啥"）
3. 统计每个问题的出现频次
4. 标注每个问题是否得到解答、解答是否一致
5. 按频次排序，标注高频问题（被问 2 次以上的）

输出格式：
| 问题（归并后）| 出现次数 | 原始表述示例 | 是否有解答 | 解答是否一致 |

约束：归并基于问题本质，不硬合并不同问题；频次基于实际记录；不臆造没出现的问题。

质量标准：高频问题准确识别，同义问题正确归并，为 FAQ 提供可靠来源。

常见失败情况：把不同问题硬归并；漏掉隐式求助；频次统计不准。
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下高频问题识别是否存在以下问题：
1. 是否把本质不同的问题硬归并了？
2. 是否漏掉了隐式求助形式的问题？
3. 出现次数统计是否基于实际记录？
4. 是否臆造了没实际出现的问题？

【粘贴高频问题识别】

逐条说明问题。
```

##### D. 返修优化版

```
根据自查意见修正高频问题识别。拆分误并的问题，补回隐式问题，校正频次，删除臆造问题。

原始识别：【粘贴原始识别】
自查意见：【粘贴自查结果】
```

#### 3.5 交付给下游节点

将高频问题清单复制，交给节点 4（有效回答提取专家）。

#### 3.6 人工验收清单

- [ ] 是否没有把不同问题硬归并？
- [ ] 是否覆盖了隐式求助？
- [ ] 出现次数是否基于实际记录？

### 节点 4：有效回答提取专家

#### 4.1 节点定位

为高频问题提取群里的有效回答，区分"权威答案""可参考答案""错误或过时答案"。不是群里所有回答都对，要甄别。

#### 4.2 输入与输出

**输入**：节点 3 高频问题 + 节点 1 清洗记录 + 关键人物信息

**输出**：问题对应的有效回答（答案 + 回答者 + 可信度）

#### 4.3 使用顺序

1. 先用「快速生成版」提取回答。
2. 回答有冲突或需判断权威性时，改用「专家增强版」。
3. 用「自查审稿版」检查是否把错误回答当有效、可信度判断是否合理。
4. 有问题则用「返修优化版」修正。
5. 对照 4.6 验收清单确认，通过后交给节点 8。

#### 4.4 提示词包

##### A. 快速生成版

```
你是一位回答提取员。请为高频问题提取群里的回答。

高频问题：【粘贴节点3】
清洗记录：【粘贴节点1】
关键人物（权威/负责人）：【填入】

输出：
| 问题 | 回答 | 回答者 | 可信度（权威/参考/存疑）|
权威人物或被多人认可的答案标"权威"；个人看法标"参考"；有冲突或被质疑的标"存疑"。
```

##### B. 专家增强版

```
你是一位有效回答甄别专家。

任务：为高频问题提取有效回答，甄别可信度。

输入：
- 高频问题：【粘贴节点3】
- 清洗记录：【粘贴节点1】
- 关键人物：【填入群里的权威/负责人】

处理步骤：
1. 为每个高频问题找出群里的所有回答
2. 甄别每个回答的可信度：
   - 权威：来自负责人/权威人物，或被多人确认无异议
   - 可参考：个人经验分享，未经权威确认
   - 存疑：有冲突答案、被质疑、或明显过时
3. 对同一问题的多个回答，标注是否一致、有无冲突
4. 对有冲突的回答，并列呈现，标注"需确认正确答案"
5. 不自己编造"正确答案"，只提取群里实际有的

输出格式：
| 问题 | 回答 | 回答者 | 可信度 | 一致性（与其他回答）|

约束（关键）：不编造群里没有的答案；不擅自判定哪个答案对（除非有权威表态）；有冲突的答案并列并标注需确认；过时答案标注。

质量标准：回答可信度甄别合理，权威答案和个人看法区分，冲突答案不被擅自裁决。

常见失败情况：把个人看法当权威答案；自己编造"标准答案"；冲突答案只取一个不标注分歧。
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下有效回答提取是否存在以下问题：
1. 是否把个人看法标成了"权威答案"？
2. 是否自己编造了群里没有的"标准答案"？
3. 有冲突的回答是否并列呈现并标注"需确认"？
4. 可信度判断是否有依据（权威性/多人确认）？

【粘贴有效回答提取】

逐条说明问题。
```

##### D. 返修优化版

```
根据自查意见修正回答提取。将误判的权威答案降级，删除编造的标准答案，并列呈现冲突答案，补充可信度依据。

原始提取：【粘贴原始提取】
自查意见：【粘贴自查结果】
```

#### 4.5 交付给下游节点

将有效回答复制，交给节点 8（FAQ 生成专家）。只有权威或一致的答案才进 FAQ。

#### 4.6 人工验收清单

- [ ] 是否没有把个人看法当权威答案？
- [ ] 是否没有编造群里没有的标准答案？
- [ ] 冲突答案是否并列标注"需确认"？

### 节点 5：共识结论整理专家（招牌节点）

#### 5.1 节点定位

**这是本工作流的招牌节点。** 准确区分群聊内容的性质——这是知识沉淀最关键、最容易出错的环节。本节点必须把内容分入以下类别：

| 类别 | 判定标准 | 沉淀去向 |
|------|---------|---------|
| 已达成共识 | 多人认可且无明显反对，或负责人拍板 | 可沉淀为结论 |
| 个人观点 | 仅某人提出，未获群体认可 | 保留为参考，不当结论 |
| 仍有争议 | 存在对立意见，未达成一致 | 标注争议，待解决 |
| 需负责人确认 | 涉及决策但负责人未表态 | 标注待确认 |
| 高频答案 | 反复出现的有效问答 | 进 FAQ |
| 背景信息 | 有价值但非结论性的讨论 | 保留为背景 |

群聊最大的陷阱是把"有人说过"当成"群体共识"，把热闹讨论当成"已经定了"。

3. 用「自查审稿版」检查是否把观点误判为共识。
4. 有问题则用「返修优化版」修正。
5. 对照 5.6 验收清单确认，通过后交给节点 6 和节点 8。

## 常见错误

**错误 1：把个人观点当成群体共识**

表现：某个活跃成员发表了一个看法，没人特别回应，整理时却写成"大家一致认为"，沉淀成了错误的"共识"。

修复：执行节点 5（招牌节点），共识必须有"多人认可或权威拍板"的依据，沉默不等于认可，个人观点单独保留为参考。

**错误 2：把存疑答案写进正式 FAQ**

表现：群里对某问题有几个不同答案还在争论，整理时随便选一个写进 FAQ，新人照着做出了错。

修复：执行节点 4 和节点 8，有冲突的答案并列标注"待确认"，只有权威或一致的答案才进正式 FAQ。

**错误 3：把聊天记录原样存档当成知识库**

表现：把群聊导出存个文件就算"沉淀"了，实际还是没人翻、找不到、新人照样重复问。

修复：完整执行工作流，从清洗到分类到 FAQ 到知识库结构，把零散聊天转化为可检索、可复用的结构化知识。

## 延伸玩法

- **变体 1：FAQ 速建版**：只走节点 1（清洗）→ 节点 3（高频问题）→ 节点 4（有效回答）→ 节点 8（FAQ），快速从群聊建立 FAQ。
- **变体 2：定期沉淀版**：每周/每月定期对群聊做一次沉淀，节点 9 增量更新知识库，逐步积累，避免一次性处理海量记录。
- **进阶组合**：与"工作流 16（会议纪要工作流）"结合，群聊里的决策用决策提取逻辑处理；与"工作流 20（评论分析工作流）"结合，对大型社群的海量讨论先做观点聚类再沉淀。
