# 工作流 24：从票据到报销申请的 AI 多专家财务报销工作流

**适用岗位**：行政人员、财务专员、各部门报销经办人、财务主管

**真实场景**：月底报销季来了，员工手上一堆票据——有发票、有收据、有网购截图、有境外小票——需要在财务截止日前整理好、归类对、填写准，还要避免被财务退回反复折腾。

**最终目标**：一套完整的报销交付物，包含：票据整理清单、费用归类表、异常票据识别报告、报销申请表草稿、附件整理说明、审核预检清单。

> ⚠️ **合规声明**：本工作流严禁编造或推断任何票据信息。所有票据字段（金额、日期、商家名称、税号）必须来自票据原件。无法确认的信息一律标注「需人工核实」，不得自行填写。

---

## 输入材料清单

- 所有票据原件（发票、收据、网购截图、电子票据，扫描件或照片均可）
- 报销单位的费用分类标准（如有，提供财务科目对照表）
- 差旅/活动背景说明（出差目的、活动名称、参与人员）
- 公司报销政策（标准餐补、住宿上限、差旅标准等）
- 历史报销记录（如有，用于对照）

---

## 工作流总览

```
节点 1：票据信息提取专家
  ↓ 输出：票据信息清单（字段化）
节点 2：费用归类专家
  ↓ 输出：费用分类对照表
节点 3：异常票据识别专家（招牌节点）
  ↓ 输出：异常票据报告（分级标注）
节点 4：政策合规检查专家
  ↓ 输出：合规检查报告
节点 5：报销申请表生成专家
  ↓ 输出：报销申请表草稿
节点 6：附件整理说明专家
  ↓ 输出：附件清单与整理说明
节点 7：财务审核预检专家
  ↓ 输出：审核预检清单
节点 8：退回问题处理专家
  ↓ 输出：退回问题处理记录
节点 9：报销归档专家
  ↓ 输出：报销归档记录
最终交付：完整报销申请包
```

---

## 专家节点详解

每个节点包含六部分：节点定位、输入与输出、使用顺序、提示词包（A 快速 / B 专家 / C 自查 / D 返修）、交付给下游节点、人工验收清单。

**C 自查审稿版和 D 返修优化版是当前节点内部的工作模式，不是新的专家角色。**

> 核心原则：票据信息只来自票据原件。无法识别的字段标注「需人工核实」，不猜测，不填写默认值，不编造。

---

### 节点 1：票据信息提取专家

#### 1.1 节点定位

将每张票据的关键字段逐一提取为结构化信息，为后续归类和审核打基础。票据信息混乱或不完整是报销被退回的首要原因，先提取清楚才能准确判断。

#### 1.2 输入与输出

**输入**：所有票据原件（扫描件/照片/电子票据）

**输出**：票据信息清单（每张票据一行：编号、类型、日期、金额、商家、税号、票据状态）

#### 1.3 使用顺序

1. 先用「快速生成版」批量提取票据字段。
2. 票据清晰度较差或字段复杂时，用「专家增强版」。
3. 用「自查审稿版」检查是否有推断填写的字段。
4. 有问题则用「返修优化版」修正。
5. 对照 1.6 验收清单确认，通过后交给节点 2 和节点 3。

#### 1.4 提示词包

##### A. 快速生成版

```
你是票据信息提取专家。根据以下票据描述，逐张提取关键字段，输出表格：票据编号 | 票据类型 | 开票日期 | 金额（含税）| 商家/供应商名称 | 税号（如有）| 票据状态（清晰/模糊/残缺）。无法从票据识别的字段填写「需人工核实」，不得推断或填写默认值。
票据信息：【粘贴每张票据的描述或 OCR 识别内容，逐张标编号】
```

##### B. 专家增强版

```
你是票据字段提取专家，专注于将各类票据转化为结构化可核查的信息记录。

输入：票据原件描述（OCR 识别结果或人工描述）

对每张票据提取以下字段：
1. 票据编号（自编，如 R001、R002）
2. 票据类型：增值税专用发票 / 增值税普通发票 / 收据 / 网购截图 / 电子发票 / 境外票据 / 其他
3. 开票日期（年/月/日，格式统一）
4. 金额（含税总金额，单位元，保留两位小数）
5. 商家/供应商名称（票面原文）
6. 税号（如票面有，原文抄录；无则填「无」）
7. 票据备注（票面注明的用途，如有）
8. 票据状态：清晰 / 模糊（需人工确认）/ 残缺（关键字段缺失）

约束：
- 所有字段必须来自票据原件，不得推断
- 无法识别的字段一律填「需人工核实」，不填猜测值
- 金额模糊时填「需人工核实」，不按常见金额推测

输出：结构化表格，每张票据一行
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下票据信息清单：
① 是否有任何字段是推断填写的（不是来自票据原件）？
② 模糊或无法识别的字段是否已标注「需人工核实」？
③ 金额是否保留两位小数？
④ 日期格式是否统一？
【粘贴清单内容】
逐条列出问题。
```

##### D. 返修优化版

```
根据自查意见修正票据清单。保留：来自票据原件的准确字段。修正：将推断值替换为「需人工核实」、统一日期格式、修正金额格式。
原始清单：【粘贴原文】
自查意见：【粘贴自查结果】
```

#### 1.5 交付给下游节点

将票据信息清单交给节点 2（费用归类）和节点 3（异常识别）。标注所有「需人工核实」项，这些项在报销提交前必须由经办人手动确认。

#### 1.6 人工验收清单

- [ ] 每张票据都有编号且不重复
- [ ] 所有字段来自票据原件，无推断值
- [ ] 无法识别的字段已标注「需人工核实」
- [ ] 金额格式统一（两位小数，单位元）
- [ ] 日期格式统一

### 节点 2：费用归类专家

#### 2.1 节点定位

将每张票据归入对应的费用科目。归类错误会导致会计科目不平、报销被退回、甚至影响税务处理。归类标准以公司财务科目对照表为准，无对照表时按常规科目处理并标注供财务确认。

#### 2.2 输入与输出

**输入**：票据信息清单（节点 1）+ 公司费用分类标准（如有）

**输出**：费用分类对照表（票据编号 → 建议科目 → 归类依据 → 需确认项）

#### 2.3 使用顺序

1. 先用「快速生成版」生成初版分类。
2. 科目较复杂或有混合消费票据时，用「专家增强版」。
3. 用「自查审稿版」检查归类是否有依据。
4. 有问题则用「返修优化版」修正。
5. 对照 2.6 验收清单确认，**财务人员复核**后交给节点 4。

#### 2.4 提示词包

##### A. 快速生成版

```
你是费用归类专家。根据以下票据清单，将每张票据归入对应费用科目（差旅费/餐费/办公用品/交通费/住宿费/会议费/培训费/其他），给出归类依据（票据类型+商家名称）。无法确定科目的标注「需财务确认」，不强行归类。
票据清单：【粘贴节点 1 清单】
费用科目标准：【粘贴公司标准，或填「使用常规科目」】
```

##### B. 专家增强版

```
你是财务费用归类专家，专注于将票据准确归入对应会计科目。

输入：票据信息清单 + 公司费用分类标准

执行步骤：
1. 逐张归类：基于票据类型、商家名称、票面用途，归入最匹配的费用科目
2. 归类依据：每条说明"根据XX（票面信息）归入XX科目"
3. 混合消费处理：一张发票含多类费用时，标注「混合消费，需拆分或人工确认」
4. 无法确定项：商家名称无法判断消费类型的，标注「需人工核实：请确认该票据的消费用途」

常见科目参考（无公司标准时使用）：
- 差旅费：机票、火车票、高铁票、汽车票
- 住宿费：酒店、民宿发票
- 餐费：餐厅发票、餐饮收据（需在公司政策范围内）
- 交通费：出租车、滴滴、地铁、停车费
- 办公用品：文具、耗材、小额设备
- 会议费：会议室、会议服务
- 培训费：课程、培训机构
- 其他：无法归类的标注科目为「其他」并说明原因

约束：归类建议仅供参考，最终科目由财务人员确认。

输出：分类对照表（票据编号 | 建议科目 | 归类依据 | 是否需确认）
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下费用分类对照表：
① 每条归类是否有票据字段作为依据（不是猜测）？
② 混合消费票据是否已标注「需拆分或人工确认」？
③ 无法确定科目的是否已标注「需财务确认」？
【粘贴分类表内容】
逐条列出问题。
```

##### D. 返修优化版

```
根据自查意见修正分类表。保留：有票据字段依据的归类。修正：将猜测归类替换为「需财务确认」、标注混合消费票据。
原始分类表：【粘贴原文】
自查意见：【粘贴自查结果】
```

#### 2.5 交付给下游节点

将分类对照表交给财务人员确认后，连同票据清单一起传递给节点 3 和节点 4。

#### 2.6 人工验收清单

- [ ] 每张票据都有归类结果（或标注「需财务确认」）
- [ ] 归类有票据字段依据
- [ ] 混合消费票据已标注
- [ ] 财务人员已复核分类结果

### 节点 3：异常票据识别专家（招牌节点）

#### 3.1 节点定位

在提交报销前，主动识别可能导致退回或合规风险的异常票据。异常识别不是为了"抓人"，而是让经办人在提交前就知道哪些票据需要补充说明、哪些需要替换、哪些可能无法报销——避免提交后被财务退回反复折腾。

> **识别边界**：异常识别基于票据本身的可观察特征（字段缺失、格式问题、逻辑矛盾）和公司报销政策。不对票据真实性做最终判断，存疑票据一律标注「需人工核实」，由人工决定是否接受。严禁基于推断编造票据问题，也严禁对存疑票据擅自标注为「伪造」。

#### 3.2 输入与输出

**输入**：票据信息清单（节点 1）+ 公司报销政策

**输出**：异常票据报告（分三级：关注 / 需补充说明 / 建议退回，每项附具体异常描述）

#### 3.3 使用顺序

1. 先用「快速生成版」快速扫描明显异常。
2. 用「专家增强版」系统检查全部异常类型。
3. 用「自查审稿版」确认异常标注有依据且分级准确。
4. 有问题则用「返修优化版」修正。
5. 对照 3.6 验收清单确认，将报告交给经办人处理后再提交报销。

#### 3.4 提示词包

##### A. 快速生成版

```
你是异常票据识别专家。根据以下票据清单，识别可能存在问题的票据，分三级标注：①关注（轻微问题，可附说明后报销）②需补充说明（缺少必要信息，需经办人补充）③建议退回（不符合报销条件，建议不提交）。每条异常必须说明具体问题（来自票据字段），不得编造异常。无法判断的标注「需人工核实」。
票据清单：【粘贴节点 1 清单】
报销政策：【粘贴公司政策，或填「使用通用标准」】
```

##### B. 专家增强版

```
你是财务票据合规审核专家，专注于在报销提交前识别异常票据，帮助经办人提前处理问题。

输入：票据信息清单 + 公司报销政策

逐一检查以下异常类型：

**字段类异常：**
1. 日期异常：票据日期晚于报销日期、票据日期与出差/活动时间不符
2. 金额异常：金额字段为「需人工核实」（无法读取）、金额明显异常（如住宿单日超过政策上限）
3. 税号缺失：增值税发票缺少税号（影响税务抵扣）
4. 商家名称模糊：商家名称无法识别（票面模糊）

**格式类异常：**
5. 非正式票据：收据、网购截图、境外票据（是否符合公司政策）
6. 手写修改：票面有手写涂改痕迹（标注位置）
7. 复印件：票据为复印件而非原件

**逻辑类异常：**
8. 时间重叠：同一天有两张异地票据（逻辑上不可能同时存在）
9. 超标准：金额超过公司政策上限（餐标/住宿标准）
10. 用途不符：票面用途与申报用途不一致

**标注规则：**
- 关注：问题轻微，附说明后通常可报销
- 需补充说明：缺少必要信息，经办人需补充材料或说明
- 建议退回：不符合报销条件，建议不纳入本次报销
- 需人工核实：无法从票据信息判断是否异常，需人工确认

约束：
- 每条异常必须说明具体依据（哪个字段，什么问题）
- 不得对票据真实性做最终判断（不写「伪造」、「虚假」等定性词）
- 存疑的情况一律标注「需人工核实」

输出：异常票据报告（票据编号 | 异常等级 | 异常描述 | 建议处理方式）
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下异常票据报告：
① 每条异常标注是否有票据字段中的具体依据（不是推断）？
② 是否有对票据真实性做最终定性（如「伪造」、「虚假」）？
③ 无法判断的是否已标注「需人工核实」而非强行归类？
④ 异常等级（关注/需补充/建议退回）是否与问题严重程度匹配？
【粘贴报告内容】
逐条列出问题。
```

##### D. 返修优化版

```
根据自查意见修正异常报告。保留：有票据字段依据的异常标注。修正：删除无依据的推断性标注、将定性词替换为描述性词（如「需人工核实是否真实」代替「伪造」）、调整与问题不符的异常等级。
原始报告：【粘贴原文】
自查意见：【粘贴自查结果】
```

#### 3.5 交付给下游节点

将异常票据报告交给经办人，由经办人决定：补充说明、替换票据、或从本次报销中移除。处理完异常后，更新后的票据清单交给节点 4。

#### 3.6 人工验收清单

- [ ] 10 类异常类型均已检查
- [ ] 每条异常有具体票据字段依据
- [ ] 无对票据真实性的最终定性（无「伪造」等词）
- [ ] 「需人工核实」项已标注（未强行归类）
- [ ] 经办人已确认所有异常项的处理方式

### 节点 4：政策合规检查专家

#### 4.1 节点定位

对照公司报销政策，检查费用是否在允许报销的范围和标准内。不同公司的报销政策差异很大（餐标、住宿上限、出行方式要求），本节点确保提交的报销申请不会因政策违规被退回。

#### 4.2 输入与输出

**输入**：费用分类对照表（节点 2）+ 异常处理后的票据清单 + 公司报销政策

**输出**：合规检查报告（合规项 / 超标项 / 不在报销范围项）

#### 4.3 使用顺序

1. 先用「快速生成版」快速检查明显超标项。
2. 政策较复杂或有特殊费用时，用「专家增强版」。
3. 用「自查审稿版」确认检查覆盖所有政策要求。
4. 有问题则用「返修优化版」修正。
5. 对照 4.6 验收清单确认，超标项处理后交给节点 5。

#### 4.4 提示词包

##### A. 快速生成版

```
你是报销政策合规专家。根据以下费用清单和报销政策，检查每笔费用的合规性，输出：合规（符合政策）/ 超标（超出上限，说明超出金额）/ 不在范围（该类费用不在报销范围内）。每条结论附政策依据。无对应政策的标注「需财务确认」。
费用清单：【粘贴节点 2 分类表】
报销政策：【粘贴政策内容】
```

##### B. 专家增强版

```
你是财务报销政策合规审查专家，专注于在申请提交前识别超标和不合规费用。

输入：费用分类对照表 + 公司报销政策

检查维度：
```

### 节点 5：报销申请表生成专家

#### 5.3 使用顺序

1. 先用「快速生成版」生成基础表格。

#### 7.2 输入与输出

2. 用「专家增强版」系统检查所有常见退回原因。
3. 用「自查审稿版」确认无遗漏。

### 节点 8：退回问题处理专家

#### 8.1 节点定位

#### 8.2 输入与输出

**输入**：财务退回说明 + 原报销申请材料

#### 8.3 使用顺序

1. 收到退回说明后，用「快速生成版」分析原因。

## 常见错误

**错误 1：模糊字段自行填写**

表现：票据金额模糊，经办人凭印象填了一个"差不多"的金额。

修复：节点 1 所有无法确认的字段一律标注「需人工核实」，金额不确认不提交。

---

**错误 2：票据归类错误导致会计科目不平**

表现：出租车费被归入差旅费（合理），但高铁票也被归入交通费（应是差旅费），导致科目拆分不一致。

修复：节点 2 的归类标准以公司财务科目对照表为准；无对照表时，向财务确认后再填写。

---

**错误 3：异常票据未处理就提交**

表现：节点 3 识别出有一张日期是三个月前的发票（超出报销有效期），但经办人没有处理就一并提交，导致整份报销被退回。

修复：节点 3 的异常报告中「建议退回」等级的票据，必须在提交前由经办人决定处理方式，不得跳过。

---

**错误 4：合计金额与票据不一致**

表现：申请表合计 3,240 元，但票据加总实际是 3,420 元，导致被财务发现退回。

修复：节点 5 的自查审稿版强制做加总验算，任何差异都必须修正后才能提交。

## 延伸玩法

- **差旅报销批量处理**：将节点 1-3 应用于整个团队的差旅票据，统一提取后再分人归类，提高团队报销效率。
- **报销政策快速查询**：将公司报销政策整理成节点 4 的政策对照表，在节点 4 的 B 专家增强版中常备，每次报销直接调用。
- **与工作流 16 组合**：将会议纪要（工作流 16）中记录的会议费用和参与人员信息，作为节点 5 报销申请表的事由说明依据，一致性更高，审批通过率更高。
