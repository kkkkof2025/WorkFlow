# 工作流 29：从日记周报到成长复盘的 AI 多专家工作流

## 基本信息

- **工作流名称**：从日记周报到成长复盘的 AI 多专家工作流
- **适用岗位**：职场人、自学者、项目负责人、学生、自由职业者
- **真实场景**：一名产品经理月底坐下来想做复盘，翻出一个月的日报、周报、会议记录，发现每个月复盘都写的是同样的问题，但下个月照旧。需要一个能真正找出问题根源和行为模式的复盘工具，而不是写一篇自我感觉良好的总结。
- **最终目标**：生成一份有诊断深度的复盘报告，包含事件分类、情绪变化分析、**反复出现的问题模式**识别、成长证据、决策复盘和下阶段可执行行动计划。不是鸡汤，不是美化，是真实的诊断。

---

## 输入材料清单

1. 原始记录（日记、日报、周报、任务完成情况、会议记录等）
2. 复盘周期（一周 / 一个月 / 一个项目周期）
3. 上一次复盘报告（如有）
4. 本阶段的主要目标或任务列表（如有）
5. 特别想重点复盘的事件（可选）

---

## 工作流总览

```
[日记/周报/任务记录]
         ↓
  专家1：原始记录整理
         ↓
  专家2：事件分类
         ↓
  专家3：情绪变化识别
         ↓
  专家4：问题模式识别 ★招牌节点
         ↓
  专家5：成长证据提取
         ↓
  专家6：决策复盘
         ↓
  专家7：下阶段行动
         ↓
  专家8：风险提醒
         ↓
  专家9：复盘报告生成
         ↓
[复盘报告（成果+问题模式+决策分析+行动计划）]
```

## 专家节点详解

### 专家 1：原始记录整理专家

#### 1.1 节点定位

将凌乱的日记、日报、周报整理为时间轴上的事件清单，去掉无信息量的流水账，保留有实质内容的事件。

#### 1.2 输入与输出

- **输入**：原始记录文本（日记/日报/周报）
- **输出**：事件清单（时间 + 事件描述 + 结果/状态）

#### 1.3 使用顺序

第一步，为后续所有分析节点提供结构化的原材料。

#### 1.4 提示词包

**A. 快速生成版**

```
以下是我[X月]的工作/生活记录，请整理为事件清单，每条包含：时间、事件描述（1-2句）、结果/状态（完成/未完成/进行中/搁置）：
[粘贴原始记录]
```

**B. 专家增强版**

```
【角色】你是一名信息整理专家，擅长从流水账式记录中提取有实质内容的事件。

【任务】将以下原始记录整理为结构化事件清单。

【输入材料】
- 原始记录：[粘贴文本]
- 时间范围：[起止日期]

【处理步骤】
1. 逐段阅读，识别有实质内容的事件（不是"上班/吃饭/开了个会"这类流水账）
2. 每个事件提炼为1-2句描述，包含：做了什么/结果是什么/有什么感受
3. 标注事件状态：完成 / 未完成 / 搁置 / 进行中
4. 标注事件类型（初步）：工作任务 / 人际沟通 / 决策 / 学习 / 突发情况
5. 保留原始记录中的情绪信号词（"又" "但是" "没想到" "终于"等），不要删除

【输出格式】
| 日期 | 事件描述 | 状态 | 类型 | 情绪信号 |

【约束条件】不添加原始记录中没有的内容；不美化或贬低已发生的事件

【质量标准】整理后的事件清单能让人在10分钟内了解整个复盘周期发生了什么

【常见失败情况】把"开了个会"保留为一条事件；删除了带情绪色彩的词语
```

**C. 自查审稿版**

> 说明：本版本是当前节点的内部审查模式，不是新专家角色。

```
检查事件清单：
1. 是否保留了情绪信号词？
2. 是否排除了无实质内容的流水账？
3. 每条事件是否有明确的状态标注？
[粘贴事件清单]
```

**D. 返修优化版**

> 说明：本版本是当前节点的内部优化模式，不是新专家角色。

```
根据审查意见修正事件清单：
[审查意见] [原清单]
修正：补充情绪信号词，删除无实质内容条目，补充缺失的状态标注。
```

#### 1.5 交付给下游节点

将事件清单交给**事件分类专家（节点2）**和**情绪变化识别专家（节点3）**。

#### 1.6 人工验收清单

- [ ] 是否保留了原始记录中的情绪信号词？
- [ ] 是否去掉了无实质内容的流水账？
- [ ] 每条事件是否有状态标注？

### 专家 2：事件分类专家

#### 2.1 节点定位

将事件清单按类别归类，为模式识别（节点4）提供按类聚合的视角。

#### 2.2 输入与输出

- **输入**：事件清单（节点1）
- **输出**：分类后的事件矩阵（类别 × 事件 × 结果）

#### 2.3 使用顺序

第二步，在事件清单完成后进行。

#### 2.4 提示词包

**A. 快速生成版**

```
请将以下事件清单按类别归类，类别建议：工作任务执行/人际沟通协作/个人决策/学习成长/突发应对。并统计每类的完成率：
[粘贴事件清单]
```

**B. 专家增强版**

```
【角色】你是一名行为分析专家，擅长从事件中识别行为模式。

【任务】对事件清单进行分类，为后续模式识别做准备。

【输入材料】事件清单：[节点1输出]

【处理步骤】
1. 将事件归入以下类别（每个事件只归一个主类）：
   - 工作任务执行（计划内的任务推进）
   - 人际沟通协作（与他人的互动、协商、冲突）
   - 个人决策（需要做选择的时刻）
   - 学习成长（主动学习、技能提升）
   - 突发应对（计划外的情况处理）
2. 统计每类事件数量和完成率（完成事件/该类总事件）
3. 标注每类中"结果不如预期"的事件（这些是后续重点分析对象）
4. 识别哪类事件占比最高（可能暗示注意力分配问题或工作性质）

【输出格式】
分类矩阵（类别/事件数/完成率/不如预期事件数）+ 类别占比分析

【约束条件】不改变事件内容；分类有争议时取主要属性

【质量标准】分类后能清楚看出"哪类事情做得好/哪类做得差"

【常见失败情况】类别过多（每种情况一类）；不统计完成率只分类
```

**C. 自查审稿版**

> 说明：本版本是当前节点的内部审查模式，不是新专家角色。

```
检查事件分类：
1. 是否统计了每类的完成率？
2. 是否标注了"结果不如预期"的事件？
3. 类别总数是否合理（5-8类，不超过10类）？
[粘贴分类结果]
```

**D. 返修优化版**

> 说明：本版本是当前节点的内部优化模式，不是新专家角色。

```
根据审查修正分类：
[审查意见] [原分类]
修正：补充完成率统计，标注不如预期事件，合并细碎类别。
```

#### 2.5 交付给下游节点

将分类矩阵交给**问题模式识别专家（节点4）**，重点传递"结果不如预期"事件列表。

#### 2.6 人工验收清单

- [ ] 是否统计了每类事件的完成率？
- [ ] 是否标注了"结果不如预期"的事件？
- [ ] 类别数量是否合理（不超过 10 类）？

### 专家 3：情绪变化识别专家

#### 3.1 节点定位

从事件记录中识别情绪变化的规律，找出哪类事件持续引发负面情绪，哪类事件带来正向体验。

#### 3.2 输入与输出

- **输入**：事件清单（含情绪信号词）（节点1）
- **输出**：情绪变化曲线描述 + 高频负面情绪触发事件类型

#### 3.3 使用顺序

第三步，与事件分类并行或之后进行。

#### 3.4 提示词包

**A. 快速生成版**

```
请从以下事件清单中，识别情绪变化规律：哪类事件总是带来负面情绪？哪类带来正向体验？高频出现的情绪信号词是什么？
[粘贴事件清单（含情绪信号词）]
```

**B. 专家增强版**

```
【角色】你是一名情绪分析专家，专注于从文字记录中识别情绪模式，而不是心理咨询。

【任务】分析事件记录中的情绪变化规律。

【输入材料】带情绪信号词的事件清单：[节点1输出]

【处理步骤】
1. 识别情绪信号词（"又/终于/没想到/烦/好不容易/浪费时间"等）
2. 将情绪信号归类：负面（挫败/焦虑/厌烦/委屈）/ 正面（成就/轻松/意外惊喜）/ 中性
3. 统计高频触发负面情绪的事件类型（不是单个事件，而是类型）
4. 统计高频触发正面情绪的事件类型
5. 识别情绪与工作绩效的关联（情绪低落的周/天，任务完成率是否明显下降）

【输出格式】
情绪信号词频统计 + 高频负面触发类型（含说明）+ 高频正面触发类型 + 情绪-绩效关联说明

【约束条件】
- 不做心理诊断，不使用"抑郁/焦虑症"等临床词汇
- 只分析文字记录中出现的信号，不推断未记录的状态
- 情绪分析结论必须有原文中的证据（引用具体词语或语句）

【质量标准】情绪分析结论必须可以被原文中的具体词语支撑

【常见失败情况】没有引用原文证据就下情绪结论；使用临床诊断词汇
```

**C. 自查审稿版**

> 说明：本版本是当前节点的内部审查模式，不是新专家角色。

```
检查情绪分析：
1. 每个情绪结论是否有原文词语支撑？
2. 是否避免了临床诊断词汇？
3. 是否区分了事件类型而不是单一事件？
[粘贴情绪分析结果]
```

**D. 返修优化版**

> 说明：本版本是当前节点的内部优化模式，不是新专家角色。

```
根据审查修正情绪分析：
[审查意见] [原分析]
修正：为结论补充原文引用，删除临床词汇，将单一事件分析改为类型分析。
```

#### 3.5 交付给下游节点

将高频负面情绪触发类型交给**问题模式识别专家（节点4）**，作为模式识别的辅助维度。

#### 3.6 人工验收清单

- [ ] 每个情绪结论是否有原文词语支撑？
- [ ] 是否避免了临床诊断词汇？
- [ ] 是否识别了高频负面情绪触发的事件类型（而不是单一事件）？

### 专家 4：问题模式识别专家 ★招牌节点

#### 4.1 节点定位

识别在复盘周期内**反复出现的问题行为模式**。不是分析单次失误，而是找到跨事件的规律性问题。不写鸡汤，只写诊断。

#### 4.2 输入与输出

- **输入**：分类后的"结果不如预期"事件（节点2）、情绪分析（节点3）、上一次复盘报告（如有）
- **输出**：问题模式清单（每个模式含：模式名称、出现次数、典型事件证据、触发条件、代价）

#### 4.3 使用顺序

第四步，是复盘工作流最核心的分析节点。

#### 4.4 提示词包

**A. 快速生成版**

```
请从以下"结果不如预期"的事件中，识别反复出现的问题模式（不是单次失误，而是规律）：
[粘贴不如预期事件列表]
```

**B. 专家增强版**

```
【角色】你是一名行为诊断专家，专门识别人的行为模式和系统性问题，不做心理咨询，只做行为分析。你的分析风格是直接、具体、有证据、不美化。

【任务】从以下事件数据中识别反复出现的问题模式。

【输入材料】
- 结果不如预期的事件列表：[节点2的"不如预期"事件]
- 情绪分析报告：[节点3输出]
- 上期复盘报告（如有）：[粘贴]

【处理步骤】
对每个候选模式，按以下结构完整分析：

**模式A：拖延**
- 识别方式：任务截止日前最后X天才开始/反复延后任务完成时间/大量任务状态为"搁置"
- 分析维度：是哪类任务容易拖延？（复杂/无明确截止/需要协调他人/不擅长的领域）
- 触发条件：什么情况下拖延会发生？（任务不明确时/同时有多件事时/情绪低落时）
- 与上期比较：上期有无同样模式？有则说明是稳定模式，无则说明是本期特有

**模式B：过度承诺**
- 识别方式：答应了但最终未兑现的事项占比/被催促的频率/向别人道歉/解释的频率
- 分析维度：哪类请求容易答应？（上级要求/突发性请求/当面提出的请求）
- 触发条件：在什么情境下会承诺超出能力的事？
- 代价：过度承诺导致了哪些具体后果

**模式C：沟通不清**
- 识别方式：需要多次确认才能推进/被反馈"理解不一致"/反复解释同一件事
- 分析维度：是信息发出不清晰，还是信息接收有偏差？哪个环节断裂？
- 触发条件：异步沟通？多方沟通？时间紧迫时？

**模式D：目标漂移**
- 识别方式：月初计划与月末实际执行内容有明显偏差/原定重要任务被搁置/大量时间在计划外任务
- 分析维度：目标是如何漂移的？（被他人打断/自己主动转移/环境变化）
- 触发条件：什么类型的干扰最容易导致目标漂移？

**模式E：复盘不落地**
- 识别方式：上期复盘提出的改进建议，本期是否真的执行了？如果有上期复盘报告，逐条核查
- 分析维度：哪类改进建议最难落地？（需要持续习惯的/依赖他人配合的/需要资源的）
- 触发条件：为什么知道了问题还是没有改？

【对每个识别到的模式，输出】：
- 模式名称（简短，如"面对复杂任务的拖延"而不是只说"拖延"）
- 出现次数（本周期内）
- 典型事件证据（引用1-2个具体事件）
- 触发条件（具体情境）
- 本期代价（因为这个模式，实际付出了什么代价）
- 与上期比较（如有上期数据）

【重要约束】
- 不分析没有证据的模式（原文中没有出现过的行为不推断）
- 不美化：如果某个模式反复出现5次以上，明确说明"这是稳定的行为模式，不是偶发"
- 不鸡汤：不说"只要你意识到这个问题，就已经进步了"这类安慰性话语
- 不诊断性格：不说"你是完美主义者/拖延症患者"等性格标签，只描述行为
- 如果没有找到某个模式的证据，明确说"本期无[X]模式的明显证据"

【质量标准】
- 每个识别到的模式有至少1个具体事件作为证据
- 触发条件是可以观察到的情境描述，不是"心情不好时"等模糊原因
- 与上期对比能说明模式是在改善还是在固化

【常见失败情况】
- 把单次失误误判为模式（1次不够，至少2次同类才算模式）
- 触发条件写"当我不够努力时"等归因错误
- 写了改进建议但没有说改进什么、改到什么程度
```

**C. 自查审稿版**

> 说明：本版本是当前节点的内部审查模式，不是新专家角色。

```
对以下问题模式分析做审查：
1. 每个模式是否有至少2个具体事件证据（不是单次失误）？
2. 触发条件是否是可观察的情境，而不是"心情不好"等模糊原因？
3. 是否有明确说明哪些模式是"稳定模式"（反复≥3次）？
4. 是否避免了鸡汤式收尾（"意识到问题就是进步"）？
5. 是否避免了性格诊断（"你是拖延症/完美主义者"）？
[粘贴问题模式分析]
```

**D. 返修优化版**

> 说明：本版本是当前节点的内部优化模式，不是新专家角色。

```
根据审查意见修正问题模式分析：
[审查意见] [原分析]
修正：删除证据不足的模式，将模糊触发条件改为可观察情境，删除鸡汤收尾，删除性格诊断，补充稳定性判断。
```

#### 4.5 交付给下游节点

将问题模式清单（特别是稳定模式）交给**下阶段行动专家（节点7）**，作为改进行动的来源。

#### 4.6 人工验收清单

- [ ] 每个模式是否有至少 2 个具体事件证据？
- [ ] 触发条件是否是可观察的情境描述？
- [ ] 是否明确标注了"稳定模式"（出现≥3次）？
- [ ] 是否避免了鸡汤式收尾？
- [ ] 是否避免了性格诊断标签？

### 专家 5：成长证据提取专家

#### 5.1 节点定位

从事件记录中提取真实的成长证据，而不是主观感受。只记录可以被事件支撑的进步，不编造。

#### 5.2 输入与输出

- **输入**：事件清单（节点1）、分类矩阵（节点2）
- **输出**：成长证据清单（每条证据含事件引用和能力/结果维度）

#### 5.3 使用顺序

第五步，与问题模式分析并行或之后进行，提供平衡视角。

#### 5.4 提示词包

## 常见错误

| 错误表现 | 修正方式 |
|---------|---------|
| 把单次失误写成"问题模式" | 模式至少要有2次同类事件证据；单次失误记录在事件清单，不上升为模式 |
| 复盘报告越写越长但没有行动计划 | 改进行动必须是复盘报告的核心部分，且不超过3条 |
| 问题模式全是外部原因（"都是别人的问题"） | 重新聚焦在自己能控制的行为上；外部因素可以作为触发条件，但改进方向必须是自己的行为 |
| 每次复盘都发现同样问题但从不改变 | 激活节点4的"模式E：复盘不落地"分析，专门分析为什么知道了但没做 |
| 成长证据全是主观感受 | 每条成长证据必须引用具体事件；感受不算证据 |
| 行动计划写了8条，全都是高优先级 | 强制限制3条；删除掉的条目保存为"候选清单"，下次复盘再评估 |

## 人工验收清单

- [ ] 事件清单是否保留了情绪信号词？
- [ ] 每个问题模式是否有至少 2 个具体事件证据？
- [ ] 是否避免了把单次失误误判为模式？
- [ ] 是否明确标注了稳定模式（出现≥3次）？
- [ ] 是否区分了决策质量和决策结果？
- [ ] 成长证据是否全部有具体事件支撑（无主观感受）？
- [ ] 改进行动是否不超过 3 条？
- [ ] 每条改进行动是否有可检验的完成标准？
- [ ] 是否避免了鸡汤式表述和性格诊断标签？
- [ ] 风险预警信号是否是可观察的具体迹象？
- [ ] 复盘报告是否可以作为下次复盘的对比基准？

## 延伸玩法

1. **跨期模式追踪**：保留3个月以上的复盘报告，每季度用节点4的问题模式识别专家做一次"跨期分析"，识别哪些模式在多个周期都出现（超稳定模式，需要更系统的改变）。

2. **决策日志**：在节点6之外，单独建立"决策日志"，记录每次重要决策时的信息状态和框架。三个月后回看，判断自己的决策质量是否在提升。

3. **行动完成率追踪**：每次复盘前，先核查上次的3条改进行动完成了几条。如果连续两次完成率低于50%，说明行动难度过高，需要重新设计。

4. **团队复盘适配**：将节点2的事件分类替换为"项目任务/团队协作/资源争议/客户反馈"四类，节点4的模式识别替换为团队级问题模式（不归因到个人），用于项目团队的迭代回顾。
