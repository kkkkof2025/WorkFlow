# 工作流 17：从零散工作材料到领导汇报稿的 AI 多专家工作流

**适用岗位**：职能人员、项目负责人、部门助理、管理人员、汇报材料撰写者

**真实场景**：领导让你明天汇报近期工作，你手上是一堆零散的材料——做过的事、遇到的问题、几个数据、一些未完成的项。你需要把它们整理成一份重点清楚、问题不甩锅、有价值的汇报稿，而不是流水账或空洞官话。

**最终目标**：一份务实的领导汇报稿，包含：提炼后的重点成果、恰当表达的问题与风险、数据支撑、清晰的逻辑结构、分寸得当的措辞，以及对领导可能追问的预判准备。

---

## 输入材料清单

- 零散工作材料（做过的事、进展、数据、问题、未完成项）
- 汇报对象（汇报给谁：直属领导 / 跨部门 / 高层）
- 汇报场景（例会 / 专项汇报 / 述职 / 紧急汇报）
- 汇报时长或篇幅要求（口头 5 分钟 / 书面一页）
- 特别关切（领导近期关注什么，如有）

---

## 工作流总览

```
节点 1：原始材料整理专家
  ↓ 输出：分类后的工作材料清单
节点 2：汇报对象分析专家
  ↓ 输出：汇报对象关注点 + 表达基调
节点 3：重点成果提炼专家
  ↓ 输出：有价值的成果（而非事务罗列）
节点 4：问题与风险表达专家（招牌节点）
  ↓ 输出：得体表达的问题/风险/困难/建议
节点 5：数据支撑专家
  ↓ 输出：核对后的数据支撑点
节点 6：逻辑结构专家
  ↓ 输出：汇报逻辑框架
节点 7：措辞分寸专家
  ↓ 输出：措辞调整建议
节点 8：汇报稿生成专家
  ↓ 输出：完整汇报稿
节点 9：领导追问预判专家
  ↓ 输出：追问预判 + 应答要点
最终交付：领导汇报稿 + 应答准备
```

---

## 专家节点详解

每个节点包含六部分：节点定位、输入与输出、使用顺序、提示词包（A 快速 / B 专家 / C 自查 / D 返修）、交付给下游节点、人工验收清单。

**C 自查审稿版和 D 返修优化版是当前节点内部的工作模式，不是新的专家角色。**

> 核心原则：汇报是把"做了什么"提炼成"有什么价值"，把"遇到的问题"表达得**实事求是又不推责**。不编造成果，不夸大也不隐瞒问题。

---

### 节点 1：原始材料整理专家

#### 1.1 节点定位

将零散的工作材料分类整理，分清哪些是成果、哪些是进展、哪些是问题、哪些是数据。汇报前先把手里的"原材料"理清，才能提炼。

#### 1.2 输入与输出

**输入**：零散工作材料 + 汇报场景

**输出**：分类后的材料清单（成果 / 进行中 / 问题 / 数据 / 未完成）

#### 1.3 使用顺序

1. 先用「快速生成版」分类整理。
2. 材料杂乱或跨多个项目时，改用「专家增强版」。
3. 用「自查审稿版」检查分类是否准确、是否遗漏。
4. 有问题则用「返修优化版」修正。
5. 对照 1.6 验收清单确认，通过后交给节点 3。

#### 1.4 提示词包

##### A. 快速生成版

```
你是一位工作材料整理员。请将以下零散材料分类。

工作材料：【粘贴所有零散材料】

按以下类别整理：
1. 已完成的成果
2. 进行中的工作
3. 遇到的问题
4. 相关数据
5. 未完成/待推进的事项
忠实分类，不夸大成果，不隐瞒问题。
```

##### B. 专家增强版

```
你是一位工作材料梳理专家。

任务：将零散工作材料分类整理，为汇报提炼打基础。

输入：
- 工作材料：【粘贴所有材料】
- 汇报场景：【填入例会/述职等】
- 汇报周期：【填入本周/本月/本季度】

处理步骤：
1. 按类别归集：已完成成果 / 进行中工作 / 遇到的问题 / 数据 / 未完成事项
2. 对每项标注与汇报周期的相关性（本期的 / 跨期延续的）
3. 标注每项的重要程度（核心工作 / 常规事务）
4. 标记信息不全的材料（如只有结论没有数据支撑）

输出格式：
| 类别 | 内容 | 重要程度 | 信息完整度 |

约束：忠实分类，不夸大成果也不淡化问题；不确定归类的标注存疑；不补充材料中没有的内容。

质量标准：材料分类清晰，核心工作和常规事务区分开，为后续提炼成果和表达问题做好准备。

常见失败情况：把常规事务当成核心成果；遗漏了问题类材料只留成果；把进行中的工作写成已完成。
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下材料分类是否存在以下问题：
1. 是否把进行中的工作误归为"已完成成果"？
2. 是否遗漏了问题类材料（只整理了成果）？
3. 核心工作和常规事务是否区分开？
4. 是否补充了材料中没有的内容？

【粘贴材料分类】

逐条说明问题。
```

##### D. 返修优化版

```
根据自查意见修正材料分类。将误判为已完成的进行中工作归位，补回遗漏的问题，区分核心与常规，删除凭空补充的内容。

原始分类：【粘贴原始分类】
自查意见：【粘贴自查结果】
```

#### 1.5 交付给下游节点

将分类材料清单复制，交给节点 3（重点成果提炼专家）。汇报对象分析（节点 2）可并行进行。

#### 1.6 人工验收清单

- [ ] 是否没有把进行中工作误写成已完成？
- [ ] 是否保留了问题类材料？
- [ ] 核心工作和常规事务是否区分？

### 节点 2：汇报对象分析专家

#### 2.1 节点定位

分析汇报对象的关注点和沟通偏好，确定汇报的基调和重点。同样的工作，向直属领导和向高层汇报，重点和措辞完全不同。

#### 2.2 输入与输出

**输入**：汇报对象 + 汇报场景 + 特别关切（如有）

**输出**：汇报对象关注点 + 表达基调 + 内容取舍建议

#### 2.3 使用顺序

1. 先用「快速生成版」分析汇报对象。
2. 汇报对象层级高或情况复杂时，改用「专家增强版」。
3. 用「自查审稿版」检查关注点判断是否合理。
4. 有问题则用「返修优化版」修正。
5. 对照 2.6 验收清单确认，通过后交给节点 3 和节点 6。

#### 2.4 提示词包

##### A. 快速生成版

```
你是一位汇报沟通顾问。请分析汇报对象的关注点。

汇报对象：【填入直属领导/跨部门/高层】
汇报场景：【填入】
领导近期关注（如有）：【填入或留空】

输出：
1. 汇报对象最关心什么（结果/过程/数据/风险）
2. 表达基调建议（详细/精炼/正式/务实）
3. 哪些内容该重点讲，哪些可略过
```

##### B. 专家增强版

```
你是一位职场汇报沟通专家。

任务：分析汇报对象，确定汇报的重点取舍和表达基调。

输入：
- 汇报对象：【填入层级和角色】
- 汇报场景：【填入】
- 特别关切：【填入领导近期关注的事项，如有】

分析维度：
1. 关注重点（该层级通常关心结果还是过程、数据还是趋势、风险还是机会）
2. 信息颗粒度（要详细过程还是只要结论）
3. 表达基调（正式/务实/简洁）
4. 内容取舍（哪些必讲、哪些选讲、哪些不必讲）
5. 沟通禁忌（该对象反感什么，如冗长、报喜不报忧、甩锅）

输出格式：
- 关注点分析
- 表达基调建议
- 内容取舍清单（必讲/选讲/略过）
- 沟通注意事项

约束：基于汇报对象的层级和场景合理推断，不臆测具体个人性格（除非用户提供）；建议务实，不教人迎合或粉饰。

质量标准：汇报者据此知道该讲什么、用什么基调、避免什么。

常见失败情况：不分对象一刀切；建议报喜不报忧；颗粒度判断错误（给高层讲太多细节）。
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下汇报对象分析是否存在以下问题：
1. 关注点判断是否符合该汇报对象的层级（给高层是否建议讲太多细节）？
2. 是否建议了"报喜不报忧"或粉饰（不应该）？
3. 内容取舍是否合理？

【粘贴汇报对象分析】

逐条说明问题。
```

##### D. 返修优化版

```
根据自查意见修正汇报对象分析。校正颗粒度建议，去除报喜不报忧的倾向，调整内容取舍。

原始分析：【粘贴原始分析】
自查意见：【粘贴自查结果】
```

#### 2.5 交付给下游节点

将汇报对象关注点和表达基调复制，交给节点 3（重点成果提炼专家）和节点 6（逻辑结构专家）。

#### 2.6 人工验收清单

- [ ] 关注点判断是否符合汇报对象层级？
- [ ] 是否没有建议报喜不报忧？
- [ ] 内容取舍是否合理？

### 节点 3：重点成果提炼专家

#### 3.1 节点定位

把"做了很多事"提炼成"有价值的成果"。汇报最忌讳流水账——做了 ABCDE 一长串，领导记不住也看不出价值。这一步要提炼出真正值得说的成果。

#### 3.2 输入与输出

**输入**：节点 1 分类材料 + 节点 2 汇报对象关注点

**输出**：重点成果清单（成果 + 价值 + 支撑事实）

#### 3.3 使用顺序

1. 先用「快速生成版」提炼成果。
2. 工作琐碎、价值不明显时，改用「专家增强版」。
3. 用「自查审稿版」检查是否夸大、是否把事务当成果。
4. 有问题则用「返修优化版」修正。
5. 对照 3.6 验收清单确认，通过后交给节点 5。

#### 3.4 提示词包

##### A. 快速生成版

```
你是一位成果提炼顾问。请把工作提炼成有价值的成果。

工作材料（已分类）：【粘贴节点1的成果和进行中部分】
汇报对象关注点：【粘贴节点2】

输出：
| 成果 | 体现的价值 | 支撑事实 |
要求：提炼"价值"而非罗列"做了什么"；不夸大，价值必须有事实支撑。
```

##### B. 专家增强版

```
你是一位汇报成果提炼专家。

任务：将零散的工作提炼为有价值、领导记得住的成果。

输入：
- 已分类材料：【粘贴节点1】
- 汇报对象关注点：【粘贴节点2】
- 汇报周期：【填入】

处理步骤：
1. 从"做了什么"中识别"产生了什么价值"（解决了什么问题、带来什么改善、推进了什么目标）
2. 把同类琐碎事务归并为一个有意义的成果（不逐条罗列）
3. 为每个成果找到支撑事实（数据、完成的交付物、可验证的结果）
4. 按对汇报对象的重要性排序
5. 区分"实打实的成果"和"过程性进展"，不把忙碌当成果

输出格式：
| 成果 | 价值（解决了什么/改善了什么）| 支撑事实 | 重要性 |

约束（关键）：不夸大、不编造价值；价值必须有事实支撑；没有实际价值的常规事务不强行包装成成果；区分成果和苦劳。

质量标准：成果清单让领导能快速看到工作的实际价值，每个成果经得起追问。

常见失败情况：把"开了多少会、发了多少文档"当成果；夸大价值缺乏支撑；流水账式罗列不提炼。
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下成果提炼是否存在以下问题：
1. 是否把"忙碌/苦劳"（开会、发文档）当成了"成果"？
2. 每个成果的价值是否有事实支撑（而非空泛宣称）？
3. 是否夸大了价值或编造了没有的成果？
4. 是否还是流水账式罗列，没有真正提炼？

【粘贴成果提炼】

逐条说明问题。
```

##### D. 返修优化版

```
根据自查意见修正成果提炼。剔除把苦劳当成果的项，为价值补充事实支撑，删除夸大编造，归并罗列项为有意义的成果。

原始成果提炼：【粘贴原始内容】
自查意见：【粘贴自查结果】
```

#### 3.5 交付给下游节点

将重点成果清单复制，交给节点 5（数据支撑专家）。问题表达（节点 4）可并行进行。

#### 3.6 人工验收清单

- [ ] 是否没有把忙碌/苦劳当成果？
- [ ] 每个成果的价值是否有事实支撑？
- [ ] 是否没有夸大或编造成果？

### 节点 4：问题与风险表达专家（招牌节点）

#### 4.1 节点定位

**这是本工作流的招牌节点。** 解决汇报中最微妙、最考验分寸的部分——怎么表达问题、风险、困难和建议。常见困境：

| 困境 | 错误做法 | 本节点要解决的 |
|------|---------|--------------|
| 怎么写问题但不甩锅 | 把责任推给别人/客观条件 | 客观陈述问题，聚焦解决而非追责 |
| 怎么写风险但不制造恐慌 | 危言耸听或轻描淡写 | 如实评估风险等级，给出应对 |
| 怎么写困难但不显得推责 | 强调困难为完不成找借口 | 说明困难同时表明已采取的应对 |
| 怎么写建议但不越权 | 替领导做决定或指手画脚 | 提供建议供参考，把决策权留给领导 |

汇报里的问题表达，既不能粉饰太平（领导被蒙在鼓里），也不能甩锅诉苦（显得不担当）。

#### 4.2 输入与输出

**输入**：节点 1 分类材料（问题/未完成部分）+ 节点 2 汇报对象

**输出**：得体表达的问题/风险/困难/建议（每项含表达方式和分寸说明）

#### 4.3 使用顺序

1. 先用「快速生成版」表达问题和风险。
2. 问题敏感或涉及他人责任时，必须使用「专家增强版」。
3. 用「自查审稿版」检查是否甩锅、是否制造恐慌、是否越权。
4. 有问题则用「返修优化版」修正。
5. 对照 4.6 验收清单确认，通过后交给节点 6。

#### 4.4 提示词包

##### A. 快速生成版

```
你是一位汇报表达顾问。请帮助得体地表达问题和风险。

问题与未完成事项：【粘贴节点1的问题部分】
汇报对象：【粘贴节点2】

对每个问题/风险，给出表达方式，遵循：
1. 客观陈述问题，不推责给他人或客观条件
2. 如实说风险，不夸大不淡化
3. 说困难时同时说明已采取的应对
4. 提建议但把决策权留给领导（用"建议""供参考"而非"必须""应该"）

输出：| 问题/风险 | 得体表达 | 配套的应对/建议 |
```

##### B. 专家增强版

```
你是一位资深汇报沟通专家，擅长把问题表达得实事求是又得体。

任务：帮助汇报者得体地表达问题、风险、困难和建议。

输入：
- 问题与未完成事项：【粘贴节点1相关部分】
- 汇报对象：【粘贴节点2】
- 这些问题的实际责任归属（如涉及）：【填入或留空】

四类表达原则：

1. 问题表达（不甩锅）
   - 客观陈述发生了什么，聚焦"现状和影响"而非"谁的错"
   - 涉及他人责任时，陈述事实即可，不主动归咎也不替人背锅
   - 表明自己已做或将做什么

2. 风险表达（不制造恐慌）
   - 如实评估风险等级（高/中/低），不夸大也不轻描淡写
   - 说明风险触发条件和潜在影响
   - 配套给出应对措施或建议，让风险"可控感"而非"失控感"

3. 困难表达（不推责）
   - 说困难是为了让领导了解真实情况，不是为完不成找借口
   - 每说一个困难，同时说明已采取的应对和还需要的支持
   - 避免诉苦式表达

4. 建议表达（不越权）
   - 用"建议""供参考""是否可以考虑"等措辞
   - 提供选项和利弊，把决策权留给领导
   - 不替领导拍板，不指手画脚

输出格式：
| 事项 | 类型 | 得体表达 | 配套应对/建议 | 分寸说明 |

约束（关键）：不甩锅、不诉苦、不危言耸听、不越权；问题如实但聚焦解决；建议措辞留有余地。

质量标准：领导读后能了解真实问题和风险，感受到汇报者的担当和分寸，而非推责或邀功。

常见失败情况：把问题归咎于客观条件或他人（甩锅）；强调困难为完不成铺垫（推责）；建议用命令式口吻（越权）；风险危言耸听或刻意淡化。
```

### 节点 5：数据支撑专家

#### 5.1 节点定位

为成果和问题找到数据支撑，并核对数据的准确性。有数据的汇报更可信，但数据必须真实准确——错误的数据比没有数据更糟。

#### 5.2 输入与输出

**输入**：节点 3 成果清单 + 原始材料中的数据

**输出**：核对后的数据支撑点（数据 + 对应成果/问题 + 口径说明）

#### 5.3 使用顺序

1. 先用「快速生成版」匹配数据支撑。
2. 数据多或口径需说明时，改用「专家增强版」。
3. 用「自查审稿版」检查数据是否真实、口径是否清楚。
4. 有问题则用「返修优化版」修正。
5. 对照 5.6 验收清单确认，通过后交给节点 6。

### 节点 7：措辞分寸专家

#### 7.1 节点定位

调整汇报的措辞分寸，让表达务实得体，既不卑微也不张扬，去除空话套话。措辞是汇报的"最后一公里"，分寸不当会让好内容打折扣。

#### 7.2 输入与输出

**输入**：节点 6 逻辑框架 + 节点 4 问题表达 + 节点 2 表达基调

**输出**：措辞调整建议（原表述 → 建议表述 + 理由）

#### 7.3 使用顺序

1. 先用「快速生成版」调整措辞。
2. 涉及敏感表达或正式场合时，改用「专家增强版」。
3. 用「自查审稿版」检查是否有空话套话、分寸是否得当。
4. 有问题则用「返修优化版」修正。
5. 对照 7.6 验收清单确认，通过后交给节点 8。

#### 7.4 提示词包

##### A. 快速生成版

```

调整原则：
1. 去除空话套话（赋能、抓手、闭环、提效等）
2. 务实具体，不夸大不卑微
3. 数字和事实说话，少用形容词

```

##### B. 专家增强版

```
你是一位职场措辞分寸专家。

输入：

- 问题表达：【粘贴节点4】

常见失败情况：用空泛词替换了具体内容；为显得"有水平"堆砌术语；成果夸大或问题表达过于卑微。

逐条说明问题。

##### B. 专家增强版

```
你是一位汇报应答准备专家。

任务：预判领导可能的追问，准备应答要点。

输入：
- 汇报稿：【粘贴节点8】
- 问题与风险：【粘贴节点4】
- 汇报对象关注点：【粘贴节点2】

预判维度：
1. 成果追问（这个成果具体怎么实现的、价值怎么衡量、能否持续）
2. 数据追问（数据来源、口径、为什么是这个数）
3. 问题追问（问题根因、为什么会发生、怎么解决、什么时候解决）

- 应答方向（如实回答 / 需要补充确认 / 提供方案）

## 常见错误

**错误 1：把"做了很多事"直接当汇报内容**

表现：汇报变成流水账，列了一大堆做过的事，领导记不住也看不出价值。

修复：执行节点 3，从"做了什么"提炼"产生了什么价值"，归并琐碎事务，突出真正的成果。

**错误 2：问题表达甩锅或诉苦**

表现：说问题时把责任推给客观条件或别的部门，或者强调困难为完不成铺垫，显得不担当。

修复：执行节点 4（招牌节点），客观陈述问题聚焦解决，说困难时同时说应对，把握"如实但不推责"的分寸。

**错误 3：用空话套话掩盖内容空洞**

表现：通篇"赋能、抓手、闭环、深入推进"，听起来很热闹，实际没说清楚做了什么、有什么价值。

修复：执行节点 7，清除空泛词，用数字、事实、具体结果说话。

## 延伸玩法

- **变体 1：书面报告版**：将汇报稿扩展为正式书面报告，增加背景说明和详细数据附表，适合需要存档的正式汇报。
- **变体 2：紧急汇报版**：对突发问题的紧急汇报，重点走节点 4（问题表达）→ 节点 6（结构）→ 节点 9（追问预判），快速准备"问题-影响-应对-建议"四段式汇报。
- **进阶组合**：与"工作流 16（会议纪要工作流）"结合，把会议纪要的决策和成果作为汇报材料输入；与"工作流 18（数据分析报告工作流）"结合，用数据分析结论作为汇报的数据支撑。
