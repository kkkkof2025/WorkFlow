# 工作流 16：从会议录音或文字到可追踪纪要的 AI 多专家工作流

**适用岗位**：行政人员、项目专员、部门助理、会议组织者、团队负责人

**真实场景**：一场一小时的会议刚结束，你手上是一份录音转文字稿（或自己记的零散笔记），需要在当天整理出一份能发给所有参会人、能被追踪执行的会议纪要——而不是把发言流水账复制一遍。

**最终目标**：一份可追踪的会议纪要，包含：会议基本信息、议题清单、关键观点、明确区分的决策事项、可执行的待办任务（含责任人和时间）、风险与争议点，以及会后追踪清单。

---

## 输入材料清单

- 会议记录（录音转文字稿 / 速记 / 个人笔记，任一形式）
- 会议基本信息（会议主题、时间、参会人名单、主持人）
- 会议目的（如已知：决策会 / 同步会 / 头脑风暴）
- 已有的会议议程（如有）
- 需要重点关注的事项（如有特定关切）

---

## 工作流总览

```
节点 1：会议材料整理专家
  ↓ 输出：清洗后的结构化会议记录
节点 2：议题识别专家
  ↓ 输出：议题清单（按讨论顺序）
节点 3：发言观点提取专家
  ↓ 输出：各议题下的关键观点（标注发言人）
节点 4：决策事项提取专家（招牌节点）
  ↓ 输出：分级的决策事项清单
节点 5：待办任务拆解专家
  ↓ 输出：可执行待办清单
节点 6：责任人与时间节点校准专家
  ↓ 输出：补全责任人和时间的待办清单
节点 7：风险与争议点识别专家
  ↓ 输出：风险与争议清单
节点 8：纪要格式生成专家
  ↓ 输出：完整会议纪要
节点 9：会后追踪清单专家
  ↓ 输出：会后追踪清单
最终交付：可追踪会议纪要
```

---

## 专家节点详解

每个节点包含六部分：节点定位、输入与输出、使用顺序、提示词包（A 快速 / B 专家 / C 自查 / D 返修）、交付给下游节点、人工验收清单。

**C 自查审稿版和 D 返修优化版是当前节点内部的工作模式，不是新的专家角色。**

> 核心原则：纪要必须区分**事实、观点、决策、待办、风险**。不能把所有发言都写成"会议决定"，也不能编造会议没有产生的结论。

---

### 节点 1：会议材料整理专家

#### 1.1 节点定位

将杂乱的录音转文字稿或零散笔记整理为结构清晰、可逐段处理的会议记录。原始记录常有口语重复、跑题、口误，先清洗才能准确提取信息。

#### 1.2 输入与输出

**输入**：原始会议记录 + 会议基本信息

**输出**：清洗后的结构化会议记录（去除口语冗余，保留完整信息，标注发言人）

#### 1.3 使用顺序

1. 先用「快速生成版」清洗记录。
2. 记录混乱或多人交叉发言时，改用「专家增强版」。
3. 用「自查审稿版」检查是否丢失了关键信息、是否擅自归纳。
4. 有问题则用「返修优化版」修正。
5. 对照 1.6 验收清单确认，通过后交给节点 2 和节点 3。

#### 1.4 提示词包

##### A. 快速生成版

```
你是一位会议记录整理员。请整理以下会议记录。

原始记录：【粘贴录音转文字或笔记】
参会人：【填入名单】

要求：
1. 去除口语重复、语气词、明显口误
2. 保留所有实质信息，不归纳不删减观点
3. 标注每段发言的发言人（能识别时）
4. 标记听不清或不确定的部分
```

##### B. 专家增强版

```
你是一位专业会议记录整理员。

任务：将原始会议记录清洗为结构清晰的可处理记录。

输入：
- 原始记录：【粘贴录音转文字或笔记】
- 参会人名单：【填入】
- 会议主题：【填入】

处理步骤：
1. 去除口语冗余（重复、语气词、跑题闲聊），但保留全部实质内容
2. 按发言人归集发言（标注谁说了什么）
3. 对识别不清的发言人标注"发言人不明"，不臆测
4. 标记不确定/听不清的部分（用【存疑】标注），不脑补
5. 保持时间顺序，便于后续追溯

输出格式：
- 结构化记录（按时间顺序，每段标注发言人）
- 存疑清单（听不清/发言人不明的部分）

约束（关键）：只做清洗和归集，不做归纳、不下结论、不补充原文没有的内容；不确定的一律标注，绝不脑补。

质量标准：清洗后的记录信息无损失，后续节点可据此准确提取议题、观点、决策。

常见失败情况：清洗时擅自归纳导致观点失真；脑补听不清的内容；把闲聊误删但其实含有效信息。
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下清洗后的记录是否存在以下问题：
1. 是否在清洗时擅自归纳或下了原文没有的结论？
2. 是否脑补了听不清的内容（应标注【存疑】而非编造）？
3. 是否丢失了某些看似闲聊但含有效信息的内容？
4. 发言人标注是否准确（不确定的是否标"发言人不明"）？

【粘贴清洗后的记录】

逐条说明问题。
```

##### D. 返修优化版

```
根据自查意见修正清洗记录。还原被擅自归纳的观点，将脑补内容改为【存疑】标注，补回误删的有效信息，修正发言人标注。

原始清洗记录：【粘贴原始记录】
自查意见：【粘贴自查结果】
```

#### 1.5 交付给下游节点

将清洗后的结构化记录复制，交给节点 2（议题识别专家）和节点 3（发言观点提取专家）。

#### 1.6 人工验收清单

- [ ] 是否保留了全部实质信息，没有擅自归纳？
- [ ] 听不清的部分是否标注【存疑】而非脑补？
- [ ] 发言人标注是否准确（不确定的标"发言人不明"）？

---

### 节点 2：议题识别专家

#### 2.1 节点定位

从会议记录中识别出讨论了哪些议题，为后续按议题组织纪要打下框架。会议常在多个议题间跳跃，先把议题理清，纪要才有结构。

#### 2.2 输入与输出

**输入**：节点 1 清洗后的记录 + 会议议程（如有）

**输出**：议题清单（议题名称 + 讨论范围 + 是否有结论）

#### 2.3 使用顺序

1. 先用「快速生成版」识别议题。
2. 议题交叉或会议无明确议程时，改用「专家增强版」。
3. 用「自查审稿版」检查议题是否完整、是否有遗漏或合并不当。
4. 有问题则用「返修优化版」修正。
5. 对照 2.6 验收清单确认，通过后交给节点 3。

#### 2.4 提示词包

##### A. 快速生成版

```
你是一位会议议题梳理员。请从以下记录识别讨论的议题。

会议记录：【粘贴节点1的记录】
会议议程（如有）：【填入或留空】

输出：
1. 议题清单（按讨论顺序，每个议题一句话概括）
2. 每个议题是否产生了结论（是/否/部分）
```

##### B. 专家增强版

```
你是一位会议议题分析专家。

任务：从会议记录中准确识别所有讨论议题。

输入：
- 会议记录：【粘贴节点1的记录】
- 会议议程：【填入或留空】

处理步骤：
1. 识别记录中讨论的每个独立议题
2. 对每个议题概括讨论范围（围绕什么展开）
3. 标注每个议题的讨论状态（已有结论 / 讨论未决 / 临时提及待后续）
4. 如有议程，对比实际讨论与议程的差异（议程外新增的、议程内未讨论的）

输出格式：
| 议题 | 讨论范围 | 讨论状态 | 是否在原议程 |

约束：议题划分基于实际讨论内容，不强行套议程；一个话题被反复讨论应合并为一个议题；不把闲聊当议题。

质量标准：议题清单完整覆盖会议实际讨论，颗粒度适中（不过碎也不过粗）。

常见失败情况：把同一议题的多次讨论拆成多个；把闲聊或寒暄当议题；遗漏了简短但重要的议题。
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下议题清单是否存在以下问题：
1. 是否有同一议题被拆成了多个（应合并）？
2. 是否把闲聊或寒暄当成了议题？
3. 是否遗漏了简短但重要的议题？
4. 讨论状态（已结论/未决/待后续）标注是否准确？

【粘贴议题清单】

逐条说明问题。
```

##### D. 返修优化版

```
根据自查意见修正议题清单。合并重复议题，剔除闲聊，补回遗漏议题，校正讨论状态。

原始议题清单：【粘贴原始清单】
自查意见：【粘贴自查结果】
```

#### 2.5 交付给下游节点

将议题清单复制，交给节点 3（发言观点提取专家）。观点将按议题归集。

#### 2.6 人工验收清单

- [ ] 议题是否完整覆盖会议实际讨论？
- [ ] 是否没有把闲聊当议题？
- [ ] 讨论状态标注是否准确？

---

### 节点 3：发言观点提取专家

#### 3.1 节点定位

提取各议题下参会人的关键观点，并标注发言人。观点是观点，不是决策——这一步只忠实记录"谁提出了什么看法"，不做判断。

#### 3.2 输入与输出

**输入**：节点 1 清洗记录 + 节点 2 议题清单

**输出**：各议题下的关键观点（观点 + 发言人 + 是否有分歧）

#### 3.3 使用顺序

1. 先用「快速生成版」提取观点。
2. 观点多或有对立时，改用「专家增强版」。
3. 用「自查审稿版」检查观点是否忠实、是否标错发言人。
4. 有问题则用「返修优化版」修正。
5. 对照 3.6 验收清单确认，通过后交给节点 4。

#### 3.4 提示词包

##### A. 快速生成版

```
你是一位会议观点记录员。请提取各议题下的关键观点。

会议记录：【粘贴节点1】
议题清单：【粘贴节点2】

输出（按议题组织）：
- 议题X
  - 观点：【内容】（发言人：XX）
  - 观点：【内容】（发言人：XX）
忠实记录观点，不评判、不归纳为结论。
```

##### B. 专家增强版

```
你是一位会议观点记录专家。

任务：按议题提取参会人的关键观点，忠实标注发言人。

输入：
- 会议记录：【粘贴节点1】
- 议题清单：【粘贴节点2】

处理步骤：
1. 在每个议题下，提取参会人的关键观点
2. 每条观点标注发言人（不确定的标"发言人不明"）
3. 标注观点之间的关系（一致 / 补充 / 分歧 / 对立）
4. 区分"个人观点"和"代表团队/部门的表态"
5. 不把观点升级为决策（决策由节点4处理）

输出格式（按议题）：
- 议题 X
  | 观点 | 发言人 | 与其他观点关系 | 性质（个人/团队表态）|

约束（关键）：忠实记录，不评判观点对错，不把观点写成结论；发言人不确定时标注而非臆测；保留对立观点，不只记录一方。

质量标准：观点记录全面、准确归属，分歧得到保留，不被误写为已达成的共识。

常见失败情况：把某人的观点写成"会议认为"；只记录主导者观点忽略反对意见；把观点提升为决策。
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下观点提取是否存在以下问题：
1. 是否把某个人的观点写成了"会议认为/大家认为"（个人观点≠集体结论）？
2. 是否只记录了主导者的观点，遗漏了反对或不同意见？
3. 发言人标注是否准确（不确定的是否标注）？
4. 是否把观点提前写成了决策？

【粘贴观点提取】

逐条说明问题。
```

##### D. 返修优化版

```
根据自查意见修正观点提取。还原被夸大为集体结论的个人观点，补回遗漏的不同意见，修正发言人标注，将误写的决策降回观点。

原始观点提取：【粘贴原始内容】
自查意见：【粘贴自查结果】
```

#### 3.5 交付给下游节点

将各议题观点复制，交给节点 4（决策事项提取专家）。决策判定基于这些观点和讨论。

#### 3.6 人工验收清单

- [ ] 是否没有把个人观点写成集体结论？
- [ ] 是否保留了对立和不同意见？
- [ ] 发言人标注是否准确？

---

### 节点 4：决策事项提取专家（招牌节点）

#### 4.1 节点定位

**这是本工作流的招牌节点。** 准确区分会议中"真正决定了的事"和"只是讨论过的事"。会议纪要最常见、也最危险的错误，就是把讨论当成决定。本节点必须把事项分入以下类别：

| 类别 | 判定标准 |
|------|---------|
| 已明确决定 | 有明确结论且无人反对（或决策人拍板） |
| 讨论未决 | 讨论过但没有形成结论 |
| 需进一步确认 | 倾向某方案但还需确认才能定 |
| 有责任人无截止时间 | 定了谁做，但没定什么时候 |
| 有截止时间无明确责任人 | 定了时间，但没明确谁负责 |

把"讨论过"误写成"已决定"，会导致没人真正执行却以为定了；反之会导致已定的事被当成还在讨论。

#### 4.2 输入与输出

**输入**：节点 3 各议题观点 + 节点 1 清洗记录

**输出**：分级的决策事项清单（事项 + 类别 + 依据 + 缺失信息）

#### 4.3 使用顺序

1. 先用「快速生成版」初步分类决策事项。
2. 会议决策模糊或表态含混时，必须使用「专家增强版」。
3. 用「自查审稿版」检查是否把讨论误判为决定。
4. 有问题则用「返修优化版」修正。
5. 对照 4.6 验收清单确认，通过后交给节点 5。

#### 4.4 提示词包

##### A. 快速生成版

```
你是一位会议决策识别员。请从会议讨论中识别并分类决策事项。

议题观点：【粘贴节点3】
会议记录：【粘贴节点1相关部分】

请把每个事项归入以下类别之一：
1. 已明确决定（有明确结论且无人反对）
2. 讨论未决（讨论过但无结论）
3. 需进一步确认（倾向某方案但需确认）
4. 有责任人无截止时间
5. 有截止时间无明确责任人

每个事项注明归类依据（记录中的哪句话支持这个判断）。
不要把"讨论过"写成"已决定"。
```

##### B. 专家增强版

```
你是一位资深会议决策分析专家，深知"讨论"和"决定"的区别。

任务：从会议讨论中准确识别决策事项，按确定程度严格分类。

输入：
- 议题观点：【粘贴节点3】
- 会议记录：【粘贴节点1】
- 会议性质（决策会/讨论会）：【填入或留空】

分类标准（逐项判定）：
1. 已明确决定：有明确结论，决策人拍板或全员无异议
2. 讨论未决：充分讨论但未形成结论
3. 需进一步确认：有倾向方案但需进一步确认/上级批准才能定
4. 有责任人无截止时间：明确了谁做，但未定时间
5. 有截止时间无明确责任人：定了时间节点，但未明确负责人

处理步骤：
1. 逐个事项判定类别
2. 为每个判定提供依据（引用记录中支持判断的原话）
3. 对"已决定"类，标注决策人和决策内容
4. 对信息不全的类别（4、5），明确标注缺失什么
5. 对模糊表态（"应该可以""回头再说"），归入"讨论未决"或"需确认"，不归入"已决定"

输出格式：
| 事项 | 类别 | 判定依据（原话引用）| 缺失信息 | 决策人（如有）|

约束（关键）：模糊表态不得归为"已决定"；每个判定必须有记录中的原话支撑；信息不全必须标注，不补全。

质量标准：决策分类准确，纪要读者能清楚知道哪些真定了、哪些还没定、哪些缺信息。

常见失败情况：把"我觉得可以"这类倾向性发言当成决定；把热烈讨论当成已达成一致；漏标缺失的责任人或时间。
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

【粘贴决策事项分类】

### 节点 5：待办任务拆解专家

#### 5.1 节点定位

将决策事项转化为具体、可执行的待办任务。决策是"要做什么"，待办是"具体怎么落实"——一个决策可能拆成多个待办。

#### 5.2 输入与输出

**输入**：节点 4 决策事项清单

**输出**：待办任务清单（任务 + 来源决策 + 初步责任人/时间线索）

#### 5.3 使用顺序

1. 先用「快速生成版」拆解待办。
2. 决策复杂或需拆多步时，改用「专家增强版」。
3. 用「自查审稿版」检查待办是否可执行、是否凭空新增。
4. 有问题则用「返修优化版」修正。
5. 对照 5.6 验收清单确认，通过后交给节点 6。

#### 5.4 提示词包

##### A. 快速生成版

```

决策事项：【粘贴节点4中"已决定"和"需确认"的事项】

输出：
| 待办任务 | 来源决策 | 记录中提到的责任人 | 记录中提到的时间 |
任务用动词开头，具体可执行。只拆解记录中实际涉及的任务，不新增。
```

##### B. 专家增强版

```

输入：
- 决策事项清单：【粘贴节点4】

处理步骤：
1. 对每个"已决定"事项，拆解为具体待办（一个决策可能含多个任务）
2. 待办用动词开头，明确"做什么、产出什么"
3. 标注每个待办来源于哪个决策
4. 从记录中提取已提到的责任人和时间线索（没提到则留空待节点6处理）
5. 对"需进一步确认"的事项，拆出"确认动作"作为待办

| 待办任务 | 来源决策 | 已知责任人 | 已知时间 | 备注 |

质量标准：待办具体可执行，可追溯到来源决策，未明确的责任人/时间如实留空。

常见失败情况：凭空补充会议没提到的任务；自作主张分配责任人；把"讨论未决"的事项也拆成待办。

```

【粘贴待办清单】

逐条说明问题。

原始待办清单：【粘贴原始清单】

从纪要中提炼出需要会后跟进的事项，生成可勾选的追踪清单。纪要是记录，追踪清单是行动——确保会议成果真正落地。

#### 9.2 输入与输出

**输入**：节点 8 会议纪要

**输出**：会后追踪清单（待办跟进 + 待确认事项 + 下次会议复查项）

#### 9.3 使用顺序

1. 先用「快速生成版」生成追踪清单。
2. 跟进项多或需分类管理时，改用「专家增强版」。
3. 用「自查审稿版」检查清单是否覆盖所有需跟进项。
4. 有问题则用「返修优化版」补充。
5. 对照 9.6 验收清单确认。

#### 9.4 提示词包

##### A. 快速生成版

```
你是一位会后跟进清单整理员。请从纪要提炼追踪清单。

会议纪要：【粘贴节点8】

输出可勾选清单：
- [ ] 待办跟进（任务 + 责任人 + 时间）
- [ ] 待确认事项（需补充责任人/时间/决策的）
- [ ] 下次会议需复查的事项
```

##### B. 专家增强版

```
你是一位会议跟进管理专家。

任务：从会议纪要提炼可执行的会后追踪清单。

输入：
- 会议纪要：【粘贴节点8】

处理步骤：
1. 提取所有待办，整理为可勾选跟进项（含责任人和时间）
2. 提取所有"待确认"事项，列为需补充清单（谁来确认、确认什么）
3. 提取"讨论未决"事项，列为下次会议复查项
4. 标注每项的优先级或时间紧迫度
5. 给出建议的首次跟进时间

输出格式：
- 待办跟进清单（可勾选，含责任人/截止时间）
- 待确认清单（事项/确认人/确认内容）
- 下次会议复查清单
- 建议首次跟进时间

约束：清单内容全部来自纪要，不新增；待确认和待复查项要清楚指出缺什么。

质量标准：照此清单可逐项跟进，确保会议成果落地、遗留问题不丢失。

常见失败情况：只列待办漏掉待确认项；跟进清单脱离纪要凭空添加；缺失项不指出需补充什么。
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下会后追踪清单是否存在以下问题：
1. 是否覆盖了纪要中所有待办、待确认、待复查事项？
2. 清单内容是否全部来自纪要（无凭空新增）？
3. 待确认项是否指出了需补充什么？
4. 是否可逐项勾选跟进？

【粘贴会后追踪清单】

逐条说明问题。
```

##### D. 返修优化版

```
根据自查意见完善追踪清单。补全遗漏的跟进项，删除凭空新增内容，明确待确认项的补充需求。

原始清单：【粘贴原始清单】
自查意见：【粘贴自查结果】
```

#### 9.6 人工验收清单

- [ ] 是否覆盖了所有待办、待确认、待复查事项？
- [ ] 清单内容是否全部来自纪要？
- [ ] 待确认项是否指出了需补充什么？

## 常见错误

**错误 1：把讨论当成决定**

表现：会议只是讨论了某方案的可能性，纪要里却写成"会议决定采用 X 方案"，导致没人真正去推进却以为已经定了。

修复：执行节点 4（招牌节点）的严格分类，"已明确决定"必须有原话支撑，模糊表态归入"讨论未决"或"需确认"。

**错误 2：待办没有责任人和时间却不标注**

表现：纪要列了一堆待办，但很多没写谁负责、什么时候完成，发出去后无人认领。

修复：执行节点 6，缺失的责任人和时间如实标注"待确认"，并在"需会后确认事项"中汇总，而非留空或自行指派。

**错误 3：纪要夹带会议没说的内容**

表现：AI 整理时"脑补"了一些看似合理但会议没讨论的结论或建议。

修复：每个节点都坚持"只整理实际发生的内容，不新增"，节点 8 自查时核对纪要内容是否都能在记录中找到来源。

## 人工验收清单

- [ ] 纪要是否清楚区分了"已决定"和"讨论未决"？
- [ ] 每个"已决定"事项是否真的在会议中拍板了？
- [ ] 待办的缺失责任人/时间是否标注"待确认"？
- [ ] 纪要是否没有夹带会议未实际产生的内容？
- [ ] 会后追踪清单是否覆盖所有需跟进事项？

## 延伸玩法

- **变体 1：快速纪要版**：对短会或同步会，只走节点 1（整理）→ 节点 4（决策提取）→ 节点 6（待办校准）→ 节点 8（生成），省略观点和风险节点。
- **变体 2：决策追溯版**：对重大决策会，强化节点 4 和节点 3，详细记录每个决策的讨论过程和各方观点，便于日后追溯决策依据。
- **进阶组合**：与"工作流 17（领导汇报工作流）"结合，将会议纪要的决策和成果作为汇报材料的输入；多次会议的追踪清单可汇总成项目进展看板。
