# 从文章笔记到个人知识库的 AI 多专家整理工作流

## 基本信息

- **工作流名称**：从文章笔记到个人知识库的 AI 多专家整理工作流
- **适用岗位**：知识工作者、学生、研究人员、内容创作者、职场人
- **真实场景**：一名产品经理收藏了 300 多篇文章、做了大量会议笔记，但每次需要用某个知识点时完全找不到，或者找到了也不记得当时看完的感悟。需要把这些碎片整理成可以真正复用的个人知识库。
- **最终目标**：将碎片化的文章、笔记、链接整理为结构化的个人知识库，每条知识不只是摘要，而是包含概念、解释、例子、适用场景、与已有知识的关系和可行动建议的完整卡片。

---

## 输入材料清单

1. 待整理的原始材料（文章链接、笔记文本、截图文字、会议记录等）
2. 使用的笔记工具（Notion / Obsidian / 飞书 / 纸质等）
3. 知识库的用途（工作参考 / 备考 / 创作素材 / 个人成长等）
4. 现有标签或分类体系（如已有，提供现有结构）
5. 整理频率预期（每天/每周/每月维护）

---

## 工作流总览

```
[原始材料（文章/笔记/链接）]
         ↓
  专家1：材料清洗
         ↓
  专家2：主题识别
         ↓
  专家3：知识卡片生成 ★招牌节点
         ↓
  专家4：标签体系
         ↓
  专家5：关联关系
         ↓
  专家6：行动清单
         ↓
  专家7：复习提醒
         ↓
  专家8：知识库结构
         ↓
  专家9：更新维护
         ↓
[知识卡片库 + 标签体系 + 关联图 + 行动清单]
```

## 专家节点详解

### 专家 1：材料清洗专家

#### 1.1 节点定位

过滤掉无价值的收藏内容，保留真正有用的材料，避免"把垃圾整理得更整齐"。

#### 1.2 输入与输出

- **输入**：原始材料列表（文章/笔记/链接）
- **输出**：筛选后的材料清单（保留 / 删除 / 存档三类）

#### 1.3 使用顺序

第一步，整理前的"瘦身"。跳过这步会导致后续节点处理大量无用内容。

#### 1.4 提示词包

**A. 快速生成版**

```
我有以下[X]份材料要整理成知识库，用途是[工作/备考/创作]。
请帮我判断每份材料是"保留整理"、"直接删除"还是"归档备用"：
[粘贴材料标题或摘要列表]
```

**B. 专家增强版**

```
【角色】你是一名知识管理顾问，擅长信息筛选和价值判断。

【任务】对以下材料做清洗分类。

【输入材料】
- 材料列表：[逐条列出标题/摘要/来源]
- 知识库用途：[工作参考/备考/创作等]

【处理步骤】
1. 对每条材料，判断它属于哪类：
   - **保留整理**：与用途直接相关，有具体知识点可提取
   - **归档备用**：有一定参考价值但目前用不到，不值得花时间整理
   - **直接删除**：时效性已过/与用途无关/只是当时一时兴起收藏/重复内容
2. 给出每条判断的理由（1句话）
3. 统计三类比例，如果"保留整理"超过80%，提示用户标准可能太宽松

【输出格式】
| 序号 | 标题/摘要 | 分类 | 理由 |
|------|---------|------|------|

【约束条件】不根据材料的"看起来很有价值"做判断，而是根据与实际用途的相关性

【质量标准】"保留整理"的比例通常不超过60%，否则说明清洗标准太宽松

【常见失败情况】因为舍不得删而把所有内容都分类为"保留"；没有说明判断理由
```

**C. 自查审稿版**

> 说明：本版本是当前节点的内部审查模式，不是新专家角色。

```
检查材料清洗结果：
1. "保留整理"比例是否超过了60%？如超过，是否说明了原因？
2. 每条判断是否给出了具体理由？
3. 是否存在只因为"收藏了很久"就保留的材料？
[粘贴清洗结果]
```

**D. 返修优化版**

> 说明：本版本是当前节点的内部优化模式，不是新专家角色。

```
根据审查意见重新清洗：
[审查意见] [原清洗结果]
修正：对"保留整理"中与用途关联度低的内容改为"归档备用"，补充理由。
```

#### 1.5 交付给下游节点

将"保留整理"清单交给**主题识别专家（节点2）**。

#### 1.6 人工验收清单

- [ ] 每条材料是否都有分类判断结果和理由？
- [ ] "保留整理"比例是否合理（通常不超过60%）？
- [ ] 是否避免了"舍不得删所以全部保留"？

### 专家 2：主题识别专家

#### 2.1 节点定位

识别筛选后材料覆盖的主题领域，为建立标签体系和知识卡片分类提供基础。

#### 2.2 输入与输出

- **输入**：清洗后的材料清单（节点1）
- **输出**：主题列表（3-8个主题，含每个主题下的材料数量）

#### 2.3 使用顺序

第二步，在材料清洗后进行，为知识卡片生成（节点3）提供主题框架。

#### 2.4 提示词包

**A. 快速生成版**

```
以下是我整理的材料清单，请识别出3-8个核心主题，并说明每个主题下有哪些材料：
[粘贴材料清单]
```

**B. 专家增强版**

```
【角色】你是一名信息架构师，擅长从混乱的内容中识别结构。

【任务】从以下材料中识别核心主题。

【输入材料】
- 材料清单：[节点1的"保留整理"列表]
- 知识库用途：[用途]

【处理步骤】
1. 阅读每条材料的标题/摘要，识别它的核心议题
2. 将相关材料归为同一主题（主题粒度：不能太宽"工作相关"，不能太窄"2024年3月的项目"）
3. 给每个主题命名（名词短语，如"产品决策框架"、"用户研究方法"）
4. 识别主题之间的层级关系（哪些是上位主题，哪些是下位主题）
5. 标注哪些主题材料充足（≥5条），哪些材料稀少（≤2条）

【输出格式】
主题层级结构（上位主题 → 下位主题）+ 每主题材料数量 + 材料稀少的主题标注

【约束条件】主题总数3-8个（太少=粒度太粗，太多=碎片化）；主题名称用名词短语，不用动词

【质量标准】同一主题下的材料有真实的内在关联，不只是"都是工作相关"

【常见失败情况】主题层级太深（超过2层）；主题命名用"其他/杂项"
```

**C. 自查审稿版**

> 说明：本版本是当前节点的内部审查模式，不是新专家角色。

```
检查主题识别结果：
1. 主题总数是否在3-8个范围内？
2. 主题命名是否是名词短语，而不是"其他"？
3. 同一主题下的材料是否有真实关联？
[粘贴主题列表]
```

**D. 返修优化版**

> 说明：本版本是当前节点的内部优化模式，不是新专家角色。

```
根据审查修正主题结构：
[审查意见] [原主题列表]
修正：合并粒度过细的主题，拆分"其他"类，完善命名。
```

#### 2.5 交付给下游节点

将主题结构交给**知识卡片生成专家（节点3）**和**标签体系专家（节点4）**。

#### 2.6 人工验收清单

- [ ] 主题总数是否在 3-8 个范围内？
- [ ] 主题命名是否都是名词短语？
- [ ] 是否标注了材料稀少的主题？

### 专家 3：知识卡片生成专家 ★招牌节点

#### 3.1 节点定位

将每篇文章或笔记提炼为结构化的知识卡片。**不是做摘要**，而是将知识转化为六维完整卡片：概念、解释、例子、适用场景、与已有知识的关系、可行动建议。

#### 3.2 输入与输出

- **输入**：单篇文章/笔记文本 + 所属主题（节点2）
- **输出**：每篇对应的知识卡片（每张卡片包含六个维度）

#### 3.3 使用顺序

第三步，对每篇"保留整理"材料逐一处理。是工作量最大的节点，也是知识库质量的核心。

#### 3.4 提示词包

**A. 快速生成版**

```
请将以下文章/笔记提炼为知识卡片，必须包含6个部分：
1. 概念（是什么）
2. 解释（为什么/原理）
3. 例子（具体案例）
4. 适用场景（什么情况下用）
5. 与已有知识的关系（联系哪些已知概念）
6. 可行动建议（我现在能做什么）

文章内容：
[粘贴文章/笔记]
```

**B. 专家增强版**

```
【角色】你是一名知识提炼专家，专门将文章和笔记转化为高质量的知识卡片。你深知"摘要≠知识卡片"的区别。

【任务】将以下材料提炼为六维知识卡片。一篇文章可能包含1-3个核心知识点，每个知识点生成一张独立的卡片。

【输入材料】
- 材料文本：[粘贴原文/笔记]
- 所属主题：[节点2识别的主题]
- 知识库用途：[工作参考/备考/创作]

【处理步骤】
1. 识别材料中有独立价值的知识点（1篇文章通常1-3个，不强求超过3个）
2. 对每个知识点，依次填写六个维度：

   **维度1：概念（是什么）**
   - 用一句话定义这个知识点
   - 标准：读完这一句能准确解释给别人听
   - 避免：不能用"一种方法/一个概念"这类空洞表述

   **维度2：解释（为什么/原理）**
   - 说明这个知识点的底层逻辑或运作原理
   - 标准：回答"为什么它是这样的"或"它是怎么运作的"
   - 避免：不能只是复述维度1的内容

   **维度3：例子（具体案例）**
   - 提供一个真实的或贴近真实的具体例子
   - 标准：例子要能"落地"，不能是抽象举例（"比如某公司这样做了"不算具体）
   - 避免：不编造例子；如原文有例子，直接引用；如没有，标注"需补充案例"

   **维度4：适用场景（在哪里用）**
   - 说明什么情况下、什么岗位/任务下会用到这个知识
   - 标准：给出2-3个具体场景描述（不是"很多情况下都适用"）
   - 避免：过于宽泛的适用范围

   **维度5：与已有知识的关系（新旧连接）**
   - 说明这个知识点与哪些已知概念相关、有何区别或延伸
   - 标准：至少连接1个已有概念，说明是"同类/相反/上位/下位/互补"关系
   - 避免：如果完全不知道相关知识，标注"待建立连接"，不要编造

   **维度6：可行动建议（我现在能做什么）**
   - 基于这个知识点，给出1-2条今天或本周就能执行的具体行动
   - 标准：行动必须具体到"做什么"，不能是"深入思考/多加练习"
   - 避免：不能给出超出用户实际能力范围的行动建议

3. 给这张卡片起一个标题（5-10个字，点明核心知识点）

【输出格式】
---
卡片标题：[5-10字]
所属主题：[节点2主题]
来源：[文章/笔记标题]

【概念】
[一句话定义]

【解释】
[底层逻辑/原理，3-5句]

【例子】
[具体案例，如原文无则标注"需补充案例"]

【适用场景】
· 场景1：[……]
· 场景2：[……]

【与已有知识的关系】
· 与[知识点A]的关系：[同类/相反/上位/下位/互补]，具体说明：[……]
· 待建立连接：[如有不确定的关联]

【可行动建议】
· 立即可做：[具体行动，今天就能做]
· 本周可做：[稍大的行动]
---

【约束条件】
- 维度3（例子）如原文无具体案例，必须标注"需补充案例"，不自行编造
- 维度6（可行动）不能是"多了解/多思考/多练习"等无法执行的建议
- 一篇文章最多提炼3张卡片，不强行拆分

【质量标准】
- 六个维度全部填写（即使部分需要标注"需补充"）
- 卡片标题能让人在不看内容的情况下猜出大概是什么知识
- 可行动建议在24小时内能开始执行

【常见失败情况】
- 把文章摘要当成"概念"维度填进去
- 维度2和维度1高度重复
- 维度6写了"保持学习习惯/持续积累"等废话
- 一篇2000字文章提炼出7张卡片（粒度过细）
```

**C. 自查审稿版**

> 说明：本版本是当前节点的内部审查模式，不是新专家角色。

```
对以下知识卡片做审查：
1. 六个维度是否全部填写（即使部分标注"需补充"）？
2. 维度3（例子）是否是具体案例，而不是编造的或过于抽象的？
3. 维度6（可行动）是否具体到"做什么"，而不是"多思考"等废话？
4. 维度1（概念）和维度2（解释）是否是不同视角，而不是重复？
5. 卡片标题能否点明核心知识？
[粘贴知识卡片]
```

**D. 返修优化版**

> 说明：本版本是当前节点的内部优化模式，不是新专家角色。

```
根据审查意见修正知识卡片：
[审查意见] [原卡片]
修正：将摘要改写为定义，补充缺失维度，将空洞行动改为具体操作，区分概念和解释。
```

#### 3.5 交付给下游节点

将生成的知识卡片集交给**标签体系专家（节点4）**和**关联关系专家（节点5）**。

#### 3.6 人工验收清单

- [ ] 每张卡片是否包含全部六个维度？
- [ ] 维度3（例子）是否是具体案例，未编造？
- [ ] 维度6（可行动）是否具体到 24 小时内可执行？
- [ ] 维度1（概念）和维度2（解释）是否不重复？
- [ ] 单篇文章是否最多提炼 3 张卡片（未过度拆分）？

### 专家 4：标签体系专家

#### 4.1 节点定位

为知识卡片建立可复用的标签体系，避免"每张卡片标签都不一样导致无法检索"的问题。

#### 4.2 输入与输出

- **输入**：知识卡片集（节点3）、主题结构（节点2）
- **输出**：标签体系文档（含标签定义和使用规则）+ 每张卡片的推荐标签

#### 4.3 使用顺序

第四步，在知识卡片生成后进行。

#### 1.4 提示词包

**A. 快速生成版**

```
我有以下知识卡片，请帮我建立一套可复用的标签体系（不超过20个标签），并为每张卡片推荐2-4个标签：
[粘贴卡片标题列表]
```

**B. 专家增强版**

```
【角色】你是一名信息管理专家，擅长设计可持续维护的标签体系。

【任务】为以下知识卡片库建立标签体系。

【输入材料】
- 知识卡片标题列表：[列表]
- 主题结构：[节点2输出]

【处理步骤】
1. 设计三层标签：
   - **领域标签**（2-5个）：知识所属的大领域，如"产品设计"、"心理学"
   - **类型标签**（3-8个）：知识的形式，如"方法论"、"工具"、"概念定义"、"案例"
   - **行动标签**（2-5个）：与知识库用途相关，如"可直接应用"、"需深入学习"、"待验证"
2. 为每个标签写1行定义（什么内容该打这个标签）
3. 给每张卡片推荐标签（每张2-4个，覆盖三层各至少1个）

【输出格式】
标签体系文档（三层+定义）+ 卡片-标签对应表

【约束条件】标签总数不超过20个；不创建只有1张卡片适用的标签（太细）；不创建超过80%卡片都适用的标签（太泛）

【质量标准】标签体系建立后，用户能通过标签组合找到特定场景所需知识

【常见失败情况】标签数量超过20个（后期维护成本太高）；标签定义缺失导致后续乱打标签
```

**C. 自查审稿版**

> 说明：本版本是当前节点的内部审查模式，不是新专家角色。

```
检查标签体系：
1. 标签总数是否不超过20个？
2. 每个标签是否有定义？
3. 是否覆盖了三层（领域/类型/行动）？
4. 是否存在只有1张卡片适用的过细标签？
[粘贴标签体系]
```

**D. 返修优化版**

> 说明：本版本是当前节点的内部优化模式，不是新专家角色。

```
根据审查修正标签体系：
[审查意见] [原标签体系]
修正：合并细碎标签，补充标签定义，确保三层覆盖完整。
```

```
每周（周日）：复习高频卡片，更新行动清单，检查新增卡片标签（约30分钟）
每季度：健康检查，删除低频无用卡片，更新主题结构（约1小时）
放弃信号：连续3周未维护 → 立即简化规则
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 常见错误

| 错误表现 | 修正方式 |
|---------|---------|
| 知识卡片只有摘要，缺少解释/例子/行动建议 | 强制填写六个维度，缺少例子时标注"需补充案例"而不是跳过 |
| 标签过多（50+个标签） | 限制总标签不超过20个，合并细碎标签 |
| 整理完后从不复习 | 建立复习计划，设置"每周30分钟"的固定维护时间 |
| 所有卡片都没有关联关系 | 完成节点5的关联关系分析，至少为★★★重要卡片建立2-3个关联 |
| 知识库越建越大，但实际从不使用 | 定期执行节点9的健康检查，删除3个月未被查看过的低频卡片 |
| 一篇文章生成10张卡片（过度拆分） | 限制每篇文章最多3张卡片；合并粒度过细的卡片 |

## 人工验收清单

- [ ] 每张知识卡片是否包含全部六个维度？
- [ ] 维度3（例子）是否是具体案例，未编造？
- [ ] 维度6（可行动）是否具体到 24 小时内可执行？
- [ ] 标签总数是否不超过 20 个？
- [ ] 每个标签是否有定义说明？
- [ ] 是否识别了知识孤岛？
- [ ] 行动清单是否已去重？
- [ ] 复习计划总时间是否在可用维护时间内？
- [ ] 文件夹结构是否不超过 3 层？
- [ ] 新增内容录入流程是否不超过 5 步？
- [ ] 维护规则是否包含"放弃信号"条款？
- [ ] 单篇文章是否最多生成 3 张卡片？

## 延伸玩法

1. **双向链接知识网络**：在 Obsidian 中用 `[[卡片标题]]` 建立双向链接，用图谱视图可视化整个知识网络，发现意料之外的知识关联。

2. **工作场景触发**：将维度4（适用场景）与实际工作项目对应，当开始一个新项目时，先用场景标签搜索知识库，看有哪些相关知识可以复用。

3. **定期知识输出**：每月基于知识库写一篇3-5分钟可读的总结（一个主题下的核心洞见），既是知识输出也是质量检验——能清楚表达的知识才是真正掌握的知识。

4. **团队知识共享**：将知识库中带"可直接应用"行动标签的卡片，定期提炼为团队内部的"方法论速查手册"，把个人知识库变为团队资产。
