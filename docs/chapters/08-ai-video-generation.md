# 工作流 08：从静态画面到可生成视频片段的 AI 多专家工作流

**适用岗位**：短视频创作者、AI 视频创作者、导演、分镜师、宣传人员

**真实场景**：你已经有了一批静态图像或分镜设计，想用 Kling、即梦、Sora 等 AI 视频工具将其生成为动态视频片段，但经常遇到肢体变形、画面崩坏、镜头抖动无法控制、生成结果与预期差距大等问题。

**最终目标**：一套经过可行性评估、优化的 AI 视频生成提示词组，每个片段都有明确的动作设计、镜头运动方案和物理运动审查结果，生成成功率高，失败时有明确的修复方向。

---

## 输入材料清单

- 静态图像或图像描述（一张或多张，说明画面内容）
- 目标视频工具（Kling / 即梦 / Sora / Seedance / Runway）
- 每个片段的目标时长（2秒 / 3秒 / 4秒 / 5秒）
- 视频整体情绪方向（平静/激烈/舒缓/震撼）
- 是否有系列连贯要求（单片段 / 需要多片段接续）

---

## 工作流总览

```
节点 1：画面输入检查专家
  ↓ 输出：画面可行性评估报告 + 简化建议
节点 2：动作设计专家
  ↓ 输出：每个片段的动作方案（主体动作 + 速度 + 幅度）
节点 3：镜头运动专家
  ↓ 输出：镜头运动方案（运动类型 + 方向 + 速度）
节点 4：物理运动审查专家（招牌节点）
  ↓ 输出：物理运动审查报告 + 修改建议
节点 5：视频提示词生成专家
  ↓ 输出：每个片段的完整视频提示词（英文）
节点 6：时长与节奏专家
  ↓ 输出：时长分配方案 + 片段衔接说明
节点 7：失败画面修复专家
  ↓ 输出：失败原因诊断 + 修复后提示词
节点 8：成片一致性审查专家
  ↓ 输出：多片段一致性审查报告
最终交付：可用 AI 视频生成提示词组 + 使用说明
```

---

## 专家节点详解

**C 自查审稿版和 D 返修优化版是当前节点内部的工作模式，不是新的专家角色。**

---

### 节点 1：画面输入检查专家

#### 1.1 节点定位

在正式设计动作之前，检查输入的静态画面是否适合 AI 视频生成，识别哪些画面元素会导致生成失败或质量低下。提前发现问题比事后修复高效得多——某些画面结构天生不适合 AI 视频，应该在这一步调整或简化。

#### 1.2 输入与输出

**输入**：静态图像描述 + 目标视频工具

**输出**：画面可行性评估报告（每个画面：可行/需调整/不建议，附原因）+ 简化建议

#### 1.3 使用顺序

1. 先用「快速生成版」得到可行性评估初稿。
2. 有复杂场景或多人物时，改用「专家增强版」。
3. 用「自查审稿版」确认评估是否遗漏了关键问题。
4. 有问题则用「返修优化版」补充。
5. 对照 1.6 验收清单确认，通过后交给节点 2。

#### 1.4 提示词包

##### A. 快速生成版

```
你是一位 AI 视频生成可行性评估师。请评估以下静态画面是否适合用 AI 视频工具生成为视频片段。

画面描述：【填入画面内容】
目标工具：【填入工具名称】

对每个画面输出：
1. 可行性评级（可行 / 需调整 / 不建议）
2. 主要风险（哪些元素容易导致生成失败）
3. 简化建议（如何调整画面设计降低生成难度）
```

##### B. 专家增强版

```
你是一位资深 AI 视频导演，深度使用过 Kling/即梦/Sora/Runway，了解各工具的能力边界和失败模式。

任务：对以下静态画面进行系统性可行性评估，识别每个元素在视频生成时的风险等级。

输入：
- 画面描述：【填入画面内容】
- 目标工具：【填入工具名称】
- 目标时长：【填入每段秒数】

评估维度：
1. 主体复杂度（人物数量、动作复杂度、是否有细节肢体如手指）
2. 场景复杂度（背景元素数量、是否有反射/透明/液体等难生成元素）
3. 动作难度（是否有快速运动、物理交互、变形动作）
4. 镜头运动适配（画面是否支持预期的镜头运动，空间是否足够）
5. 工具能力匹配（该工具对此类内容的实际成功率）

风险等级：高风险（经常失败）/ 中风险（偶尔失败）/ 低风险（成功率高）

约束：评估必须基于实际工具能力，不能给出"AI 已经很强"等含糊评价。

输出：逐元素风险评估表 + 画面简化建议 + 工具适配建议
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下可行性评估是否存在以下问题：
1. 是否有高风险元素被标记为"可行"而没有给出简化建议？
2. 简化建议是否具体可执行（不能只说"简化画面"）？
3. 是否考虑了目标工具的实际能力限制（不能假设工具"万能"）？

【粘贴可行性评估报告】

逐条说明问题。
```

##### D. 返修优化版

```
根据自查意见补充可行性评估。为高风险元素补充具体简化建议，修正过于乐观的可行性评级，补充工具能力限制说明。

原始评估：【粘贴原始评估报告】
自查意见：【粘贴自查结果】
目标工具：【填入工具名称】
```

#### 1.5 交付给下游节点

将可行性评估报告和简化建议完整复制，交给节点 2（动作设计专家）。动作设计必须在可行性评估确认的范围内进行。

#### 1.6 人工验收清单

- [ ] 是否为每个画面都给出了明确的可行性评级？
- [ ] 高风险元素是否都附有具体的简化建议？
- [ ] 评估是否考虑了目标工具的实际能力？

---

### 节点 2：动作设计专家

#### 2.1 节点定位

在可行性确认的基础上，为每个视频片段设计主体的动作方案，包括动作类型、速度和幅度。动作设计必须在 AI 视频工具能力范围内——过于复杂的动作设计在节点 4 会被审查退回。

#### 2.2 输入与输出

**输入**：节点 1 的可行性评估报告（了解动作风险边界）+ 视频情绪方向

**输出**：每个片段的动作方案（动作类型 + 速度 + 幅度 + 起止状态）

#### 2.3 使用顺序

1. 先用「快速生成版」得到动作方案初稿。
2. 情绪或叙事要求复杂时，改用「专家增强版」。
3. 用「自查审稿版」检查动作是否在可行性范围内。
4. 有问题则用「返修优化版」简化。
5. 对照 2.6 验收清单确认，通过后交给节点 3 和节点 4。

#### 2.4 提示词包

##### A. 快速生成版

```
你是一位 AI 视频动作设计师。请为以下视频片段设计动作方案。

画面描述：【填入画面内容】
可行性边界：【粘贴节点1中该画面的风险评估摘要】
情绪方向：【填入情绪关键词】
片段时长：【填入秒数】

输出：
1. 主体动作描述（从动作开始到结束的完整过程）
2. 动作速度（慢速/中速/快速）
3. 动作幅度（微小/中等/大幅）
4. 起止状态（片段开始时的状态 → 片段结束时的状态）
```

##### B. 专家增强版

```
你是一位专注于 AI 视频内容的动作导演，深知哪些动作在 AI 视频工具中成功率高，哪些容易失败。

任务：为以下视频片段设计在 AI 工具能力范围内的动作方案，在叙事表达和生成成功率之间找到最佳平衡。

输入：
- 画面描述：【填入画面】
- 可行性评估结果：【粘贴节点1的评估报告】
- 情绪方向：【填入情绪关键词】
- 片段时长：【填入秒数】
- 目标工具：【填入工具名称】

动作设计原则：
- 优先选择单一方向的线性运动（前进/转身/低头），而非多轴复合动作
- 慢速动作比快速动作成功率更高
- 微表情和细微肢体动作比大幅运动更稳定
- 如果画面有多个主体，优先设计主体 A 动，其余主体保持静止或微动

起止状态描述格式：
[开始状态：姿势 + 朝向 + 位置] → [结束状态：姿势 + 朝向 + 位置]

约束：动作描述必须是可以被 AI 理解的英文指令，不能使用比喻性描述。

输出：动作方案（含原因说明）+ 备选简化方案（如果首选方案风险较高）
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下动作方案是否存在以下问题：
1. 是否有复合多轴动作（同时涉及转身+移动+表情变化）超出 AI 工具能力？
2. 动作速度是否与片段时长匹配（5秒内完成大幅快速动作往往失败）？
3. 起止状态描述是否具体（"站着"不具体，"standing upright, facing camera, arms at sides"具体）？
4. 是否有多主体同时进行复杂动作的设计（高风险）？

【粘贴动作方案内容】

逐条说明问题。
```

##### D. 返修优化版

```
根据自查意见简化动作方案。将复合动作拆分为单一方向运动，降低动作速度，补充具体的起止状态描述，减少同时运动的主体数量。

原始方案：【粘贴原始动作方案】
自查意见：【粘贴自查结果】
可行性边界：【粘贴节点1的风险评估摘要】
```

#### 2.5 交付给下游节点

将动作方案（含起止状态）完整复制，同时交给：
- **节点 3（镜头运动专家）**：根据动作设计配套镜头运动。
- **节点 4（物理运动审查专家）**：审查动作的物理合理性和生成可行性。

#### 2.6 人工验收清单

- [ ] 每个动作是否都是单一方向的线性运动（没有复合多轴动作）？
- [ ] 起止状态是否描述精确（姿势+朝向+位置）？
- [ ] 动作复杂度是否在节点 1 评估的可行性范围内？

---

### 节点 3：镜头运动专家

#### 3.1 节点定位

为每个视频片段设计镜头运动方案，包括运动类型、方向和速度。镜头运动不是越复杂越好——AI 视频工具对镜头运动有明确的能力边界，错误的镜头设计会导致画面抖动或出现奇怪的空间扭曲。

#### 3.2 输入与输出

**输入**：节点 2 的动作方案 + 节点 1 的可行性评估

**输出**：每个片段的镜头运动方案（运动类型 + 方向 + 速度 + 运动幅度）

#### 3.3 使用顺序

1. 先用「快速生成版」得到镜头运动方案初稿。
2. 有特殊电影化镜头需求时，改用「专家增强版」。
3. 用「自查审稿版」检查镜头运动是否与主体动作配合、是否超出工具能力。
4. 有问题则用「返修优化版」简化。
5. 对照 3.6 验收清单确认，通过后交给节点 4。

#### 3.4 提示词包

##### A. 快速生成版

```
你是一位 AI 视频镜头设计师。请为以下视频片段设计镜头运动方案。

主体动作方案：【粘贴节点2的动作描述】
可行性评估：【粘贴节点1中该片段的风险评级】
目标工具：【填入工具名称】

输出：
1. 镜头运动类型（固定/推/拉/横移/跟拍/旋转/手持抖动）
2. 运动方向（向前/向后/向左/向右/向上/向下）
3. 运动速度（极慢/慢/中/快）
4. 对应英文描述词（直接用于提示词）
```

##### B. 专家增强版

```
你是一位电影级短视频镜头设计师，熟悉 AI 视频工具的镜头运动成功率规律。

任务：为以下视频片段设计镜头运动方案，确保镜头运动强化情绪，且在目标工具能力范围内。

输入：
- 主体动作方案：【粘贴节点2的完整动作描述】
- 片段情绪方向：【填入情绪关键词】
- 目标工具：【填入工具名称】

镜头运动选择原则（按成功率排序）：
- 高成功率：固定镜头（static）、缓慢推进（slow push in）、缓慢拉出（slow pull back）
- 中成功率：横向平移（pan left/right）、向上倾斜（tilt up/down）
- 低成功率：快速运动、旋转镜头（orbit）、手持晃动

主体动作与镜头运动的协调原则：
- 主体向前运动时，镜头可以跟随推进或保持固定
- 主体保持静止时，镜头运动更自由（推/拉/移均可）
- 主体有大幅运动时，镜头应保持固定或极慢运动（防止画面混乱）

约束：不能同时设计主体大幅运动 + 镜头快速运动（双重运动超出工具处理能力）。

输出：镜头运动方案 + 对应英文描述词 + 协调说明（为什么这样配合主体动作）
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下镜头运动方案是否存在以下问题：
1. 是否有主体大幅运动 + 镜头快速运动的双重运动设计（高失败率）？
2. 低成功率的旋转或手持晃动镜头是否有必要（是否有更稳定的替代方案）？
3. 镜头运动方向是否与主体动作方向冲突（主体向左但镜头也向左，会挤出画面）？

【粘贴镜头运动方案内容】

逐条说明问题。
```

##### D. 返修优化版

```
根据自查意见修改镜头运动方案。将双重高速运动替换为"主体运动+镜头固定"或"主体静止+镜头缓慢运动"，消除运动方向冲突。

原始方案：【粘贴原始镜头运动方案】
自查意见：【粘贴自查结果】
主体动作方案：【粘贴节点2的动作描述（确保镜头与动作配合）】
```

#### 3.5 交付给下游节点

将镜头运动方案（含英文描述词）复制，交给节点 4（物理运动审查专家）和节点 5（视频提示词生成专家）。

#### 3.6 人工验收清单

- [ ] 镜头运动是否避免了与主体大幅动作的双重高速运动叠加？
- [ ] 是否优先选择了高成功率的镜头运动类型？
- [ ] 镜头运动方向是否不与主体运动方向产生空间冲突？

---

### 节点 4：物理运动审查专家（招牌节点）

#### 4.1 节点定位

**这是本工作流的招牌节点。** 在生成视频提示词之前，系统性评估动作设计和镜头运动中存在的物理问题和 AI 生成限制。AI 视频工具遵循近似的物理规律，但有明确的失败模式——这个节点的职责是识别"生成后大概率会出现什么问题"，并在提示词阶段就修正，而不是等到生成失败后再返工。

**重点检查七类问题**：
1. 动作是否过于复杂（超出工具处理能力）
2. 镜头是否过度运动（导致画面扭曲或失稳）
3. 人物肢体是否容易变形（手指、四肢、面部的高风险姿势）
4. 是否有多人交互导致画面崩坏
5. 是否有不合理的物理运动（违反重力、穿插物体）
6. 是否适合拆成多个短镜头生成
7. 当前工具是否适合处理该类运动

#### 4.2 输入与输出

**输入**：节点 2 动作方案 + 节点 3 镜头运动方案 + 节点 1 可行性评估

**输出**：物理运动审查报告（七类问题逐一评估）+ 修改建议（通过/需修改/建议拆分）

#### 4.3 使用顺序

1. 先用「快速生成版」快速识别明显的物理问题。
2. 有人物交互或复杂场景时，必须使用「专家增强版」。
3. 用「自查审稿版」确认审查是否遗漏了潜在问题。
4. 根据审查结论：通过则进入节点 5；需修改则返回节点 2 或节点 3；需拆分则在本节点完成拆分方案后进入节点 5。
5. 对照 4.6 验收清单确认。

#### 4.4 提示词包

##### A. 快速生成版

```
你是一位 AI 视频物理运动审查员。请检查以下动作和镜头运动设计是否存在生成失败风险。

动作方案：【粘贴节点2的动作描述】
镜头运动方案：【粘贴节点3的镜头描述】
目标工具：【填入工具名称】
片段时长：【填入秒数】

请逐一检查以下7类问题：
1. 动作复杂度是否超出工具能力？
2. 镜头运动是否过度？
3. 人物肢体（手指/四肢/面部）是否有高变形风险的姿势？
4. 是否有多人同时交互的高风险场景？
5. 是否有违反物理规律的运动（飘浮/穿插/违反重力）？
6. 是否适合拆分为多个更短的片段？
```

- [ ] 预防措施是否可应用到其他类似片段？

### 节点 8：成片一致性审查专家

#### 8.1 节点定位

**这是一个跨内容的全局审查节点**，检查所有视频片段在视觉风格、角色外貌、场景元素和情绪节奏上的一致性。单个片段可能各自没问题，但拼接成完整视频时，可能出现风格漂移、场景跳跃、角色变脸等问题。

#### 8.2 输入与输出

**输入**：所有片段的视频提示词（最终版）+ 节点 6 的衔接方案

**输出**：多片段一致性审查报告（问题清单 + 通过/需修改结论）

#### 8.3 使用顺序

1. 先用「快速生成版」做快速一致性扫描。
2. 系列片段较多时，改用「专家增强版」做系统审查。
3. 用「自查审稿版」确认审查是否遗漏跨片段问题。
4. 通过后整合最终提示词组；有问题则返回对应节点修改。
5. 对照 8.6 验收清单确认。

#### 8.4 提示词包

##### A. 快速生成版

```
请对以下多个视频片段的提示词进行一致性检查。

所有片段提示词（含序号）：【粘贴节点5最终提示词列表】
衔接方案：【粘贴节点6的衔接说明】

检查项：
1. 风格标签是否在所有片段中保持一致？
2. 如有角色，角色描述词是否在所有片段完全相同？
3. 光线和时间设定是否在连续片段中保持合理连续（不能上一片段是白天，下一片段突然是夜晚）？
4. 是否有任何片段的时长不在 2-5 秒范围内？

输出：问题清单 + 通过/需修改结论
```

##### B. 专家增强版

```
你是一位多片段 AI 视频成品质量审查专家。

任务：对以下所有视频片段提示词进行系统性一致性审查，确保拼接后形成连贯的完整视频。

输入：
- 所有片段提示词：【粘贴节点5的完整列表（含时长）】
- 衔接方案：【粘贴节点6的完整衔接说明】

1. 视觉风格一致性（所有片段的风格标签和画风词是否相同）
2. 角色一致性（有角色的片段，描述词是否逐字相同）
3. 场景逻辑一致性（光线、时间、地点的连续性是否合理）
4. 运动连贯性（相邻片段的主体起止状态是否能自然衔接）
5. 时长节奏一致性（时长分配是否符合节点6的节奏设计）
6. 技术参数一致性（目标工具、时长格式等是否统一）

评分（每维度满分20分，合格线80分）

输出：六维度审查表 + 总分 + 具体问题列表 + 通过/需修改结论
```

## 常见错误

**错误 1：跳过物理运动审查直接生成**

表现：第一轮生成后发现大量肢体变形、镜头抖动等问题，需要大量返工。

修复：节点 4 是本工作流最关键的质量门，必须执行。尤其是有人物手部、多人交互或复杂镜头运动的片段，跳过审查的代价是高得多的返工成本。

**错误 2：主体动作和镜头同时快速运动**

表现：视频画面混乱，空间关系扭曲，主体在画面中位置不稳。

修复：执行节点 4 的审查条目 2（镜头过度运动）和条目 1（动作复杂度），确认二者不同时处于高速状态。原则是：主体大幅动作 + 镜头固定，或主体静止 + 镜头缓慢运动。

**错误 3：系列片段各自生成，不做一致性审查**

表现：每个片段单独看都还行，但拼接后风格不统一，第 1 段温暖夕阳光，第 3 段突然变成冷蓝调。

修复：必须执行节点 8，在所有片段提示词都生成后做统一的一致性审查，确认风格词、角色词、光线设定在跨片段时的一致性。

## 延伸玩法

- **变体 1：循环视频版**：将最后一个片段的结束状态设计为与第一个片段的起始状态相同，生成无缝循环视频，用于背景视频或直播背景。
- **变体 2：慢动作版**：所有动作方案选择"慢速"，配合工具的慢动作参数，生成电影级慢动作片段。
- **进阶组合**：将"工作流 06（视频分镜工作流）"的分镜脚本输出直接作为本工作流节点 1 的输入，实现从故事文案到 15 秒电影视频的全链路生产。
