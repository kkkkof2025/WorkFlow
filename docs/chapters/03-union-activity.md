# 工作流 03：从活动素材到有温度的宣传故事的 AI 多专家工作流

**适用岗位**：工会干部、宣传员、企业文化岗、HR 品牌

**真实场景**：一场职工运动会刚结束，或者一位劳模刚受到表彰，领导要求三天内产出新闻稿、短视频脚本和公众号推文。你手头只有一堆照片、一份活动方案和几条采访时记下的零散笔记。

**最终目标**：产出一套有温度、有细节、有真实人物的宣传内容包，包括新闻稿、短视频脚本、发布标题和宣传总结。

---

## 输入材料清单

开始前请准备以下材料：

- 活动基本信息（时间、地点、参与人数、活动名称）
- 采访记录或现场笔记（哪怕只有几句话也算）
- 参与人物信息（姓名、岗位、一句话背景）
- 活动照片说明（描述几张有代表性照片里有什么）
- 活动主题或口号（如有）

---

## 工作流总览

```
节点 1：活动素材整理专家
  ↓ 输出：结构化素材包（时间线 + 人物列表 + 关键事件）
节点 2：人物故事挖掘专家
  ↓ 输出：人物故事提案（含人物、背景、变化、细节）
节点 3：情绪主线设计专家
  ↓ 输出：情绪主线方案（主情绪 + 叙事角度 + 核心金句方向）
节点 4：新闻稿生成专家
  ↓ 输出：新闻稿（800 字左右）
节点 5：短视频脚本专家
  ↓ 输出：60 秒短视频脚本
节点 6：标题与金句专家
  ↓ 输出：标题组（5 条）+ 金句组（3 条）
节点 7：真实性审查专家（跨内容全局审查）
  ↓ 输出：真实性审查报告
节点 8：宣传复盘专家
  ↓ 输出：复盘报告 + 改进清单
最终交付：完整宣传内容包
```

---

## 专家节点详解

每个节点包含六部分：节点定位、输入与输出、使用顺序、提示词包（A 快速 / B 专家 / C 自查 / D 返修）、交付给下游节点、人工验收清单。

**关于节点 7（真实性审查专家）**：这是一个跨内容的全局合规审查节点，负责对新闻稿和短视频脚本进行事实核查，与其他节点的"自查审稿版"性质不同——它是一个独立的审查节点，而不是某个节点的内部模式。

**其他节点的 C 自查审稿版和 D 返修优化版是当前节点内部的工作模式，不是新的专家角色。**

---

### 节点 1：活动素材整理专家

#### 1.1 节点定位

把散乱的活动素材整理成结构化的素材包，建立时间线、人物档案和关键事件清单，为后续所有节点提供统一的素材底座。素材整理质量直接决定人物故事挖掘的深度——没有好的素材包，人物故事节点挖掘不出真实细节。

#### 1.2 输入与输出

**输入**：活动基本信息 + 采访笔记 + 照片说明 + 人物信息

**输出**：结构化素材包（时间线 + 人物档案 + 关键事件清单 + 可用细节清单）

#### 1.3 使用顺序

1. 先用「快速生成版」得到结构化素材包初稿。
2. 需要深度挖掘故事线索时，改用「专家增强版」。
3. 用「自查审稿版」检查人物档案是否有具体言行、时间线是否完整。
4. 有问题则用「返修优化版」补充。
5. 对照 1.6 验收清单确认，通过后将素材包交给节点 2 和节点 4。

#### 1.4 提示词包

##### A. 快速生成版

```
请将以下活动素材整理成结构化的素材包。

活动素材：【粘贴所有原始素材，包括笔记、采访记录、照片描述】

输出：
1. 活动时间线（按时间顺序列出关键事件）
2. 人物档案（姓名、岗位、在活动中的角色、有记录的言行）
3. 关键事件清单（最有故事性的 3-5 个瞬间）
4. 可用细节清单（具体的数字、场景描述、人物表情或行为）
```

##### B. 专家增强版

```
你是一位纪实报道编辑，擅长从零散素材中提炼故事线索。

任务：对以下活动素材进行系统整理，重点挖掘可以讲故事的材料。

输入：
- 活动素材：【粘贴所有原始素材】

整理维度：
1. 时间线（活动前/活动中/活动后，各有哪些关键节点）
2. 人物档案（每个出现的人：姓名、岗位、背景、具体行为或发言）
3. 情绪变化点（哪个瞬间有情绪转折，例如：原本紧张的选手赢得了比赛）
4. 可视化细节（可以用画面呈现的具体场景）
5. 数据亮点（参与人数、获奖情况、持续年限等有说服力的数字）

输出：素材包（分类整理）+ 遗漏信息清单（需要补充采访的问题）
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下素材包是否存在以下问题：
1. 是否有人物只有名字和岗位，没有任何具体言行记录？
2. 时间线是否有明显的空白段（某个时段完全没有内容）？
3. 是否缺少具体数字或可量化的信息？
4. 关键事件是否有不够具体、无法还原场景的描述？

【粘贴素材包】

逐条说明问题，提示需要补充什么材料。
```

##### D. 返修优化版

```
根据自查意见，补充素材包中的缺失内容。对于无法补充的信息，标注"待补充"并说明如何获取。

原始素材包：【粘贴原始素材包】
自查意见：【粘贴自查结果】
```

#### 1.5 交付给下游节点

将完整素材包（含时间线、人物档案、关键事件、可用细节）复制，同时交给：

- **节点 2（人物故事挖掘专家）**：用于挖掘人物故事。
- **节点 4（新闻稿生成专家）**：作为新闻稿的事实底座。

#### 1.6 人工验收清单

- [ ] 是否至少有 2 个人物有具体的言行记录（不只是名字和岗位）？
- [ ] 关键事件是否有具体的场景描述（能想象出画面）？
- [ ] 是否有至少 1 个可用的具体数字？

---

### 节点 2：人物故事挖掘专家

#### 2.1 节点定位

从整理好的素材中找出最有故事感的人物线索，提炼人物的背景、变化和关键细节，为宣传内容提供真实的人物支撑。没有真实人物的工会宣传，读完留不下任何印象。

#### 2.2 输入与输出

**输入**：节点 1 的结构化素材包

**输出**：人物故事提案（1-2 个人物，含背景、参与动机、活动中的变化、关键细节、故事方向）

#### 2.3 使用顺序

1. 先用「快速生成版」得到人物故事提案初稿。
2. 需要深度挖掘人物故事弧线时，改用「专家增强版」。
3. 用「自查审稿版」检查故事是否有真实变化、细节是否具体。
4. 有问题则用「返修优化版」调整。
5. 对照 2.6 验收清单确认，通过后将故事提案交给节点 3、节点 4、节点 5。

#### 2.4 提示词包

##### A. 快速生成版

```
请根据以下素材包，找出最有故事感的 1-2 个人物，并为每个人物提炼一个故事方向。

素材包：【粘贴节点 1 的素材包】

对每个人物输出：
- 人物背景（2 句话）
- 参与活动的动机或代表性（为什么他/她特别值得写）
- 活动中发生了什么变化（从哪里到哪里）
- 最打动人的一个细节（具体的言行或瞬间）
- 故事方向（一句话概括这个故事讲什么）
```

##### B. 专家增强版

```
你是一位人物特稿编辑，专注于发现普通人身上的故事价值。

任务：从以下素材包中识别最具故事潜力的人物，提炼完整的人物故事方案。

素材包：【粘贴节点 1 的素材包】

人物评估标准：
1. 有变化（参与活动前后有什么不同）
2. 有细节（有具体的可还原的场景）
3. 有代表性（这个人代表一类群体，读者容易产生共鸣）
4. 不只是获奖代表（最好是普通参与者，而不只是官方代言人）

对推荐人物输出：
- 人物小传（100 字以内，第三人称）
- 故事弧线（背景 → 触发点 → 活动中的关键时刻 → 变化或收获）
- 可用细节清单（可以直接写入文章或脚本的具体描述）
- 适合切入的叙事角度（例如：从他的手说起 / 从那句话说起）
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下人物故事提案是否存在以下问题：
1. 故事是否过于"高大全"，缺少普通人的真实感（只有荣誉没有挣扎）？
2. 细节是否足够具体（能还原画面，还是只是描述性词语）？
3. 故事弧线是否有明确的变化（从什么状态到什么状态）？
4. 是否过度美化，遮蔽了真实的困难或代价？

【粘贴人物故事提案】

逐条说明问题。
```

##### D. 返修优化版

```
根据自查意见调整人物故事提案。补充真实的困难或挣扎，使细节更具体，确保故事弧线有明确的变化。

原始提案：【粘贴原始提案】
自查意见：【粘贴自查结果】
```

#### 2.5 交付给下游节点

将推荐人物的故事方向和可用细节清单复制，同时交给：

- **节点 3（情绪主线设计专家）**：用于设计宣传情绪主线。
- **节点 4（新闻稿生成专家）**：作为新闻稿人物故事段的素材。
- **节点 5（短视频脚本专家）**：作为脚本的核心故事来源。

#### 2.6 人工验收清单

- [ ] 人物是否有具体的、可还原画面的细节（而不只是形容词）？
- [ ] 故事是否有明确的"变化"（从 A 到 B，而不是一直很好）？
- [ ] 故事是否来自真实记录，而不是凭空扩展？

---

### 节点 3：情绪主线设计专家

#### 3.1 节点定位

确定整套宣传内容的情绪主线和叙事角度，让新闻稿、短视频和标题都围绕同一个情绪核心展开。没有统一的情绪主线，三件内容各说各话，宣传效果大打折扣。

#### 3.2 输入与输出

**输入**：节点 2 的人物故事提案 + 活动主题

**输出**：情绪主线方案（主情绪 + 叙事角度 + 核心金句方向 + 各内容形式的情绪应用建议）

#### 3.3 使用顺序

1. 先用「快速生成版」得到情绪主线初稿。
2. 需要区分不同平台的情绪处理差异时，改用「专家增强版」。
3. 用「自查审稿版」检查情绪描述是否可操作、叙事角度是否来自真实故事。
4. 有问题则用「返修优化版」修正。
5. 对照 3.6 验收清单确认，通过后将情绪主线方案交给节点 4、节点 5、节点 6。

#### 3.4 提示词包

##### A. 快速生成版

```
请根据以下人物故事提案，确定宣传内容的情绪主线。

人物故事提案：【粘贴节点 2 的提案】
活动主题：【填入活动主题或口号】

输出：
1. 主情绪（温情/敬意/自豪/激励/共鸣/感动，选一个）
2. 叙事角度（从哪里切入，例如：从一双手 / 从一句话 / 从一个眼神）
3. 核心金句方向（整套内容想让读者记住什么）
4. 新闻稿情绪应用建议
5. 短视频情绪应用建议
```

##### B. 专家增强版

```
你是一位品牌故事策划师，专注于有温度的企业文化内容。

任务：根据以下信息，设计完整的情绪主线方案，让这次活动的宣传内容具有统一的情感逻辑。

输入：
- 人物故事提案：【粘贴节点 2 的提案】
- 活动背景：【填入活动类型和组织背景】
- 目标读者：【填入主要读者群，例如：本单位职工/社会公众/上级领导】

设计要素：
1. 情绪锚点（整套内容中最打动人的那个瞬间是什么）
2. 情绪弧线（读者的情绪应如何从进入到高潮到回落）
3. 叙事角度（如何切入让读者觉得"说的是我们"）
4. 核心金句方向（让读者带走一句话，那句话是什么感觉）
5. 各平台情绪差异（公众号/内网/抖音对情绪处理的不同要求）

输出：情绪主线方案文档 + 各内容形式的情绪应用指引
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下情绪主线方案是否存在以下问题：
1. 情绪主线是否过于单一或笼统（例如"正能量"这种无法操作的描述）？
2. 叙事角度是否能让目标读者产生共鸣（而不只是让领导满意）？
3. 核心金句方向是否真的来自真实故事，而不是口号式表达？

【粘贴情绪主线方案】

逐条说明问题。
```

##### D. 返修优化版

```
根据自查意见修改情绪主线方案。将笼统的情绪描述变成可操作的叙事指引，确保叙事角度来自真实故事。

原始方案：【粘贴原始方案】
自查意见：【粘贴自查结果】
```

#### 3.5 交付给下游节点

将情绪主线方案（含叙事角度和各平台应用建议）完整复制，同时交给：

- **节点 4（新闻稿生成专家）**：用于设定新闻稿的情绪基调。
- **节点 5（短视频脚本专家）**：用于设定脚本的情绪节奏。
- **节点 6（标题与金句专家）**：作为生成标题和金句的方向依据。

#### 3.6 人工验收清单

- [ ] 情绪主线是否明确到"用什么场景、什么细节来表达"？
- [ ] 叙事角度是否具体（而不是"从活动整体说起"）？
- [ ] 核心金句方向是否来自真实人物故事，而不是凭空提炼的口号？

---

### 节点 4：新闻稿生成专家

#### 4.1 节点定位

按照标准新闻稿结构，结合人物故事和情绪主线，生成有温度但符合新闻写作规范的活动新闻稿。新闻稿的难点在于：既要真实具体，又要避免变成干燥的流水账；既要有温度，又不能变成空洞的颂歌。

#### 4.2 输入与输出

**输入**：节点 1 素材包 + 节点 2 人物故事提案 + 节点 3 情绪主线方案

**输出**：新闻稿（800 字左右，含导语、主体、人物故事段、结语）

#### 4.3 使用顺序

1. 先用「快速生成版」得到新闻稿初稿。
2. 需要更强的叙事层次和语言打磨时，改用「专家增强版」。
3. 用「自查审稿版」检查空洞套话、人物细节真实性。
4. 有问题则用「返修优化版」修正。
5. 对照 4.6 验收清单确认，通过后同步给节点 6（标题生成）和节点 7（真实性审查）。

#### 4.4 提示词包

##### A. 快速生成版

```
请根据以下素材，生成一篇工会活动新闻稿。

活动素材包：【粘贴节点 1 的素材包摘要】
人物故事：【粘贴节点 2 的人物故事提案】
情绪主线：【粘贴节点 3 的情绪主线和叙事角度】

格式：
1. 导语（100 字左右，交代时间、地点、规模和意义）
2. 主体（活动概况，3-4 段）
3. 人物故事段（200 字左右，用具体细节讲一个真实人物的故事）
4. 结语（100 字左右，升华意义但不空洞）

总字数约 800 字。不使用"振奋人心""意义深远""取得圆满成功"等模板套话。
```

##### B. 专家增强版

```
你是一位企业新闻撰稿人，专注于写有温度、有故事、不空洞的工会活动报道。

任务：根据以下信息，生成一篇工会活动新闻稿。

输入：
- 素材包：【粘贴节点 1 的素材包】
- 人物故事：【粘贴节点 2 的人物故事提案（含可用细节）】
- 情绪主线：【粘贴节点 3 的方案】

写作要求：
- 导语不从"近日"开头，用一个有画面感的细节切入
- 人物故事段必须包含至少一句真实的人物发言或意译表达
- 不使用"在 XX 领导的带领下""取得了圆满成功"等模板化表达
- 所有数字必须来自素材包，不能编造
- 结语不以"让我们共同期待"收尾

约束：文章中"振奋人心""意义深远""赋能"等空洞词汇出现即视为质量不合格。

输出：新闻稿全文（约 800 字）
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下新闻稿是否存在以下问题：
1. 导语是否从"近日"或时间地点开头（这是最常见的无聊开头）？
2. 人物故事段是否有真实细节，还是只有概括性描述？
3. 是否有"振奋人心""意义深远""取得圆满成功"等模板套话？
4. 是否有编造的数字或无法从素材中核实的描述？
5. 结语是否是模板化收尾？

【粘贴新闻稿全文】

逐条说明问题，标注有问题的具体句子。
```

##### D. 返修优化版


### 节点 7：真实性审查专家

#### 7.1 节点定位

#### 7.2 输入与输出

**输入**：节点 4 新闻稿 + 节点 5 短视频脚本 + 节点 1 原始素材包（用于比对事实）

**输出**：真实性审查报告（问题清单 + 严重程度分级 + 通过/需修改结论）

#### 7.3 使用顺序

1. 先用「快速生成版」做基础事实核查。

## 常见错误

**错误 1：跳过人物故事，直接写新闻稿**

表现：新闻稿全是活动概况，没有人物，读完不知道记住了谁，也感受不到温度。

修复：必须先完成节点 2，确定人物故事和可用细节，再进入节点 4。没有人物的工会宣传稿是失败的。

**错误 2：真实性审查跳过或走形式**

表现：内容里出现了素材里没有的数字，或者人物说了素材里没有记录的话。

修复：每次完成节点 4 和节点 5 后，必须执行节点 7 的真实性审查，逐条比对素材包。

**错误 3：金句变成口号**

表现："为职工服务，为企业发展贡献力量"这类金句无法打动任何人，只是完成任务。

修复：金句必须来自真实人物的真实表达，或者真实场景的具体描述。如果想不到，说明节点 2 的细节挖掘还不够深。

## 延伸玩法

- **变体 1：系列化**：用节点 2 挖掘出的多个人物，为同一次活动做成系列报道，每期聚焦一个人。
- **变体 2：年度总结版**：将全年多次活动的节点 1 素材包汇总输入，用节点 3 设计年度情绪主线，生成年度宣传总结。
- **进阶组合**：与"工作流 01（短视频内容创作工作流）"结合，用本工作流产出的短视频脚本，进入工作流 01 的节点 5（分镜设计），完成从脚本到视频的全链路生产。
