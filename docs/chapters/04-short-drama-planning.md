# 工作流 04：从关键词到完整短剧提案的 AI 多专家工作流

**适用岗位**：短剧编剧、内容策划、IP 开发者、自媒体团队

**真实场景**：你拿到一个关键词或热门话题（例如"霸总逆袭""女性复仇""古代穿越"），需要在三天内产出一份完整的短剧提案，包含类型定位、故事梗概、主角设定、分集结构和平台推荐策略，可以直接投给平台或制作团队。

**最终目标**：一份完整的短剧提案书，包含：题材定位报告、爆点要素清单、故事梗概（含主线支线）、人物关系图文版、分集大纲（15-20集）、每集情绪钩子设计、平台适配策略。

---

## 输入材料清单

开始前请准备以下材料：

- 核心关键词或话题方向（例如："女主穿越成炮灰""男主失忆后爱上仇人"）
- 目标平台（抖音短剧 / 快手小剧场 / 腾讯微短剧 / 其他）
- 集数目标（15集 / 20集 / 30集）
- 受众偏好（女频 / 男频 / 通吃，年龄层偏好）
- 参考竞品（可选，1-2部同类型短剧名称）

---

## 工作流总览

```
节点 1：题材定位专家
  ↓ 输出：题材类型 + 受众画像 + 市场空间评估
节点 2：爆点判断专家
  ↓ 输出：爆点要素清单 + 情节雷区清单
节点 3：故事梗概专家
  ↓ 输出：故事梗概（主线 + 支线，600-1000字）
节点 4：人物关系专家
  ↓ 输出：主要角色档案 + 人物关系网络
节点 5：分集结构专家
  ↓ 输出：分集大纲（每集一句话 + 结尾悬念）
节点 6：情绪钩子专家
  ↓ 输出：每集开头钩子 + 结尾悬念设计
节点 7：平台适配专家
  ↓ 输出：平台适配建议 + 发布策略
节点 8：提案审稿专家
  ↓ 输出：提案审查报告 + 修改建议
最终交付：完整短剧提案书
```

---

## 专家节点详解

每个节点包含六部分：节点定位、输入与输出、使用顺序、提示词包（A 快速 / B 专家 / C 自查 / D 返修）、交付给下游节点、人工验收清单。

**C 自查审稿版和 D 返修优化版是当前节点内部的工作模式，不是新的专家角色。**

---

### 节点 1：题材定位专家

#### 1.1 节点定位

从关键词出发，判断短剧的类型属性、目标受众和市场竞争空间。题材定位是整个提案的地基——定位不清，后续梗概和分集结构都会出现方向偏差。

#### 1.2 输入与输出

**输入**：核心关键词 + 目标平台 + 受众偏好

**输出**：题材类型标签（2-3个）+ 核心受众画像 + 市场空间评估（竞争度、机会点）+ 内容方向建议

#### 1.3 使用顺序

1. 先用「快速生成版」得到题材定位初稿。
2. 需要更深入的市场分析时，改用「专家增强版」。
3. 用「自查审稿版」检查定位是否清晰、受众是否具体。
4. 有问题则用「返修优化版」修正。
5. 对照 1.6 验收清单确认，通过后交给节点 2。

#### 1.4 提示词包

##### A. 快速生成版

```
你是一位短剧内容策划师。请根据以下关键词，判断短剧题材定位和目标受众。

关键词：【填入关键词】
目标平台：【填入平台】

输出：
1. 题材类型标签（2-3个，例如：霸总爱情 / 女频逆袭 / 古装玄幻）
2. 核心受众画像（年龄、性别、核心情绪诉求）
3. 市场竞争评估（高/中/低竞争，说明理由）
4. 内容方向建议（如何做出差异化，一句话）
```

##### B. 专家增强版

```
你是一位资深短剧项目开发总监，熟悉各平台算法偏好和用户消费心理。

任务：对以下关键词进行系统性题材定位分析，输出可执行的内容方向。

输入：
- 关键词：【填入关键词】
- 目标平台：【填入平台】
- 受众偏好：【填入女频/男频/通吃及年龄层】
- 参考竞品（可选）：【填入同类短剧名称】

分析步骤：
1. 判断该关键词对应的主流叙事类型（权谋 / 甜宠 / 逆袭 / 悬疑等）
2. 识别目标受众的核心情绪诉求（爽感 / 共鸣 / 刺激 / 代入感）
3. 评估市场饱和度（同类内容数量、近期是否有爆款）
4. 找出差异化切入点（哪个角度竞争最少、潜力最大）

约束：不写"潜力巨大""前景广阔"等空洞表达；差异化建议必须具体到叙事策略层面。

输出格式：题材定位报告（含类型标签、受众画像、市场评估、差异化方向，共4个模块）
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下题材定位是否存在以下问题：
1. 题材类型标签是否过于宽泛，无法指导后续创作（例如只写"爱情类"）？
2. 受众画像是否具体到情绪诉求层面（不能只写"18-35岁女性"）？
3. 差异化建议是否是具体的叙事策略，而非"要有创意"这类废话？
4. 市场评估是否说明了理由，而不只是高/中/低的结论？

【粘贴题材定位输出内容】

逐条说明问题。
```

##### D. 返修优化版

```
根据自查意见修改题材定位报告。保留已确定的类型方向，针对每条问题做具体调整。

原始输出：【粘贴原始题材定位】
自查意见：【粘贴自查结果】

重点修正：类型标签精确化、受众画像具体化、差异化建议可操作化。
```

#### 1.5 交付给下游节点

将题材定位报告完整复制，交给节点 2（爆点判断专家）。节点 2 将基于定位方向判断哪些情节要素能引发传播。

#### 1.6 人工验收清单

- [ ] 题材类型标签是否精确到亚类型（不是宽泛的"爱情"，而是"霸总甜宠"或"虐恋反转"）？
- [ ] 受众画像是否包含了核心情绪诉求（不只是年龄性别）？
- [ ] 差异化方向是否是具体可执行的叙事策略？

---

### 节点 2：爆点判断专家

#### 2.1 节点定位

基于题材定位，分析哪些情节要素、人设冲突和叙事结构能引发用户传播和留存。爆点判断决定了故事梗概的骨架——哪些情节必须有，哪些是雷区。

#### 2.2 输入与输出

**输入**：节点 1 的题材定位报告

**输出**：爆点要素清单（5-8条必须包含的情节要素）+ 情节雷区清单（3-5条要避免的常见失误）

#### 2.3 使用顺序

1. 先用「快速生成版」得到爆点要素初稿。
2. 需要更深入分析传播机制时，改用「专家增强版」。
3. 用「自查审稿版」检查爆点是否与受众情绪对应。
4. 有问题则用「返修优化版」修正。
5. 对照 2.6 验收清单确认，通过后交给节点 3。

#### 2.4 提示词包

##### A. 快速生成版

```
你是一位短剧内容分析师。请根据以下题材定位，列出这类短剧的核心爆点要素和情节雷区。

题材定位：【粘贴节点1的题材定位报告】

输出：
1. 爆点要素清单（5-8条，说明每条为何能引发传播或留存）
2. 情节雷区清单（3-5条常见失误，说明为什么观众会流失）
```

##### B. 专家增强版

```
你是一位短剧爆款内容研究员，专注研究短视频平台的内容传播规律。

任务：基于以下题材定位，系统分析该类型短剧的爆点结构，输出可直接用于指导剧本创作的要素清单。

输入：
- 题材定位：【粘贴节点1的完整报告】

分析维度：
1. 人设爆点（什么样的人设组合最容易产生冲突和情绪共鸣）
2. 情节爆点（哪类情节节点最容易引发弹幕、转发、追更）
3. 节奏爆点（信息密度、反转频率、情绪高峰的时机设计）
4. 情节雷区（该类型短剧最常见的导致弃剧的剧情问题）

约束：每条爆点要素必须说明"为什么有效"（指向受众的哪种情绪）；不能只列通用爆点，必须针对该题材类型。

输出：爆点要素清单（带情绪机制说明）+ 情节雷区清单（带弃剧原因说明）
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下爆点要素清单是否存在以下问题：
1. 爆点要素是否过于通用，适用于所有短剧而非针对该题材类型？
2. 每条爆点是否都说明了对应的受众情绪机制？
3. 情节雷区是否是该类型的具体常见问题，而非泛泛的"剧情拖沓"？
4. 爆点与雷区之间是否存在矛盾（同一个情节既是爆点又是雷区）？

【粘贴爆点判断输出内容】

逐条说明问题。
```

##### D. 返修优化版

```
根据自查意见修改爆点清单。保留已确认的核心要素，修正过于通用的条目，补充情绪机制说明。

原始输出：【粘贴原始爆点清单】
自查意见：【粘贴自查结果】
```

#### 2.5 交付给下游节点

将爆点要素清单和情节雷区清单完整复制，交给节点 3（故事梗概专家）。节点 3 在构建故事结构时，必须确保主要情节节点覆盖爆点要素，并规避雷区。

#### 2.6 人工验收清单

- [ ] 爆点要素是否针对该题材类型，而非适用于所有短剧？
- [ ] 每条爆点是否说明了对应的受众情绪（爽感/共鸣/悬念等）？
- [ ] 情节雷区是否是该类型实际存在的常见问题？

---

### 节点 3：故事梗概专家

#### 3.1 节点定位

基于题材定位和爆点要素，构建完整的故事梗概，包括主线冲突、支线关系和故事走向。梗概质量直接决定后续分集结构的可行性——梗概太薄，15-20集撑不起来；梗概太散，分集会失去节奏。

#### 3.2 输入与输出

**输入**：节点 1 题材定位 + 节点 2 爆点要素清单

**输出**：故事梗概（600-1000字），含开篇设定、主线冲突发展、关键转折点、高潮设计、结局方向

#### 3.3 使用顺序

1. 先用「快速生成版」得到故事梗概初稿。
2. 需要更精密的情节结构时，改用「专家增强版」。
3. 用「自查审稿版」检查爆点覆盖、冲突清晰度和结构完整性。
4. 有问题则用「返修优化版」修正。
5. 对照 3.6 验收清单确认，通过后交给节点 4。

#### 3.4 提示词包

##### A. 快速生成版

```
你是一位短剧编剧。请根据以下题材定位和爆点要素，写一个完整的故事梗概。

题材定位：【粘贴节点1输出】
爆点要素：【粘贴节点2的爆点清单】
集数目标：【填入15/20/30集】

要求：
- 梗概包含：开篇人物与矛盾设定、主线冲突发展、关键转折点（至少2个）、高潮设计、结局方向
- 字数 600-1000 字
- 不使用"命运""蜕变"等空洞词汇
```

##### B. 专家增强版

```
你是一位资深短剧编剧，擅长设计情节密度高、留存率强的故事结构。

任务：根据以下输入，构建完整的故事梗概，确保主线冲突清晰、爆点要素覆盖充分。

输入：
- 题材定位报告：【粘贴节点1输出】
- 爆点要素清单：【粘贴节点2爆点清单】
- 情节雷区清单：【粘贴节点2雷区清单】
- 集数目标：【填入集数】

结构要求：
- 开篇（第1-3集范围）：核心矛盾和人物关系建立，必须包含一个强开篇钩子
- 发展段（第4-12集范围）：主线冲突升级，至少2个重大反转
- 高潮段（第13-17集范围）：最大情绪爆发点，覆盖核心爆点要素
- 收尾（最后3集）：矛盾解决方式，结局情绪定调

约束：每个关键转折点说明"触发什么情绪"；规避爆点雷区中的所有问题；不能写成流水账。

输出：完整故事梗概（800-1000字）+ 关键情节节点标注（用【】标出每个重大转折）
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下故事梗概是否存在以下问题：
1. 主线冲突是否清晰（读完梗概能否一句话说清楚核心矛盾）？
2. 节点2中的爆点要素是否在梗概中有对应的情节支撑？
3. 是否有明显的情节雷区（节点2中列出的问题）出现在梗概中？
4. 关键转折点是否至少有2个，且每个转折都能产生新的情绪高点？
5. 梗概是否足够支撑目标集数（太薄则分集时会注水）？

【粘贴故事梗概内容】

逐条说明问题。
```

##### D. 返修优化版

```
根据自查意见修改故事梗概。保留已确认的人物关系和核心设定，针对指出的问题修正故事结构。

原始梗概：【粘贴原始梗概】
自查意见：【粘贴自查结果】

重点修正：强化爆点覆盖、增加反转密度、规避雷区问题。
```

#### 3.5 交付给下游节点

将完整故事梗概复制，同时交给：
- **节点 4（人物关系专家）**：用于提取和设计主要角色。
- **节点 5（分集结构专家）**：用于拆分分集大纲。

#### 3.6 人工验收清单

- [ ] 读完梗概后是否能一句话说清楚核心矛盾？
- [ ] 节点 2 的爆点要素是否在梗概中都有对应情节？
- [ ] 梗概中是否有至少 2 个明显的反转或情绪高点？
- [ ] 梗概内容是否足以支撑目标集数，没有明显注水风险？

---

### 节点 4：人物关系专家

#### 4.1 节点定位

从故事梗概中提取主要角色，设计每个角色的身份、性格、动机和人物关系网络。人物关系设计得好，分集时的冲突来源就会清晰；设计得差，后续剧情会陷入"为推进剧情而强行制造误会"的困境。

#### 4.2 输入与输出

**输入**：节点 3 的故事梗概

**输出**：主要角色档案（3-5个角色，每个含姓名/身份/性格/动机/秘密）+ 人物关系网络（文字版）

#### 4.3 使用顺序

1. 先用「快速生成版」得到角色档案初稿。
2. 需要更深入的角色心理设计时，改用「专家增强版」。
3. 用「自查审稿版」检查角色动机是否清晰、关系是否能产生持续冲突。
4. 有问题则用「返修优化版」修正。
5. 对照 4.6 验收清单确认，通过后交给节点 5。

#### 4.4 提示词包

##### A. 快速生成版

```
你是一位短剧编剧。请根据以下故事梗概，设计主要角色档案和人物关系。

故事梗概：【粘贴节点3的梗概】

输出：
1. 主要角色档案（每个角色：姓名、身份、性格特点、核心动机、是否有秘密）
2. 人物关系网络（文字说明谁与谁是什么关系，以及这段关系的核心张力）
```

##### B. 专家增强版

```
你是一位资深短剧编剧，擅长设计有内在逻辑的人物关系系统。

任务：基于以下故事梗概，设计完整的人物关系档案，确保每个角色都有清晰的行为动机，且角色间的关系能持续产生冲突。

输入：
- 故事梗概：【粘贴节点3的完整梗概】

设计要求：
- 每个主要角色必须有：姓名、身份背景、核心性格（2-3个关键词）、核心动机（他/她最想要什么）、核心秘密或隐藏信息（为后续反转服务）
- 人物关系网络：说明每对主要角色之间的关系类型（敌对/爱恨/利用/真心等）及其背后的张力来源
- 角色动机必须能解释他们的主要行为，不能出现"为了推进剧情而莫名其妙做决定"的设计

约束：不超过5个主要角色；每个角色的秘密或隐藏信息必须与主线冲突相关。

输出：角色档案表格 + 人物关系网络说明（含张力分析）
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下角色档案和人物关系设计是否存在以下问题：
1. 是否有角色缺少清晰的行为动机（读完不知道他/她为什么这样做）？
```

## 常见错误

**错误 1：题材定位没完成就写梗概**

表现：梗概写完后发现故事定位不清晰，受众在哪里、爽点是什么都不明确，导致分集时不知道该强化哪些情节。

修复：必须先完成节点 1 和节点 2，在有清晰题材定位和爆点要素的情况下再开始写梗概。

**错误 2：分集大纲连续注水**

表现：中间有 4-6 集只有对话和感情推进，没有情节进展，导致观众流失。

修复：执行节点 5 的自查审稿版，检查是否有连续 3 集无明显情节进展，有则立即压缩或合并。

**错误 3：提案模块各自为战**

表现：题材定位说是"女性复仇爽剧"，故事梗概却写成了"虐恋疗愈"，角色动机又与两者都不匹配。

修复：执行节点 8 的内部一致性审查，确保所有模块围绕同一个题材核心展开。

## 延伸玩法

- **变体 1：IP 改编版**：将节点 3 的输入替换为已有小说的章节摘要，快速生成短剧改编提案。
- **变体 2：多平台版本**：完成一份核心提案后，用节点 7 分别生成抖音版、快手版和腾讯版三份不同适配策略。
- **进阶组合**：与"工作流 05（角色设定工作流）"结合，用本工作流的角色档案直接驱动角色设定图的生成；与"工作流 01（短视频创作工作流）"结合，将分集大纲中的高潮集快速剪辑为宣传短视频。
