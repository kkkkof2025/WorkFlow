# 工作流 30：从零散待办到可执行行动计划的 AI 多专家任务管理工作流

## 基本信息

- **工作流名称**：从零散待办到可执行行动计划的 AI 多专家任务管理工作流
- **适用岗位**：职场人、学生、管理者、自由职业者、项目执行者
- **真实场景**：一名项目经理周一早上打开各个工具，发现微信群里有 20 条待回复、飞书有 5 个任务待处理、邮件有 3 个请求、自己笔记本上还有上周没做完的 12 项任务。不知道从哪里开始，感觉什么都紧急，结果忙了一天却什么都没真正完成。
- **最终目标**：将所有零散待办整合为一个可执行的行动计划，有明确的优先级、时间估算、精力分配，让每天结束时能清楚知道"今天完成了什么、什么刻意推后了、明天先做什么"。

---

## 输入材料清单

1. 所有来源的待办事项（微信/飞书/邮件/笔记本/口头承诺）
2. 截止日期约束（哪些任务有 deadline）
3. 今日可用工作时间（小时数）
4. 精力状态自评（高/中/低）
5. 本周/本月的核心目标（如有）

---


## 专家节点详解

### 专家 1：待办收集专家

#### 1.1 节点定位

将分散在各处的待办事项统一收拢，确保没有任务遗漏，同时识别重复项和已过期项。

#### 1.2 输入与输出

- **输入**：各来源的待办清单（文字描述即可）
- **输出**：统一的原始待办清单（去重、去过期、标注来源）

#### 1.3 使用顺序

第一步，整理前必须先收集完整。跳过这步会导致后续计划缺项。

#### 1.4 提示词包

**A. 快速生成版**

```
以下是我从各个地方收集的待办，请整理成统一清单，去掉重复项和已明显过期的项：
[粘贴各来源待办]
```

**B. 专家增强版**

```
【角色】你是一名任务管理顾问，专注于帮助人把散乱信息整理为清晰可操作的任务清单。

【任务】将以下来自各来源的待办整合为统一的原始任务清单。

【输入材料】
各来源待办：[逐条列出，标明来源]

【处理步骤】
1. 逐条列出所有待办，标注来源（微信/邮件/飞书/笔记本/口头）
2. 识别并合并重复项（不同表述但实质相同的任务）
3. 标注已明显过期但未删除的项（过期任务：截止日已过且结果未提及）
4. 识别模糊任务（描述不清楚到无法执行的，如"处理一下那个事"）
5. 对模糊任务标注"需澄清"，不要猜测其含义

【输出格式】
| 序号 | 任务描述 | 来源 | 状态（待执行/需澄清/疑似过期） |

【约束条件】不添加用户未提到的任务；不删除任何任务（即使看起来不重要）；模糊任务不猜测，只标注"需澄清"

【质量标准】整理后的清单能让用户在5分钟内确认没有遗漏

【常见失败情况】自作主张删除"看起来不重要"的任务；把模糊任务补全成具体任务
```

**C. 自查审稿版**

> 说明：本版本是当前节点的内部审查模式，不是新专家角色。

```
检查待办清单：
1. 是否保留了所有任务（包括看起来不重要的）？
2. 模糊任务是否标注了"需澄清"而不是被猜测补全？
3. 是否标注了来源？
[粘贴清单]
```

**D. 返修优化版**

> 说明：本版本是当前节点的内部优化模式，不是新专家角色。

```
根据审查修正清单：
[审查意见] [原清单]
修正：补回被删除的任务，将猜测补全的模糊任务还原为"需澄清"标注。
```

#### 1.5 交付给下游节点

将统一原始清单交给**任务分类专家（节点2）**。

#### 1.6 人工验收清单

- [ ] 是否来自所有来源的任务都已收录？
- [ ] 是否标注了来源和状态？
- [ ] 模糊任务是否标注"需澄清"而不是被猜测？

#### 2.3 使用顺序

### 专家 3：优先级判断专家 ★招牌节点

#### 3.1 节点定位

基于重要性与紧急性的四象限框架，对每个任务做优先级判断，给出**真正有取舍的**优先级排序。**核心约束：不能把所有任务都排成"今天必须做"。必须给出明确的取舍建议。**

#### 3.2 输入与输出

- **输入**：分类后的任务清单（节点2）、截止日期信息、本周/月核心目标
- **输出**：四象限分类结果 + 每个象限的处理建议 + 今日执行建议（最多5项）

#### 3.3 使用顺序

第三步，是整个工作流最关键的判断节点，直接决定今天做什么、不做什么。

#### 3.4 提示词包

**A. 快速生成版**

```
请用四象限（重要且紧急/重要不紧急/紧急不重要/不重要不紧急）对以下任务排优先级。
重要：与核心目标直接相关，不做会有实质影响。
紧急：有具体截止时间，今天不做会有后果。
必须给出取舍建议，不能把所有任务都放第一象限。
核心目标：[填入]
任务清单：[粘贴]
```

**B. 专家增强版**

```
【角色】你是一名时间管理顾问，专门帮助人做真正有取舍的优先级决策，而不是让所有事情都"重要且紧急"。

【任务】对以下任务做四象限优先级判断，并给出可执行的今日安排建议。

【输入材料】
- 任务清单（已分类）：[节点2输出]
- 截止日期信息：[填写]
- 本周/月核心目标：[填写，如无则说明]
- 今日可用时间：[X小时]

【判断框架说明】
四象限定义：
- **重要**：与核心目标直接相关，或不做会对工作结果/重要关系产生实质性影响。注意：别人催你不代表重要，紧急不等于重要。
- **紧急**：有具体截止时间（今天/明天），且超时有实际后果（罚款/错过机会/影响他人工作）。

象限处理方式：
- **第一象限（重要且紧急）**：立即执行，今日必须完成。数量不能超过3个（超过说明判断标准出了问题）。
- **第二象限（重要不紧急）**：规划时间块，本周内完成。这是最容易被忽视但最有战略价值的任务。
- **第三象限（紧急不重要）**：快速处理或委托他人。不占用核心工作时间，安排在精力低谷期。
- **第四象限（不重要不紧急）**：推迟或删除。今天不做，本周不做，考虑是否真的需要做。

【处理步骤】
1. 对每个任务，先判断是否重要（对照核心目标），再判断是否紧急（是否有今天的截止时间）
2. 将任务放入对应象限
3. **如果第一象限超过3个，必须重新审查**：
   - 找出哪些"紧急"实际上不是今天必须完成（可以推一天）
   - 找出哪些"重要"实际上不是与核心目标直接相关（是习惯性觉得重要）
4. 给出今日执行建议（从第一象限取1-3个，加第三象限快速处理项，总计不超过5项）
5. 明确说明哪些任务**今天不做**以及理由

【输出格式】

## 四象限分类结果

### ■ 第一象限：重要且紧急（今日必做，最多3项）
| 任务 | 为什么重要 | 为什么紧急 | 截止时间 |

### □ 第二象限：重要不紧急（本周规划）
| 任务 | 为什么重要 | 建议安排时间 |

### △ 第三象限：紧急不重要（快速处理或委托）
| 任务 | 为什么不重要 | 处理方式（快速/委托/拒绝） |

### ○ 第四象限：不重要不紧急（推迟或删除）
| 任务 | 决策（推迟到X日/删除/定期检查） |

## 今日执行建议（不超过5项，按顺序执行）
1. [任务名]（来自第X象限，预计X分钟）
2. ...

## 今天明确不做的任务
- [任务名]：理由（推迟到[日期]/等待他人反馈/本周规划）

【约束条件】
- 第一象限任务不超过3个，超过必须重新审查并说明
- "今日执行建议"不超过5项
- "今天明确不做"必须有理由，不能只说"优先级低"
- 别人催你但与核心目标无关的任务应归入第三象限，不是第一象限
- 今天可用时间不够完成所有第一象限任务时，必须明确说明并要求用户做取舍

【质量标准】
- 执行建议项目数量与今日可用时间匹配（不能把10小时的工作放进5小时里）
- 第四象限的任务有具体的"推迟到几日/删除"决定，而不是模糊的"之后再说"

【常见失败情况】
- 把所有任务都放第一象限（没有真正判断重要性）
- 第一象限有8个任务（超过一天能处理的数量）
- "今天不做"的清单为空（说明没有做任何取舍）
- 把别人催促的任务全部判断为重要
```

**C. 自查审稿版**

> 说明：本版本是当前节点的内部审查模式，不是新专家角色。

```
对以下四象限分类做审查：
1. 第一象限任务是否不超过3个？如超过，是否说明了原因并重新审查？
2. "今日执行建议"是否不超过5项且与可用时间匹配？
3. "今天明确不做"是否有具体理由？
4. 是否有任务仅因为别人催就被放入第一象限（不符合重要性标准）？
5. 第四象限任务是否有具体推迟日期而非模糊"之后再说"？
[粘贴四象限分类结果]
```

**D. 返修优化版**

> 说明：本版本是当前节点的内部优化模式，不是新专家角色。

```

[审查意见] [原分类]
修正：
- 将第一象限超出3个的任务重新判断，降级到第二或第三象限
- 补充"今天明确不做"的具体理由
- 将"之后再说"改为具体日期
- 审查是否有因为催促而非重要性进入第一象限的任务
```

#### 3.5 交付给下游节点

将四象限分类结果和今日执行建议交给**时间估算专家（节点4）**。

#### 3.6 人工验收清单

- [ ] 今日执行建议是否不超过 5 项？
- [ ] 是否有"今天明确不做"的任务清单（非空）？
- [ ] "今天不做"的任务是否都有具体理由？
- [ ] 第四象限任务是否有具体推迟日期？
- [ ] 今日执行任务总量是否与可用时间匹配？

### 专家 4：时间估算专家

#### 4.1 节点定位

为今日执行任务估算实际需要的时间，识别"计划总比实际少估30%"的偏差风险。

#### 4.2 输入与输出

- **输入**：今日执行任务清单（节点3的今日建议）、今日可用时间
- **输出**：每项任务的时间估算 + 总时间 vs 可用时间对比

#### 4.3 使用顺序

第四步，在确认今日任务后做时间核对。

#### 1.4 提示词包

**A. 快速生成版**

```
今日可用时间：[X小时]，今日任务：[列表]。请估算每项任务的时间，并给出总计是否超出今日可用时间。
```

**B. 专家增强版**

```
【角色】你是一名时间管理顾问，专门识别任务时间低估问题。

【任务】为今日任务做时间估算。

【输入材料】
- 今日任务：[节点3的今日执行建议]
- 今日可用时间：[X小时]

【处理步骤】
1. 为每项任务估算乐观时间（最快完成）和悲观时间（有干扰时）
2. 使用悲观时间做规划，不用乐观时间
3. 总计所有任务悲观时间，与今日可用时间对比
4. 如果总时间超出可用时间，明确告知需要削减哪些任务
5. 在可用时间中留15-20%缓冲（处理意外）

【输出格式】
| 任务 | 乐观时间 | 悲观时间 | 建议用时 |
总计 vs 可用时间对比 + 是否需要削减

【约束条件】总计时间不能超过可用时间的85%（留15%缓冲）；不允许"挤一挤能做完"的乐观估算

【质量标准】用悲观时间规划后，如果不发生意外，应该能提前完成，有成就感而不是失败感

【常见失败情况】全用乐观时间导致计划总是完不成；没有留缓冲时间
```

**C. 自查审稿版**

> 说明：本版本是当前节点的内部审查模式，不是新专家角色。

```
检查时间估算：
1. 是否使用悲观时间（而非乐观时间）规划？
2. 总时间是否不超过可用时间的85%？
3. 如超出，是否给出了削减建议？
[粘贴时间估算表]
```

**D. 返修优化版**

> 说明：本版本是当前节点的内部优化模式，不是新专家角色。

```
根据审查修正时间估算：
[审查意见] [原估算]
修正：使用悲观时间，留15%缓冲，超时时给出明确削减建议。
```

#### 4.5 交付给下游节点

将时间估算结果交给**阻塞点识别专家（节点5）**和**执行计划专家（节点6）**。

#### 4.6 人工验收清单

- [ ] 是否使用悲观时间规划？
- [ ] 总时间是否不超过可用时间的 85%？
- [ ] 超出时是否给出了削减建议？

### 专家 5：阻塞点识别专家

#### 5.1 节点定位

识别今日任务中哪些任务的推进依赖外部条件（等待他人回复/需要获取信息/需要工具/需要决策授权），避免"开始了但做不下去"的情况。

#### 5.2 输入与输出

- **输入**：今日执行任务清单、时间估算（节点4）
- **输出**：阻塞点清单 + 每个阻塞点的解除前置动作

#### 5.3 使用顺序

第五步，在安排执行顺序前先识别阻塞，避免卡住。

#### 5.4 提示词包

**A. 快速生成版**

```
今日任务：[列表]。请识别哪些任务存在外部依赖或阻塞条件，并给出在正式执行前需要先做什么来解除阻塞：
```

**B. 专家增强版**

```
【角色】你是一名项目执行顾问，专门识别任务推进的阻塞点。

【任务】识别今日任务中的阻塞条件。

【输入材料】今日任务清单：[节点3+节点4输出]

【处理步骤】
1. 对每项任务，检查是否存在以下阻塞类型：
   - 信息阻塞：需要某个信息/文件/数据才能开始（在哪里获取？谁有？）
   - 人员阻塞：需要他人回复/确认/配合（谁？最晚什么时候需要回复？）
   - 决策阻塞：需要上级授权或做一个决定（谁来决策？）
   - 工具阻塞：需要某个工具/权限/环境（现在有吗？）
2. 对每个阻塞点，给出"解除前置动作"（今天开始前先做X）
3. 识别哪些任务因为阻塞无法今天完成，建议移出今日计划

【输出格式】
| 任务 | 阻塞类型 | 阻塞说明 | 解除前置动作 | 预计解除时间 |
+ 建议移出今日计划的任务

【约束条件】如果阻塞无法今天解除，必须建议移出今日计划（不能硬等）

【质量标准】今日执行开始时，所有可能的阻塞点已提前处理或明确了等待时限

【常见失败情况】识别了阻塞但没有给出解除动作；阻塞任务仍然留在今日计划中
```

## 常见错误

| 错误表现 | 修正方式 |
|---------|---------|
| 第一象限有8个任务，每个都是"今天必须做" | 重新判断：截止时间是今天吗？不做真的有实质后果吗？把超出3个的任务移到第二象限 |
| 把"别人催"直接等同于"重要" | 区分紧急和重要：别人催=紧急，不一定重要。先判断是否与核心目标相关 |
| 时间计划每小时都满，没有缓冲 | 强制留15%缓冲；用悲观时间估算；第三象限任务不进入主时间块 |
| 阻塞任务仍然留在今日计划，结果整天等人回复 | 阻塞任务必须先发出解除请求（发邮件/消息），然后从今日主计划移出，有回复时才执行 |
| 每天收集任务但从不删除，清单越来越长 | 每周五做一次"清单审查"：超过2周未动的任务强制删除或明确推迟日期 |
| 精力低时仍然死磕最难的任务，一天什么都没做完 | 低精力状态当天先从第三象限简单任务开始，积累成就感后再推进复杂任务 |

## 延伸玩法

1. **每周回顾**：每周五花15分钟做周级别的四象限分析——把本周所有任务按象限统计，看第二象限（重要不紧急）的任务是否在持续被推迟。如果是，说明被日常第三象限事务挤占，需要在日历上提前锁定第二象限时间块。

2. **任务来源分析**：记录每项任务的来源（自己产生/上级要求/同事请求/客户需求），一个月后统计来源分布。如果80%的任务来自他人，说明主动工作时间不足，需要重新划定边界。

3. **完成率追踪**：连续追踪2周的第一象限完成率。如果完成率持续低于60%，说明第一象限判断过宽（太多任务被标为必做），需要提高第一象限准入标准。

4. **精力模式建档**：连续记录2周的精力状态和任务完成质量，找出自己的个人精力高峰时间段（有人是上午9-11点，有人是下午3-5点），固定将最重要的深度工作安排在该时段，不接受会议邀请。
