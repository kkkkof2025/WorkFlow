# 工作流 25：从合同文本到风险清单的 AI 多专家法务合同初审工作流

**适用岗位**：法务专员、合同管理员、商务人员、采购经理、创业者、各部门合同经办人

**真实场景**：一份合同发过来需要审查，你没有专业法律背景，但需要在签约前识别出重要风险条款、不合理条款和缺失保护条款——既不能放过真正的风险，也不能因为过度谨慎阻碍正常业务。

**最终目标**：一套合同初审交付物，包含：合同基本信息摘要、关键条款梳理、风险条款标记报告、缺失条款识别、修改建议清单、审批决策辅助材料。

> ⚠️ **法律免责声明**：本工作流产出的所有内容**仅作初步辅助参考，不构成法律意见**。高风险条款、重大合同、涉及专业法律判断的情形，**必须咨询持牌律师**。任何基于本工作流产出的签约决策，责任由签约方自行承担。

---

## 输入材料清单

- 合同文本（完整版本，PDF 或文本格式）
- 合同背景说明（交易目的、合作方基本情况、己方主要诉求）
- 行业或合同类型（买卖合同/服务合同/租赁合同/合作协议/保密协议等）
- 公司已有的合同审查标准或红线清单（如有）
- 上一版本合同或模板（如有，用于对比）

---

## 工作流总览

```
节点 1：合同基本信息提取专家
  ↓ 输出：合同摘要（当事人/标的/金额/期限/签约日期）
节点 2：条款结构梳理专家
  ↓ 输出：条款目录与关键条款索引
节点 3：核心权利义务提取专家
  ↓ 输出：双方权利义务清单
节点 4：风险条款标记专家（招牌节点）
  ↓ 输出：分级风险条款报告
节点 5：缺失条款识别专家
  ↓ 输出：缺失或不完整条款清单
节点 6：修改建议生成专家
  ↓ 输出：条款修改建议清单
节点 7：对方立场分析专家
  ↓ 输出：对方条款立场分析 + 谈判策略建议
节点 8：审批决策辅助专家
  ↓ 输出：审批决策辅助摘要
节点 9：合同归档备忘专家
  ↓ 输出：合同归档备忘记录
最终交付：完整合同初审报告包
```

---

## 专家节点详解

每个节点包含六部分：节点定位、输入与输出、使用顺序、提示词包（A 快速 / B 专家 / C 自查 / D 返修）、交付给下游节点、人工验收清单。

**C 自查审稿版和 D 返修优化版是当前节点内部的工作模式，不是新的专家角色。**

> 核心原则：初审结论必须来自合同文本中可引用的条款。不得编造合同中没有的条款，不确定的法律判断标注「建议咨询律师」，不替代专业法律意见。

---

### 节点 1：合同基本信息提取专家

#### 1.1 节点定位

快速提取合同的基本要素，形成一份可供非法律人员快速理解合同概貌的摘要。基本信息不清楚，后续所有审查都缺乏基础。

#### 1.2 输入与输出

**输入**：合同文本全文

**输出**：合同基本信息摘要（当事人、合同类型、标的、金额、期限、管辖、签约日期）

#### 1.3 使用顺序

1. 先用「快速生成版」提取基本信息。
2. 合同结构复杂或多方协议时，用「专家增强版」。
3. 用「自查审稿版」检查提取是否有误读。
4. 有问题则用「返修优化版」修正。
5. 对照 1.6 验收清单确认，通过后交给节点 2。

#### 1.4 提示词包

##### A. 快速生成版

```
你是合同信息提取专家。根据以下合同文本，提取基本信息：①甲乙方全称及主体类型 ②合同类型 ③合同标的（做什么/买什么）④合同金额（总金额及支付方式）⑤合同期限 ⑥争议解决方式（仲裁/诉讼/管辖地）⑦签约日期。所有信息来自合同原文，无法确认的标注「需人工确认」。
合同文本：【粘贴合同全文或关键段落】
```

##### B. 专家增强版

```
你是合同要素提取专家，专注于从合同文本中准确提取关键要素。

输入：合同全文

提取以下字段（每项注明条款位置，如"第X条"）：
1. 甲方：全称、统一社会信用代码（如有）、签约代表
2. 乙方：同上
3. 合同类型：买卖/服务/租赁/合作/保密/其他
4. 合同标的：核心交易内容（一句话描述）
5. 合同金额：总金额、币种、含税/不含税、支付节点
6. 合同期限：起止日期或触发条件
7. 交付/履行标准：合同约定的验收标准或交付物
8. 争议解决：仲裁机构或管辖法院、适用法律
9. 签约日期：合同约定的签署日期
10. 特殊条款标记：是否有排他条款、竞业限制、自动续签等

约束：所有字段引用合同原文，无对应条款的填「合同未约定」。

输出：结构化摘要表
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下合同摘要：
① 每个字段是否标注了对应条款位置？
② 是否有超出合同原文的推断内容？
③ 金额是否区分了含税/不含税？
④ 争议解决方式（仲裁/诉讼）是否准确区分？
【粘贴摘要内容】
逐条列出问题。
```

##### D. 返修优化版

```
根据自查意见修正合同摘要。保留：有条款原文依据的字段。修正：补充条款位置标注、将推断内容替换为「需人工确认」、修正金额和争议解决表述。
原始摘要：【粘贴原文】
自查意见：【粘贴自查结果】
```

#### 1.5 交付给下游节点

将合同基本信息摘要交给所有后续节点使用，作为审查的基准信息。

#### 1.6 人工验收清单

- [ ] 所有字段有对应条款位置标注
- [ ] 无超出合同原文的推断内容
- [ ] 金额字段区分含税/不含税
- [ ] 争议解决方式（仲裁/诉讼）准确

### 节点 2：条款结构梳理专家

#### 2.1 节点定位

梳理合同的条款目录，识别关键条款的位置，方便审查时快速定位。合同条款通常不按重要性排列，本节点生成一份以重要性为维度的条款索引。

#### 2.2 输入与输出

**输入**：合同全文

**输出**：条款目录（原始结构）+ 关键条款索引（按重要性分组）

#### 2.3 使用顺序

1. 用「快速生成版」生成条款目录。
2. 合同篇幅长或结构复杂时，用「专家增强版」。
3. 用「自查审稿版」确认关键条款无遗漏。
4. 有问题则用「返修优化版」补全。
5. 对照 2.6 验收清单确认，交给节点 3 和节点 4。

#### 2.4 提示词包

##### A. 快速生成版

```
你是合同结构梳理专家。根据以下合同文本，生成：①原始条款目录（条款编号+标题，无标题则用首句概括）②关键条款索引（将条款分为：核心交易条款/付款条款/违约责任条款/知识产权条款/保密条款/争议解决条款/其他重要条款）。每类列出对应条款编号。
合同文本：【粘贴】
```

##### B. 专家增强版

```
你是合同条款结构分析专家，专注于将合同条款按重要性和风险维度重新索引。

输入：合同全文

步骤：
1. 原始目录：按合同编号顺序列出所有条款
2. 关键条款分组索引（标注条款编号）：
   - 核心义务条款：双方主要权利义务
   - 付款/结算条款：金额、时间、方式、条件
   - 验收/交付条款：标准、流程、时间节点
   - 违约/赔偿条款：违约情形、赔偿方式、上限
   - 知识产权条款：归属、许可、限制
   - 保密/竞业条款：保密范围、期限、竞业限制
   - 合同解除条款：单方解除权、协议解除条件
   - 不可抗力条款：定义、通知义务、后果
   - 争议解决条款：方式、管辖、适用法律
3. 缺失类别标注：上述分组中，合同未包含的类别标注「合同无此条款」

输出：两部分——原始目录 + 分组索引
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下条款索引：
① 每个分组是否都有对应条款编号（或标注「合同无此条款」）？
② 是否有重要条款被遗漏或归错分组？
③ 「合同无此条款」的分组是否已标注？
【粘贴索引内容】
逐条列出问题。
```

##### D. 返修优化版

```
根据自查意见修正索引。保留：准确的分组归类。修正：补充遗漏条款、修正归类错误、补充「合同无此条款」标注。
原始索引：【粘贴原文】
自查意见：【粘贴自查结果】
```

#### 2.5 交付给下游节点

将条款索引传递给节点 3、4、5，作为定位合同内容的导航工具。

#### 2.6 人工验收清单

- [ ] 所有条款均在原始目录中
- [ ] 9 类关键条款分组均有处理（有或标注「无此条款」）
- [ ] 条款编号引用准确

### 节点 3：核心权利义务提取专家

#### 3.1 节点定位

从合同中提取双方的核心权利和义务，以通俗语言呈现，让非法律背景的决策者能快速理解"我要做什么、对方要做什么、我能要求什么"。

#### 3.2 输入与输出

**输入**：合同全文 + 条款索引（节点 2）

**输出**：双方权利义务清单（甲方权利/甲方义务/乙方权利/乙方义务，附条款依据）

#### 3.3 使用顺序

1. 用「快速生成版」提取权利义务。
2. 合同条款较多或权利义务交叉时，用「专家增强版」。
3. 用「自查审稿版」检查是否有遗漏或误读。
4. 有问题则用「返修优化版」修正。
5. 对照 3.6 验收清单确认，交给节点 4。

#### 3.4 提示词包

##### A. 快速生成版

```
你是合同权利义务提取专家。根据以下合同，提取双方核心权利义务，格式：甲方义务（必须做什么）/ 甲方权利（可以要求什么）/ 乙方义务（必须做什么）/ 乙方权利（可以要求什么）。每条附条款编号，用通俗语言描述，保留原文关键词。
合同文本：【粘贴】
```

##### B. 专家增强版

```
你是合同权利义务分析专家，专注于将合同条款转化为清晰的双方权责清单。

输入：合同全文 + 条款索引

提取格式（每条附条款编号和原文关键语句）：

**甲方主要义务**（甲方必须履行的义务，不履行构成违约）
**甲方主要权利**（甲方有权要求对方履行或有权单方做出的行为）
**乙方主要义务**（同上）
**乙方主要权利**（同上）

特别标注：
- 附条件的义务（在特定条件下才触发的义务）
- 期限敏感的义务（有具体时间节点的）
- 不作为义务（禁止做某事）

约束：提取内容必须来自合同原文，不添加合同没有的义务推断。

输出：结构化权利义务清单
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下权利义务清单：
① 每条是否有对应条款编号？
② 是否有超出合同原文的义务推断被写入清单？
③ 期限敏感义务是否已特别标注？
【粘贴清单内容】
逐条列出问题。
```

##### D. 返修优化版

```
根据自查意见修正清单。保留：有条款依据的权利义务描述。修正：删除超出原文的推断、补充条款编号、标注期限敏感义务。
原始清单：【粘贴原文】
自查意见：【粘贴自查结果】
```

#### 3.5 交付给下游节点

将权利义务清单交给节点 4（风险标记）和节点 8（审批辅助），帮助决策者快速理解合同核心内容。

#### 3.6 人工验收清单

- [ ] 每条义务/权利有条款编号依据
- [ ] 无超出合同原文的推断内容
- [ ] 期限敏感义务已标注
- [ ] 用通俗语言表述，非法律背景人员可理解

### 节点 4：风险条款标记专家（招牌节点）

#### 4.1 节点定位

识别合同中对己方不利、不公平或高风险的条款，分级标注并说明风险原因。风险识别是合同审查的核心价值所在——不是让所有条款都"完美"，而是让签约方清楚地知道自己承担了什么风险、这些风险是否可以接受。

> **重要边界**：
> - 风险标记基于合同文本中的具体条款表述，每条风险必须引用原文
> - 不对合同条款的法律效力做最终判断（如"此条款无效"），如有疑问标注「建议咨询律师确认法律效力」
> - 高风险条款必须建议咨询律师，不得因初审结论而替代专业法律意见

#### 4.2 输入与输出

**输入**：合同全文 + 条款索引（节点 2）+ 权利义务清单（节点 3）

**输出**：分级风险条款报告（高风险 / 中风险 / 关注项，每条附原文引用 + 风险说明 + 建议）

#### 4.3 使用顺序

1. 先用「快速生成版」识别明显风险。
2. 用「专家增强版」系统扫描所有风险类型。
3. 用「自查审稿版」检查每条风险是否有原文依据。
4. 有问题则用「返修优化版」修正。
5. 对照 4.6 验收清单确认，**高风险条款必须上报法务或律师确认**。

#### 4.4 提示词包

##### A. 快速生成版

```
你是合同风险标记专家。根据以下合同，识别对己方（乙方/甲方）不利的风险条款，分三级：①高风险（可能造成重大损失或重要权利丧失）②中风险（存在不公平或模糊之处，需关注）③关注项（轻微不利但可接受）。每条引用原文条款，说明风险原因。不对法律效力做最终判断，高风险条款建议咨询律师。本分析仅作初步辅助，不构成法律意见。
合同文本：【粘贴】
己方立场：【甲方/乙方】
```

##### B. 专家增强版

```
你是合同风险识别专家，专注于从合同文本中识别对己方不利的条款，为签约决策提供初步参考。

本分析仅作初步辅助参考，不构成法律意见。高风险条款建议咨询持牌律师。

输入：合同全文 + 己方立场（甲方/乙方）

系统检查以下风险类型（每类引用原文，给出风险等级和说明）：

**付款与结算风险：**
1. 付款条件是否苛刻（触发条件过多导致难以收款）
2. 付款时间节点是否合理
3. 扣款或抵扣条款是否有上限约束

**违约责任风险：**
4. 违约金条款是否对己方过重（上限是否合理）
5. 己方违约情形是否定义过宽（模糊表述可能被对方扩大解释）
6. 对方违约的救济措施是否充分

**合同解除风险：**
7. 对方是否有单方随时解除权（无限制的任意解除）
8. 己方解除权是否受到过多限制
9. 解除后的处理安排是否公平

**知识产权风险：**
10. 成果归属是否对己方不利（己方创造的成果归对方所有）
11. 己方现有知识产权是否被过度授权

**保密与竞业风险：**
12. 保密范围是否过宽（可能影响正常业务）
13. 竞业限制是否合理（范围/期限/补偿）

**责任限制风险：**
14. 对方是否有过度的免责条款
15. 己方的责任是否被过度扩大（连带责任/无限责任）

**模糊条款风险：**
16. 关键术语是否未定义（可能产生争议）
17. 核心义务是否描述模糊（履行标准不清）

风险等级标准：
- 高风险：可能直接导致重大财务损失、重要权利丧失或合同目的无法实现
- 中风险：存在不公平或模糊，在争议时对己方不利
- 关注项：轻微不利，在正常履约时影响有限

约束：
- 每条风险必须引用合同原文（条款编号+关键语句）
- 不对条款法律效力做最终判断
- 高风险条款必须注明「建议咨询律师」
- 不确定是否构成风险的标注「建议咨询律师确认」

输出：分级风险报告（高风险/中风险/关注项，每条：条款位置+原文引用+风险说明+建议）
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下风险报告：
① 每条风险是否有合同原文引用（条款编号+关键语句）？
② 是否有对法律效力做最终判断的表述（如"此条款无效"应改为"建议咨询律师确认效力"）？
③ 高风险条款是否都已注明「建议咨询律师」？
```

## 常见错误

**错误 1：对风险条款做最终法律效力判断**

表现：报告写"第X条违反法律规定，属于无效条款"——这是专业法律意见，超出了初审范围。

修复：改为"第X条表述存在争议，建议咨询律师确认该条款在当地法律下的效力"。

---

**错误 2：将方向性修改建议当作正式法律条款**

表现：修改建议写了完整的"修改后条款全文"，被谈判人员直接发给对方——如果有法律瑕疵，可能产生更大风险。

修复：节点 6 只提供修改方向，完整的条款修改语言由律师起草。

---

**错误 3：漏掉免责声明**

表现：报告被内部传阅时，被误认为是正式法律意见，影响决策判断。

修复：节点 4、节点 6、节点 8 的每份产出物都必须包含免责声明，节点 4 自查审稿版会强制检查此项。

---

**错误 4：合同签署后不跟进履约节点**

表现：合同中约定了 30 天内付款、90 天验收期，但没有人记录这些节点，过期后产生违约争议。

修复：每份合同签署后必须执行节点 9，将关键日期录入日历提醒系统。

## 延伸玩法

- **合同模板库建设**：将节点 5（缺失条款）识别出的常见缺失项整理为本公司合同模板的标准条款，持续完善己方合同模板。
- **对比审查**：当对方提供合同时，将节点 4 的风险报告与己方合同模板对比，快速识别对方版本与己方标准版本的差异。
- **与工作流 22 组合**：在销售成交（工作流 22）后，用工作流 25 对合同进行初审，确保在签约前识别销售承诺与合同条款的不一致之处。
