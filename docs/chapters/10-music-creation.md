# 工作流 10：从主题情绪到完整 AI 歌曲方案的 AI 多专家工作流

**适用岗位**：短视频创作者、音乐爱好者、活动宣传人员、剧情创作者

**真实场景**：你想为短视频、活动或个人项目做一首原创歌曲，手里只有一个主题或一种情绪（比如"毕业季的不舍"或"夜跑时的孤独感"），但不知道怎么把它变成可以直接喂给 Suno、Udio 等 AI 音乐平台生成的完整歌曲方案。

**最终目标**：一份可直接用于 AI 音乐生成平台的完整歌曲方案，包含：主题情绪定调、风格定位、带段落标记的完整歌词、副歌记忆点设计、演唱人设说明、编曲氛围描述，以及适配目标平台的生成提示词。

---

## 输入材料清单

- 歌曲主题或情绪方向（一句话，例如："写给异地恋的深夜"）
- 使用场景（短视频BGM / 活动主题曲 / 个人创作 / 剧情插曲）
- 风格偏好（可选：流行/民谣/电子/说唱/古风等）
- 时长预期（30秒短视频版 / 完整版 2-3 分钟）
- 目标平台（Suno / Udio / 其他 AI 音乐工具）
- 语言（中文 / 英文 / 双语）

---

## 工作流总览

```
节点 1：主题情绪解析专家
  ↓ 输出：核心情绪 + 情绪层次 + 意象库
节点 2：歌曲风格定位专家
  ↓ 输出：曲风定位 + 节奏速度 + 参考风格
节点 3：歌词结构专家
  ↓ 输出：歌词整体结构方案
节点 4：副歌记忆点专家
  ↓ 输出：副歌核心句 + 记忆点设计
节点 5：段落结构与情绪递进专家（招牌节点）
  ↓ 输出：带情绪功能标注的完整段落标记
节点 6：演唱人设专家
  ↓ 输出：演唱者人设 + 声音特质描述
节点 7：编曲氛围专家
  ↓ 输出：乐器配置 + 氛围描述
节点 8：歌词审查与修正专家
  ↓ 输出：歌词审查报告 + 修正版歌词
节点 9：生成平台适配专家
  ↓ 输出：适配目标平台的完整生成提示词
最终交付：完整 AI 歌曲方案
```

## 专家节点详解

**C 自查审稿版和 D 返修优化版是当前节点内部的工作模式，不是新的专家角色。**

### 节点 1：主题情绪解析专家

#### 1.1 节点定位

将模糊的主题或情绪方向解析为有层次的情绪结构和可用的意象素材。一首好歌的情绪不是单一的——"不舍"里可能包含回忆的温暖、离别的酸楚和对未来的期待，这个节点负责把情绪展开成可写的层次。

#### 1.2 输入与输出

**输入**：歌曲主题或情绪方向 + 使用场景

**输出**：核心情绪定调 + 情绪层次（2-3层）+ 意象库（可入词的具体意象）

#### 1.3 使用顺序

1. 先用「快速生成版」得到情绪解析初稿。
2. 主题抽象或情绪复杂时，改用「专家增强版」。
3. 用「自查审稿版」检查情绪层次是否清晰、意象是否具体。
4. 有问题则用「返修优化版」补充。
5. 对照 1.6 验收清单确认，通过后交给节点 2 和节点 3。

#### 1.4 提示词包

##### A. 快速生成版

```
你是一位作词人。请将以下歌曲主题解析为可写作的情绪结构。

歌曲主题/情绪：【填入主题】
使用场景：【填入场景】

输出：
1. 核心情绪（一个主导情绪词）
2. 情绪层次（2-3个递进或交织的子情绪）
3. 意象库（5-8个可以写进歌词的具体意象，例如：旧车票/路灯/未读消息）
```

##### B. 专家增强版

```
你是一位资深作词人，擅长把抽象情绪转化为有画面感的歌词素材。

任务：对以下歌曲主题进行深度情绪解析，为后续歌词创作提供丰富的情绪层次和意象素材。

输入：
- 歌曲主题/情绪方向：【填入主题】
- 使用场景：【填入场景】
- 目标听众：【填入听众群体，影响意象选择】

解析维度：
1. 核心情绪（主导整首歌的情绪基调，一个词）
2. 情绪层次（这个情绪展开后包含哪些子情绪，它们之间是递进、对比还是交织关系）
3. 情绪转折点（如果歌曲有情绪变化，转折发生在什么时刻）
4. 意象库（提供3类意象）
   - 视觉意象（可以看到的具体物象）
   - 场景意象（可以代入的具体情境）
   - 动作意象（具体的人物动作）
5. 避免的陈词（这个主题最容易写俗的表达，应该避开）

约束：意象必须具体可感，不能是"思念""快乐"这类直接的情绪词；要给出能让听众产生画面联想的具体事物。

输出：情绪解析报告（5个维度）
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下情绪解析是否存在以下问题：
1. 意象是否足够具体（"思念"是情绪不是意象，"褪色的合照"才是意象）？
2. 情绪层次之间的关系是否清晰（递进/对比/交织）？
3. 是否列出了该主题容易写俗的陈词，以便后续避开？

【粘贴情绪解析内容】

逐条说明问题。
```

##### D. 返修优化版

```
根据自查意见修改情绪解析。将抽象情绪词替换为具体意象，明确情绪层次关系，补充陈词避雷清单。

原始解析：【粘贴原始内容】
自查意见：【粘贴自查结果】
```

#### 1.5 交付给下游节点

将情绪解析报告（核心情绪 + 情绪层次 + 意象库）复制，交给节点 2（风格定位专家）和节点 3（歌词结构专家）。

#### 1.6 人工验收清单

- [ ] 意象库中的意象是否都是具体可感的事物（而非情绪词）？
- [ ] 情绪层次之间的关系是否清晰说明？
- [ ] 是否列出了该主题需要避开的陈词俗套？

### 节点 2：歌曲风格定位专家

#### 2.1 节点定位

基于主题情绪，确定歌曲的曲风、节奏速度和参考风格，为编曲和演唱提供方向。同样的歌词，做成抒情慢歌还是电子快歌，听感天差地别——风格定位决定了整首歌的基本气质。

#### 2.2 输入与输出

**输入**：节点 1 的情绪解析 + 风格偏好（如有）

**输出**：曲风定位 + 节奏速度（BPM 区间）+ 参考风格描述（可用于平台提示词）

#### 2.3 使用顺序

1. 先用「快速生成版」得到风格定位初稿。
2. 需要精准匹配情绪或参考特定风格时，改用「专家增强版」。
3. 用「自查审稿版」检查风格是否与情绪匹配。
4. 有问题则用「返修优化版」调整。
5. 对照 2.6 验收清单确认，通过后交给节点 6 和节点 7。

#### 2.4 提示词包

##### A. 快速生成版

```
你是一位音乐制作人。请根据以下情绪解析，确定歌曲风格。

情绪解析：【粘贴节点1的核心情绪和层次】
风格偏好（如有）：【填入或留空】

输出：
1. 曲风定位（例如：抒情流行/独立民谣/Lo-fi/电子舞曲）
2. 节奏速度（慢/中/快，对应 BPM 区间）
3. 参考风格描述（用于 AI 音乐平台的风格关键词，英文优先）
```

##### B. 专家增强版

```
你是一位资深音乐制作人，熟悉各类曲风与情绪的对应关系，以及 AI 音乐平台的风格关键词。

任务：基于以下情绪解析，确定最能承载该情绪的曲风方案。

输入：
- 情绪解析：【粘贴节点1的完整报告】
- 风格偏好（如有）：【填入或留空】
- 使用场景：【填入场景，影响时长和编曲密度】

风格定位维度：
1. 主曲风（具体到亚类型，例如不是"流行"而是"city pop"或"抒情R&B"）
2. 节奏速度（BPM 区间 + 节奏型，例如：70-80 BPM 慢板抒情）
3. 情绪与曲风匹配说明（为什么这个曲风适合承载该情绪）
4. 参考风格关键词（可直接用于 Suno/Udio 的英文风格标签，例如：emotional, acoustic, melancholic, indie folk）
5. 编曲密度建议（极简/中等/丰富，影响后续编曲氛围设计）

约束：风格关键词必须是 AI 音乐平台能识别的标准英文标签；曲风选择必须与核心情绪匹配（欢快情绪不配阴郁曲风）。

输出：风格定位报告（5个维度）
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下风格定位是否存在以下问题：
1. 曲风是否与核心情绪匹配（不能给悲伤情绪配欢快舞曲）？
2. 曲风是否具体到亚类型（"流行"太宽泛，"抒情R&B"才具体）？
3. 风格关键词是否是 AI 音乐平台能识别的标准英文标签？
4. BPM 区间是否与曲风一致（抒情慢歌不应是 130 BPM）？

【粘贴风格定位内容】

逐条说明问题。
```

##### D. 返修优化版

```
根据自查意见修改风格定位。调整与情绪不匹配的曲风，细化到亚类型，修正风格关键词为标准英文标签，校正 BPM 区间。

原始定位：【粘贴原始内容】
自查意见：【粘贴自查结果】
```

将风格定位报告复制，交给节点 6（演唱人设专家）和节点 7（编曲氛围专家）。曲风和节奏会影响演唱方式和乐器配置。

#### 2.6 人工验收清单

- [ ] 曲风是否与核心情绪匹配？
- [ ] 曲风是否具体到亚类型（而非宽泛的"流行""民谣"）？
- [ ] 风格关键词是否是 AI 平台可识别的英文标签？

### 节点 3：歌词结构专家

#### 3.1 节点定位

设计歌词的整体结构框架，确定每个段落写什么内容、承担什么叙事或情绪功能。结构是歌词的骨架——先有骨架再填血肉，才能避免写成流水账或情绪散乱。

#### 3.2 输入与输出

**输入**：节点 1 的情绪解析 + 时长预期

**输出**：歌词结构方案（各段落的内容分工和情绪定位）

#### 3.3 使用顺序

1. 先用「快速生成版」得到结构方案初稿。
2. 有复杂叙事或情绪转折时，改用「专家增强版」。
3. 用「自查审稿版」检查结构是否完整、各段是否有明确分工。
4. 有问题则用「返修优化版」调整。
5. 对照 3.6 验收清单确认，通过后交给节点 4 和节点 5。

#### 3.4 提示词包

##### A. 快速生成版

```
你是一位作词人。请根据以下情绪解析，设计歌词的整体结构。

情绪解析：【粘贴节点1的输出】
时长预期：【填入短视频版/完整版】

输出：
1. 整体结构（主歌-副歌-桥段的安排）
2. 每个段落的内容分工（这一段写什么，承担什么情绪）
3. 情绪走向（从开头到结尾，情绪如何变化）
```

##### B. 专家增强版

```
你是一位资深作词人，擅长设计有逻辑、有递进的歌词结构。

任务：基于以下情绪解析，设计完整的歌词结构框架，确保每个段落各司其职。

输入：
- 情绪解析（含意象库）：【粘贴节点1的完整报告】
- 时长预期：【填入版本】
- 曲风（如已定）：【粘贴节点2的曲风，影响结构繁简】

结构设计：
1. 整体段落安排（主歌 Verse、预副歌 Pre-Chorus、副歌 Chorus、桥段 Bridge 的组合顺序）
2. 每段内容分工
   - 第一段主歌：通常建立场景和人物
   - 第二段主歌：推进或转折
   - 副歌：情绪和主题的集中爆发
   - 桥段：视角转换或情绪升华
3. 意象分配（节点1的意象库中，哪些意象用在哪个段落）
4. 情绪曲线（用文字描述从第一句到最后一句的情绪强度变化）
5. 叙事视角（第一人称/第二人称/旁观者，是否有视角变化）

约束：结构必须服务于情绪表达，不能机械套用模板；每段必须有明确的功能，不能两段写同样的内容。

输出：歌词结构方案（含段落分工表和情绪曲线说明）
```

```
原始结构：【粘贴原始内容】
```

### 节点 4：副歌记忆点专家

#### 4.1 节点定位

设计副歌的核心句和记忆点，这是整首歌最容易被记住和传唱的部分。一首歌能不能火，很大程度上取决于副歌那一两句——它需要朗朗上口、情绪饱满、易于跟唱。

#### 4.2 输入与输出

**输入**：节点 1 情绪解析 + 节点 3 歌词结构（副歌定位）

**输出**：副歌核心句（1-2 句）+ 记忆点设计说明 + 副歌完整歌词

#### 4.3 使用顺序

1. 先用「快速生成版」得到副歌初稿。
2. 需要更强的传唱度时，改用「专家增强版」。
3. 用「自查审稿版」检查副歌是否朗朗上口、情绪是否饱满。
4. 有问题则用「返修优化版」打磨。
5. 对照 4.6 验收清单确认，通过后交给节点 5。

#### 4.4 提示词包

##### A. 快速生成版

```
你是一位作词人，擅长写朗朗上口的副歌。请为以下歌曲设计副歌。

核心情绪：【粘贴节点1的核心情绪】
意象库：【粘贴节点1的意象库】
副歌定位：【粘贴节点3中对副歌的情绪定位】

输出：
1. 副歌核心句（1-2句最能代表整首歌的句子）
2. 副歌完整歌词（4-6句）
3. 记忆点说明（这个副歌为什么容易记住）
```

##### B. 专家增强版

```
你是一位金牌作词人，写过多首传唱度高的副歌，深知什么样的副歌能让人记住并跟唱。

任务：为以下歌曲设计副歌，目标是创造一个朗朗上口、情绪饱满、易于传唱的记忆点。

输入：
- 核心情绪和意象库：【粘贴节点1的报告】
- 副歌情绪定位：【粘贴节点3的副歌定位】
- 曲风和节奏：【粘贴节点2的曲风BPM】

副歌设计原则：
1. 核心句（hook）：一句最抓人的话，通常包含歌名或主题，简短有力，能独立成立
2. 传唱度设计：句式工整、有重复或排比、避免生僻字和拗口的辅音连缀
3. 情绪饱满度：副歌应是全曲情绪高峰，用词的情绪浓度高于主歌
4. 记忆钩子：可以是重复的词、独特的比喻、或意外的转折
5. 跟唱友好：考虑换气点，句子长度适合一口气唱完

输出：
- 副歌核心句（标注为 hook）
- 副歌完整歌词（4-8句）
- 记忆点设计说明（这个副歌用了什么手法制造记忆点）
- 跟唱友好度说明（换气点、句长是否适合传唱）
```

##### C. 自查审稿版

> 此为当前节点的自查模式，不是新的专家角色。

```
请检查以下副歌是否存在以下问题：
1. 核心句是否朗朗上口（有没有拗口的辅音连缀或生僻字）？
```

### 节点 5：段落结构与情绪递进专家（招牌节点）

#### 5.1 节点定位

**这是本工作流的招牌节点。** 生成适合 AI 音乐生成平台使用的段落结构标记（如 `[Intro] [Verse] [Pre-Chorus] [Chorus] [Bridge] [Outro]`），但绝不机械套用模板，而是为每个段落明确其情绪功能：

- **[Intro] 如何建立氛围**：用什么乐器或人声开场，奠定整首歌的情绪基调
- **[Verse] 如何叙事**：铺陈场景、推进故事，情绪保持克制
- **[Pre-Chorus] 如何蓄力**：从主歌到副歌的桥梁，情绪开始上扬、制造期待
- **[Chorus] 如何形成记忆点**：情绪爆发，副歌的记忆钩子在此呈现
- **[Bridge] 如何转折**：视角或情绪的转换，给听众新鲜感，避免重复疲劳
- **[Outro] 如何收束**：情绪回落或升华，给整首歌一个完整的结束

这个节点把节点 3 的结构方案和节点 4 的副歌，组装成带标记、带情绪递进逻辑的完整段落框架，是 AI 音乐生成成败的关键。

#### 5.2 输入与输出

**输入**：节点 3 歌词结构方案 + 节点 4 副歌歌词 + 节点 1 意象库

**输出**：带段落标记和情绪功能标注的完整歌词框架（含每段的情绪功能说明 + 完整歌词内容）

#### 5.3 使用顺序

1. 先用「快速生成版」生成带标记的段落框架。
2. 需要精细的情绪递进设计时，必须使用「专家增强版」。
3. 用「自查审稿版」检查段落标记是否规范、情绪递进是否合理。
4. 有问题则用「返修优化版」修正。
5. 对照 5.6 验收清单确认，通过后交给节点 6、节点 7 和节点 8。

### 节点 9：生成平台适配专家

#### 9.1 节点定位

## 常见错误

**错误 1：跳过情绪解析直接写词**

表现：歌词写出来情绪单薄，从头到尾一个调子，没有层次和起伏，听众记不住也没有共鸣。

修复：必须先完成节点 1，把"不舍"这样的笼统情绪展开成有层次的子情绪和具体意象，歌词才有血肉。

**错误 2：段落标记机械套模板**

表现：用了 `[Verse][Chorus][Bridge]` 的标准标记，但每段内容功能不清，Bridge 只是又一段主歌，整首歌平铺直叙没有递进。

修复：执行节点 5（招牌节点），为每个段落明确情绪功能——Intro 建立氛围、Verse 叙事、Pre-Chorus 蓄力、Chorus 爆发、Bridge 转折、Outro 收束，让结构服务于情绪递进。

**错误 3：歌词只读不唱就交付**

表现：歌词读着挺顺，但 AI 生成出来发现很多字唱不上去，高音处全是闭口音，长句没有换气空间。

修复：执行节点 8 的演唱适配检查，确认高音区安排开口音、长句有换气点、没有难发音的辅音连缀。

## 延伸玩法

- **变体 1：30 秒短视频版**：保留 [Intro] + [Chorus] + [Outro]，省略主歌和桥段，用节点 5 重新组装一个 30 秒的精简结构，专门用于短视频 BGM。
- **变体 2：纯器乐版**：跳过歌词相关节点（3/4/8），用节点 1 情绪 + 节点 2 风格 + 节点 7 编曲，生成纯器乐背景音乐。
- **进阶组合**：将本工作流生成的歌曲作为"工作流 01（短视频创作工作流）"或"工作流 06（视频分镜工作流）"的配乐，用情绪主线统一歌曲和画面的情感节奏。
