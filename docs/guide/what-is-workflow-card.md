# 什么是工作流卡

工作流卡是本项目的核心内容单元。它不是一条提示词，也不是一篇教程，而是一套**完整的 AI 多专家协作流程**。

## 工作流卡 vs 提示词合集

| 维度 | 提示词合集 | 工作流卡 |
|------|-----------|---------|
| 结构 | 碎片化 | 完整流程 |
| 节点关系 | 孤立 | 有交接协议 |
| 输出 | 不确定 | 明确交付物 |
| 验收 | 无 | 人工验收清单 |
| 可重复性 | 低 | 高 |

## 工作流卡的 13 个组成部分

1. **工作流名称**：清晰描述这张卡解决什么工作
2. **适用岗位**：谁会用到这张卡
3. **真实场景**：一句话描述触发这个工作流的具体情境
4. **最终目标**：完成后交付什么
5. **输入材料**：开始前需要准备什么
6. **工作流总览**：节点流程示意（文字版流程图）
7. **专家节点设计**：每个节点的详细设计
8. **节点交接方式**：上游输出如何传递给下游
9. **最终输出模板**：交付物的标准格式
10. **常见错误**：执行中容易犯的错误
11. **人工验收清单**：人工审核要检查哪些项
12. **延伸玩法**：进阶用法和变体场景
13. **适用工具**：推荐使用的 AI 工具

## 专家节点的 7 个必备要素

每个专家节点必须包含：

- **职责**：这个节点负责什么
- **输入**：接收什么材料
- **输出**：产出什么内容
- **基础提示词**：普通用户直接复用
- **进阶提示词**：复杂场景使用
- **审查提示词**：检查输出质量
- **修正提示词**：基于审查结果二次修改
