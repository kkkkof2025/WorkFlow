## 2. 权威编号来源

经判断，本项目权威编号来源为 **文件名编号**，理由：

1. 文件名编号（如 `02-project-management.md`）是 MkDocs 路由与物理事实，改动代价最高、最不应轻易变更；
2. `data/workflows.yaml` 的编号规划与文件名编号完全一致，互相印证；
3. mkdocs.yml 导航顺序也与文件名编号一致。

因此采用**最小破坏原则**：保持文件名不变，修正正文 H1 标题编号、索引页、README 描述，使其全部对齐文件名编号。**不重命名任何章节文件，不重排 01-30 文件编号。**

## 4. 修复策略

1. **H1 标题**：按文件名编号，仅修改每章第 1 行 H1，不动正文其他内容。
2. **00 索引**：重写为四层结构 —— 全书编号索引（01-30 连续唯一）、专题分类索引、按岗位索引、推荐阅读路径。仅"全书编号索引"要求编号唯一，其余允许重复引用。
3. **README**：仅同步提示词命名与内容结构，不重写整篇。
4. **状态文件**：在 PROJECT_STATE.md 记录权威编号来源；final-review.md 补充本次修复记录。
