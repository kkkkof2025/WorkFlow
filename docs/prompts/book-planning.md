# 整书规划提示词

## 用途

用于规划整本书的章节结构、工作流分配和优先级排序。

---

## 提示词

```
你是一位资深的内容产品架构师。我正在创作一本名为《AI 工作流图鉴》的电子书，收录 30 个真实岗位的多专家 AI 协作工作流。

请根据以下信息，帮我规划全书章节结构：

**书名**：AI 工作流图鉴：30 个真实岗位的多专家协作方法
**目标读者**：职场人、内容创作者、程序员、运营人员、行政人员、项目经理、教师、工会宣传员
**核心定位**：不是提示词大全，而是把真实工作拆成可执行、可交接、可验收的 AI 多专家工作流

请为每个章节输出：
1. 章节名称
2. 章节定位（面向哪类读者，解决什么问题）
3. 包含的工作流列表（每条工作流包含：编号、名称、适用岗位、核心输出）
4. 章节写作重点

要求：
- 30 个工作流必须覆盖不同岗位，不重复
- 章节之间不能高度重叠
- 每个章节 3～5 个工作流
- 输出格式为 Markdown 表格 + 章节说明
```

---

## 进阶版（用于调整章节结构）

```
你是一位内容产品架构师。请审查以下章节结构草案，判断：

1. 哪些章节存在内容重叠？
2. 哪些读者群体被忽视了？
3. 章节顺序是否符合读者的阅读路径（从通用到专业）？
4. 30 个工作流是否均匀分布在各章节？

【粘贴当前章节草案】

请输出：问题列表 + 调整建议 + 修订后的章节结构。
```
