# 工作流卡生成提示词

## 用途

用于生成单个完整工作流卡的所有内容。

---

## 基础提示词

```
你是一位 AI 工作流设计师。请根据以下信息，生成一张完整的"工作流卡"。

**工作流名称**：【填入名称】
**适用岗位**：【填入岗位】
**真实场景**：【填入一句话场景描述】
**最终目标**：【填入交付物】

请严格按照以下结构生成：

1. 适用岗位
2. 真实场景
3. 最终目标
4. 输入材料清单（3～5 项）
5. 工作流总览（文字版流程图，5～10 个节点）
6. 每个专家节点（包含：职责、输入、输出、基础提示词、进阶提示词、审查提示词、修正提示词、验收标准）
7. 节点交接说明表格
8. 最终输出模板
9. 常见错误（3 条）
10. 人工验收清单（5 条）
11. 延伸玩法（2～3 条）

要求：
- 每个专家节点必须有 4 类提示词，不能省略
- 每条提示词必须能直接复制使用，不留占位符
- 验收标准每条必须是可判断的是/否问题
- 禁止使用"赋能""智能化""提升效率"等空洞词汇
```

---

## 进阶提示词（指定专家节点）

```
你是一位 AI 工作流设计师。请生成以下工作流卡，专家节点必须按照我指定的列表设计。

**工作流名称**：【填入名称】
**适用岗位**：【填入岗位】
**真实场景**：【填入场景】
**最终目标**：【填入交付物】

**专家节点列表**（必须按此顺序）：
1. 【节点 1 名称】
2. 【节点 2 名称】
3. 【节点 3 名称】
...

对每个节点，输出：
- 职责（不超过 30 字）
- 输入（上游来源）
- 输出（具体格式）
- 基础提示词（L1，50～100 字）
- 进阶提示词（L2，100～200 字）
- 审查提示词（L3，检查具体问题）
- 修正提示词（L4，基于审查结果重写）
- 验收标准（3～5 条是/否问题）

每个节点结束后，写一行"交接说明：[上游节点] → [下游节点]，交接内容：[具体内容]"。
```
